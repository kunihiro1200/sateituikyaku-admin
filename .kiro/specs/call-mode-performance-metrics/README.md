# 通話モード実績セクション機能

## 概要

通話モードページに「実績」セクションを追加し、月次の営業実績を可視化する機能です。

## 主な機能

1. **訪問査定取得数** - その月の訪問査定の数（訪問日を参照）
2. **訪問査定取得割合** - その月の依頼件数に対する訪問査定取得数の割合（確度D・ダブりを除く）
3. **専任件数（専任割合）** - 担当別の専任媒介件数と割合
4. **他決割合（未訪問）** - 営担が空欄で他決になった案件の割合
5. **他決割合（訪問済み）** - 担当別の訪問済み他決案件の割合

## ドキュメント

- [requirements.md](./requirements.md) - 要件定義
- [design.md](./design.md) - 設計ドキュメント
- [tasks.md](./tasks.md) - 実装タスクリスト

## 実装開始方法

1. `tasks.md`を開く
2. 各タスクの横にある「Start task」ボタンをクリック
3. タスクを順番に実装

## 技術スタック

- **Backend**: Node.js + Express + Supabase
- **Frontend**: React + TypeScript + Material-UI
- **Testing**: Jest + fast-check (Property-Based Testing)

## 注意事項

- テスト関連タスクはオプション（*マーク付き）としてマークされています
- コア機能の実装を優先し、後からテストを追加することも可能です
- 月次フィルタリング機能により、過去の実績も確認できます
