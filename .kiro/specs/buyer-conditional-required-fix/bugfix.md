# Bugfix Requirements Document

## Introduction

買主詳細画面（BuyerDetailPage.tsx）において、「持家ヒアリング結果」（`owned_home_hearing_result`）フィールドが無条件に必須項目（赤枠表示）になっているバグを修正する。

本来は「問合時持家ヒアリング」（`owned_home_hearing_inquiry`）に値がある場合のみ「持家ヒアリング結果」を必須にすべきだが、直前のデプロイで実装した必須バリデーション条件が誤っており、`owned_home_hearing_inquiry` の値に関わらず常に必須扱いになっている。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 「問合時持家ヒアリング」（`owned_home_hearing_inquiry`）が空欄の状態で買主詳細画面を開く THEN システムは「持家ヒアリング結果」フィールドを必須項目として赤枠表示する

1.2 WHEN 「問合時持家ヒアリング」が空欄の状態で保存ボタンを押す THEN システムは「持家ヒアリング結果」を未入力の必須項目として警告ダイアログに表示する

### Expected Behavior (Correct)

2.1 WHEN 「問合時持家ヒアリング」（`owned_home_hearing_inquiry`）が空欄の場合 THEN システムは「持家ヒアリング結果」フィールドを必須項目として扱わず、赤枠表示しない

2.2 WHEN 「問合時持家ヒアリング」に値が入力されており、かつ受付日が2026/3/30以降の場合 THEN システムは「持家ヒアリング結果」フィールドを必須項目として赤枠表示する

2.3 WHEN 「問合時持家ヒアリング」に値が入力されており、かつ受付日が2026/3/30以降の状態で保存ボタンを押す THEN システムは「持家ヒアリング結果」が未入力の場合に警告ダイアログに表示する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 「問合時持家ヒアリング」に値があり「持家ヒアリング結果」も入力済みの場合 THEN システムは引き続き「持家ヒアリング結果」フィールドを必須エラーなしで正常表示する

3.2 WHEN 受付日が2026/3/30より前の場合 THEN システムは「持家ヒアリング結果」フィールドを必須項目として扱わない（受付日条件は変更なし）

3.3 WHEN 「問合時持家ヒアリング」が空欄の場合 THEN システムは「持家ヒアリング結果」フィールド自体を非表示にする（既存の表示制御ロジックは変更なし）

3.4 WHEN その他の必須フィールド（`distribution_type`、`inquiry_source` 等）のバリデーション THEN システムは引き続き既存の必須チェックロジックを変更なく動作させる
