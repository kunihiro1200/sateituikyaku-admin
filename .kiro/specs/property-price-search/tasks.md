# 実装計画：物件リスト価格検索機能

## 概要

`PropertyListingsPage.tsx` のクライアントサイド検索フィルタに `price` フィールドを追加する。変更は最小限で、`filteredListings` useMemo 内の1行追加が主な実装となる。オプションとしてバックエンドの `PropertyListingService` にも同様の対応を行う。

## タスク

- [x] 1. フロントエンド：filteredListings の検索フィルタに price 条件を追加
  - `frontend/frontend/src/pages/PropertyListingsPage.tsx` の `filteredListings` useMemo を修正する
  - 既存の検索フィルタ末尾に `(l.price != null ? normalizeText(String(l.price)) : '').includes(query)` を追加する
  - `null` / `undefined` チェックに `!= null` を使用して両方を除外する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

  - [ ]* 1.1 price フィールド部分一致検索のプロパティテストを作成する
    - **Property 1: price フィールドによる部分一致検索**
    - **Validates: Requirements 1.1, 1.3, 1.4**
    - fast-check を使用して `applySearchFilter` ヘルパー関数を対象にテストを作成する
    - 任意の物件リストと数値クエリに対して、price にクエリが含まれる物件が全て結果に含まれることを検証する
    - price が null / undefined の物件でエラーが発生しないことを検証する

  - [ ]* 1.2 既存フィールドの後方互換性プロパティテストを作成する
