# バグ修正要件書：売主スプレッドシート重複行追加バグ

## Introduction

通話モードページで売主情報を編集して保存すると、たまに（10回に1回程度）スプレッドシートに同じ売主番号の行が重複して追加される問題が発生しています。売主番号（B列）はキーとして機能すべきであり、同じ売主番号が複数存在することは許容されません。

この問題は、スプレッドシート同期処理において既存行の検索に失敗し、更新ではなく新規追加として処理されていることが原因と推測されます。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 通話モードページ（`/sellers/:id/call`）でステータスセクションのフィールドを編集して保存する THEN システムはスプレッドシートに同じ売主番号（例: AA13888）の新しい行を追加する

1.2 WHEN スプレッドシート同期処理が既存行の検索に失敗する THEN システムは既存行を更新せず、新しい行を追加する

1.3 WHEN 同じ売主番号の行が複数存在する THEN 既存の行はそのまま残り、新しい行が追加される（重複）

### Expected Behavior (Correct)

2.1 WHEN 通話モードページで既存の売主情報を編集して保存する THEN システムはスプレッドシートの既存行を更新する（新しい行を追加しない）

2.2 WHEN スプレッドシート同期処理が実行される THEN システムは売主番号（B列）で既存行を正確に検索し、見つかった場合は更新、見つからない場合のみ新規追加する

2.3 WHEN スプレッドシート同期が完了する THEN 売主番号（B列）は常に一意である

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 新規売主を登録する THEN システムはスプレッドシートに新しい行を追加する

3.2 WHEN 既存行の検索が成功する THEN システムは既存行を正しく更新する

3.3 WHEN スプレッドシート同期処理が実行される THEN システムは他のフィールド（名前、電話番号、メールアドレスなど）を正しく同期する

3.4 WHEN 売主番号が空欄の売主を保存する THEN システムはエラーを返す（売主番号は必須）
