# バグ修正要件ドキュメント

## はじめに

買主詳細画面において、スプレッドシートの「業者向けアンケート」列が空欄の場合、DBには null または空として保存されるべきであり、フィールド自体が非表示になるべきである。しかし実際には、スプレッドシートが空欄であるにもかかわらず「業者向けアンケート」フィールドが「未」として表示されてしまうバグが発生している。買主ID 7319 で確認されている。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN スプレッドシートの「業者向けアンケート」列が空欄のとき THEN システムは「業者向けアンケート」フィールドを「未」として表示する

1.2 WHEN DBの「業者向けアンケート」フィールドが null または空文字のとき THEN システムは「業者向けアンケート」フィールドを「未」として表示する

### 期待される動作（正しい動作）

2.1 WHEN スプレッドシートの「業者向けアンケート」列が空欄のとき THEN システムは「業者向けアンケート」フィールドを表示しない（非表示）

2.2 WHEN DBの「業者向けアンケート」フィールドが null または空文字のとき THEN システムは「業者向けアンケート」フィールドを表示しない（非表示）

### 変更しない動作（リグレッション防止）

3.1 WHEN DBの「業者向けアンケート」フィールドに有効な値（例：「済」「未」など）が保存されているとき THEN システムは CONTINUE TO その値を「業者向けアンケート」フィールドとして正しく表示する

3.2 WHEN スプレッドシートの「業者向けアンケート」列に値が入力されているとき THEN システムは CONTINUE TO その値をDBに保存し、買主詳細画面に表示する

3.3 WHEN 他の買主詳細フィールド（氏名、電話番号など）が表示されているとき THEN システムは CONTINUE TO それらのフィールドを正しく表示する
