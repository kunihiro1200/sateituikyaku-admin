# Bugfix Requirements Document

## Introduction

売主AA13729を含む複数の売主で、訪問日（visit_date）と訪問時間（visit_time）が異常な形式で保存されており、訪問日前日カテゴリーに正しく表示されない問題が発生しています。

**影響範囲**:
- 訪問日前日カテゴリーの判定ロジック（`isVisitDayBefore`）が正しく動作しない
- サイドバーに訪問日前日の売主が表示されない
- 複数の売主が影響を受けている

**根本原因**:
- スプレッドシート → データベース同期時に、訪問日・訪問時間のパース処理が不適切
- 訪問日: `2026/04/04 1899/12/30` という異常な形式（日付が2つ連結）
- 訪問時間: `1899/12/30` という日付形式（時刻ではなく日付）

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN スプレッドシートから訪問日を同期する THEN システムは `2026/04/04 1899/12/30` という異常な形式でデータベースに保存する

1.2 WHEN スプレッドシートから訪問時間を同期する THEN システムは `1899/12/30` という日付形式でデータベースに保存する

1.3 WHEN 訪問日が異常な形式（`2026/04/04 1899/12/30`）で保存されている THEN `isVisitDayBefore` 関数は正しく前営業日を判定できず、訪問日前日カテゴリーに表示されない

1.4 WHEN 訪問時間が日付形式（`1899/12/30`）で保存されている THEN フロントエンドで訪問時間が正しく表示されない

### Expected Behavior (Correct)

2.1 WHEN スプレッドシートから訪問日を同期する THEN システムは `2026/04/04` という日付のみの形式（YYYY/MM/DD）でデータベースに保存する

2.2 WHEN スプレッドシートから訪問時間を同期する THEN システムは `10:00` という時刻のみの形式（HH:MM）でデータベースに保存する

2.3 WHEN 訪問日が正しい形式（`2026/04/04`）で保存されている THEN `isVisitDayBefore` 関数は正しく前営業日を判定し、訪問日前日カテゴリーに表示する

2.4 WHEN 訪問時間が正しい形式（`10:00`）で保存されている THEN フロントエンドで訪問時間が正しく表示される

2.5 WHEN 既存の異常データ（AA13729など）が存在する THEN システムは自動的に正しい形式に修正する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 訪問日が既に正しい形式（`2026/04/04`）で保存されている THEN システムはその形式を維持し続ける

3.2 WHEN 訪問時間が既に正しい形式（`10:00`）で保存されている THEN システムはその形式を維持し続ける

3.3 WHEN 訪問日・訪問時間以外のフィールドを同期する THEN システムはそれらのフィールドの同期処理を変更しない

3.4 WHEN 訪問日前日カテゴリー以外のサイドバーカテゴリーを判定する THEN システムはそれらのカテゴリーの判定ロジックを変更しない
