# 要件定義書: 買主「★最新状況」フィールド選択肢追加

## 1. 機能概要

買主リストの詳細画面にある「★最新状況」フィールドに、新しい選択肢「BZ：Bだが次電日不要」を追加する。

## 2. 背景と目的

### 2.1 背景

現在、「★最新状況」フィールドには「AZ:Aだが次電日不要」という選択肢が存在するが、Bステータスに対応する「次電日不要」の選択肢が存在しない。

### 2.2 目的

- Bステータス（1年以内に引っ越し希望だが、この物件ではない）の買主で、次回電話日の設定が不要なケースを明確に記録できるようにする
- AZと同様のパターンでBZを追加し、ステータス管理の一貫性を保つ

## 3. 機能要件

### 3.1 選択肢の追加

**要件ID**: REQ-001

**説明**: 「★最新状況」フィールドに「BZ：Bだが次電日不要」を追加する

**受入基準**:
- GIVEN 買主詳細ページを開いている
- WHEN 「★最新状況」フィールドのドロップダウンを開く
- THEN 「BZ：Bだが次電日不要」が選択肢として表示される

### 3.2 選択肢の位置

**要件ID**: REQ-002

**説明**: 「BZ：Bだが次電日不要」は「AZ:Aだが次電日不要」の直後に配置する

**受入基準**:
- GIVEN 「★最新状況」フィールドのドロップダウンを開いている
- WHEN 選択肢リストを確認する
- THEN 「AZ:Aだが次電日不要」の次に「BZ：Bだが次電日不要」が表示される

### 3.3 既存選択肢の保持

**要件ID**: REQ-003

**説明**: 既存の全ての選択肢は削除されず、そのまま保持される

**受入基準**:
- GIVEN 「★最新状況」フィールドのドロップダウンを開いている
- WHEN 選択肢リストを確認する
- THEN 以下の既存選択肢が全て表示される:
  - A:この物件を気に入っている（こちらからの一押しが必要）
  - B:1年以内に引っ越し希望だが、この物件ではない。駐車場の要件や、日当たり等が合わない。
  - C:引っ越しは1年以上先
  - D:配信・追客不要案件（業者や確度が低く追客不要案件等）
  - 買付外れました
  - 買（一般 両手）
  - 買（一般 片手）
  - 買（専任 両手）
  - 買（専任 片手）
  - 買（他社、片手）
  - 他決
  - 2番手
  - 3番手
  - AZ:Aだが次電日不要
  - 2番手買付提出済み
  - 3番手買付提出済み

### 3.4 保存機能

**要件ID**: REQ-004

**説明**: 「BZ：Bだが次電日不要」を選択して保存できる

**受入基準**:
- GIVEN 買主詳細ページで「★最新状況」フィールドを編集している
- WHEN 「BZ：Bだが次電日不要」を選択して保存する
- THEN データベースに正しく保存される
- AND ページをリロードしても「BZ：Bだが次電日不要」が表示される

### 3.5 新規買主登録での使用

**要件ID**: REQ-005

**説明**: 新規買主登録ページでも「BZ：Bだが次電日不要」を選択できる

**受入基準**:
- GIVEN 新規買主登録ページを開いている
- WHEN 「★最新状況」フィールドのAutocompleteを開く
- THEN 「BZ：Bだが次電日不要」が選択肢として表示される

### 3.6 内覧結果ページでの使用

**要件ID**: REQ-006

**説明**: 内覧結果ページでも「BZ：Bだが次電日不要」を選択できる

**受入基準**:
- GIVEN 内覧結果ページを開いている
- WHEN 「★最新状況」フィールドのドロップダウンを開く
- THEN 「BZ：Bだが次電日不要」が選択肢として表示される

## 4. 非機能要件

### 4.1 パフォーマンス

**要件ID**: NFR-001

**説明**: 選択肢の追加によるパフォーマンスへの影響はない

**受入基準**:
- GIVEN 「★最新状況」フィールドのドロップダウンを開く
- WHEN 選択肢リストが表示される
- THEN 表示速度は既存と同じである（体感的な遅延なし）

### 4.2 互換性

**要件ID**: NFR-002

**説明**: 既存のデータとの互換性を保つ

**受入基準**:
- GIVEN 既存の買主データが存在する
- WHEN 新しい選択肢を追加する
- THEN 既存のデータは影響を受けない
- AND 既存の選択肢で保存されたデータは正しく表示される

### 4.3 保守性

**要件ID**: NFR-003

**説明**: 将来的に選択肢を追加しやすい構造を維持する

**受入基準**:
- GIVEN 選択肢定義ファイル（buyerLatestStatusOptions.ts）を確認する
- WHEN 新しい選択肢を追加する
- THEN 配列に1行追加するだけで実装できる

## 5. 制約事項

### 5.1 データベーススキーマ

- `buyers.latest_status`カラムはTEXT型であり、変更不要
- バックエンドAPIでのバリデーションは不要（任意の値を受け入れる）

### 5.2 既存機能への影響

- 既存のコンポーネント（BuyerDetailPage、NewBuyerPage、BuyerViewingResultPage）は変更不要
- 選択肢定義ファイルのみを変更すれば、全てのページで自動的に反映される

## 6. 用語集

| 用語 | 説明 |
|------|------|
| ★最新状況 | 買主の現在のステータスを表すフィールド |

| BZ | Bステータス（1年以内に引っ越し希望だが、この物件ではない）で次電日不要 |

| 買主詳細ページ | 買主の詳細情報を表示・編集するページ |
| 新規買主登録ページ | 新しい買主を登録するページ |
| 内覧結果ページ | 内覧後の結果を記録するページ |
