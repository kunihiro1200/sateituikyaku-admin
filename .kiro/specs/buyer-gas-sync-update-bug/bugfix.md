# バグ修正要件定義書：買主リストGAS同期・既存レコード更新不具合

## Introduction

買主リストのGAS（Google Apps Script）において、スプレッドシートからDBへの同期処理で、**既存レコードのカラム値の更新（UPDATE）が正しく反映されない**不具合。

新規追加（INSERT）は動作しているが、既存レコードの値を後から変更した場合にDBへ反映されない。具体的な症状として以下が確認されている：

- 買主番号7243・7246の内覧日がスプレッドシートでは2026年4月になっているが、DBでは2026年6月のまま（1日以上経過しても更新されない）
- 買主番号7312の「【問合メール】メール返信」フィールドがスプレッドシートでは「済」になっているが、DBは空欄のまま（「問合せメール未対応」として誤表示される）

GASの同期は10分ごとに実行されているが、既存カラムの値変更が反映されない状態が継続している。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN スプレッドシートの既存買主レコードのカラム値（例：内覧日）が変更される THEN GASの10分トリガーによる同期処理はDBの該当カラムを更新せず、古い値がDBに残り続ける

1.2 WHEN 買主番号7243または7246の内覧日がスプレッドシートで2026年6月から2026年4月に変更される THEN DBの内覧日カラムは2026年6月のまま更新されない

1.3 WHEN 買主番号7312の「【問合メール】メール返信」フィールドがスプレッドシートで「済」に設定される THEN DBの対応カラムは空欄のまま更新されず、サイドバーの「問合せメール未対応」カテゴリーに誤って表示され続ける

1.4 WHEN GASの同期処理が既存レコードを検出する THEN UPDATE処理が正しく実行されず、スプレッドシートの最新値がDBに反映されない

### Expected Behavior (Correct)

2.1 WHEN スプレッドシートの既存買主レコードのカラム値が変更される THEN GASの次回10分トリガー実行時にDBの該当カラムが最新値に更新される

2.2 WHEN 買主番号7243または7246の内覧日がスプレッドシートで2026年4月に変更されている THEN DBの内覧日カラムも2026年4月に更新される

2.3 WHEN 買主番号7312の「【問合メール】メール返信」フィールドがスプレッドシートで「済」に設定されている THEN DBの対応カラムも「済」に更新され、「問合せメール未対応」カテゴリーから除外される

2.4 WHEN GASの同期処理が既存レコードを検出する THEN UPDATE処理が正しく実行され、スプレッドシートの全カラムの最新値がDBに反映される

### Unchanged Behavior (Regression Prevention)

3.1 WHEN スプレッドシートに新規買主が追加される THEN システムは CONTINUE TO 新規レコードをDBにINSERTする（新規追加の動作は維持される）

3.2 WHEN 既存買主レコードの値が変更されていない THEN システムは CONTINUE TO DBの値を変更せず、既存データを保持する

3.3 WHEN GASの10分トリガーが実行される THEN システムは CONTINUE TO `buyer-column-mapping.json`の`spreadsheetToDatabaseExtended`に定義された全カラムを対象に同期処理を実行する

3.4 WHEN 売主リストのスプレッドシート同期が実行される THEN システムは CONTINUE TO 売主データを従来通り正常に同期する（買主リスト同期の修正が売主リスト同期に影響しない）

3.5 WHEN 買主サイドバーカウント更新処理が実行される THEN システムは CONTINUE TO buyer_sidebar_counts テーブルを正しく更新する
