# バグ修正要件書

## Introduction

物件リスト詳細画面（PropertyListingDetailPage.tsx）のヘッダーにある「事務へCHAT」ボタン（BB14）を押すと、チャットメッセージが送信され、確認フィールドが「未」に更新される仕様です。しかし、以下の問題が発生しています：

1. サイドバーカテゴリーが「未完了」に即時更新されない
2. スプレッドシートのDQ列「確認」に即時同期されない
3. 10分後の自動同期でスプレッドシートの空欄（未同期）がDBに同期されて「未」が消える

この問題により、ユーザーは「事務へCHAT」を送信した後、物件リストページに戻ってもサイドバーカテゴリーが更新されておらず、手動でページをリロードする必要があります。また、10分後の自動同期でスプレッドシートの空欄がDBに上書きされ、確認フィールドが「未」から空欄に戻ってしまいます。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーが物件リスト詳細画面のヘッダーにある「事務へCHAT」ボタンを押す THEN フロントエンドのローカルstate（`confirmation`）のみが「未」に更新され、キャッシュクリアとイベント発火が行われない

1.2 WHEN 「事務へCHAT」送信後に物件リストページに戻る THEN サイドバーカテゴリーが「未完了」として表示されず、古いカテゴリーのままになる

1.3 WHEN 「事務へCHAT」送信後に物件リストページのサイドバーを確認する THEN 物件が「未完了」カテゴリーに表示されず、手動でページをリロードする必要がある

1.4 WHEN 「事務へCHAT」送信時にバックエンドで確認フィールドが「未」に更新される THEN スプレッドシートのDQ列「確認」に即座に同期されず、空欄のままになる

1.5 WHEN 「事務へCHAT」送信後10分が経過する THEN 自動同期でスプレッドシートの空欄がDBに同期され、確認フィールドが「未」から空欄に上書きされる

### Expected Behavior (Correct)

2.1 WHEN ユーザーが物件リスト詳細画面のヘッダーにある「事務へCHAT」ボタンを押す THEN 確認フィールドが「未」に更新され、キャッシュクリア、sessionStorageフラグ設定、カスタムイベント発火が実行される

2.2 WHEN 「事務へCHAT」送信後に物件リストページに戻る THEN サイドバーカテゴリーが即座に「未完了」として表示される

2.3 WHEN 「事務へCHAT」送信後に物件リストページのサイドバーを確認する THEN 物件が「未完了」カテゴリーに自動的に表示され、手動リロードが不要である

2.4 WHEN 「事務へCHAT」送信時にバックエンドで確認フィールドが「未」に更新される THEN スプレッドシートのDQ列「確認」に即座に「未」が同期される

2.5 WHEN 「事務へCHAT」送信後10分が経過して自動同期が実行される THEN スプレッドシートの「未」がDBに同期され、確認フィールドが「未」のまま維持される

2.6 WHEN ユーザーが確認フィールドを手動で「済」に変更する THEN サイドバーカテゴリーから物件が消え、「未完了」カテゴリーに表示されなくなる

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーが確認フィールドを手動で「未」または「済」に変更する THEN 既存の`handleUpdateConfirmation`メソッドの動作（キャッシュクリア、sessionStorageフラグ設定、カスタムイベント発火）が継続して正しく動作する

3.2 WHEN サイドバーステータス計算ロジックで確認フィールドが「未」の場合 THEN 「未完了」ステータスが返される動作が継続する

3.3 WHEN 「事務へCHAT」送信が成功する THEN チャットメッセージが営業担当または事務チャットに正しく送信される動作が継続する

3.4 WHEN 確認フィールドが「済」の物件がある THEN サイドバーカテゴリーに「未完了」として表示されない動作が継続する

3.5 WHEN 自動同期が実行される THEN スプレッドシートとDBの双方向同期が正しく動作し、手動で変更したデータが上書きされない動作が継続する
