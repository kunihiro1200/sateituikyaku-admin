# AA13149 買主候補リスト表示修正

## 問題の概要

AA13149の物件詳細ページで、買主候補リストが0件と表示される問題。
- データベースには116件の適格買主が存在することを確認済み
- バックエンドのフィルタリングロジックは正常に動作
- 再起動後に問題が発生

## 現状分析

### 確認済み事項
1. **データベース**: 116件の適格買主が存在
2. **バックエンドサービス**: `BuyerCandidateService.ts`のフィルタリングロジックは正常
3. **APIエンドポイント**: `/api/property-listings/:propertyNumber/buyer-candidates`が存在
4. **フロントエンドコンポーネント**: `BuyerCandidateList.tsx`は正常に実装されている

### 調査が必要な箇所
1. 実際のAPIレスポンスの確認
2. フロントエンドでのデータ受信状況
3. エラーログの確認
4. 再起動前後での動作の違い

## 想定される原因

### 仮説1: APIレスポンスの問題
- APIが正しくデータを返していない可能性
- エラーが発生しているが、フロントエンドで適切に処理されていない

### 仮説2: データ構造の不一致
- バックエンドとフロントエンドでデータ構造が一致していない
- 必須フィールドが欠けている

### 仮説3: 物件データの問題
- `property_listings`テーブルのAA13149のデータに問題がある
- `distribution_areas`や`property_type`が正しく設定されていない

## 次のステップ

1. **APIエンドポイントのテスト**
   - 実際にAPIを呼び出して、レスポンスを確認
   - エラーログを確認

2. **物件データの確認**
   - AA13149の`property_listings`データを確認
   - 必要なフィールドが正しく設定されているか確認

3. **フロントエンドのデバッグ**
   - ブラウザのコンソールログを確認
   - ネットワークタブでAPIリクエスト/レスポンスを確認

4. **修正実装**
   - 問題の原因に応じた修正を実施
