# バグ修正要件ドキュメント

## はじめに

売り主リスト（売主管理システム）の査定額計算ロジックにバグがある。
AA13965のような物件では「土地（当社調べ）」「建物（当社調べ）」フィールドに独自調査値が入力されている場合、これらの値を優先して査定額計算に使用すべきだが、現状では通常の土地面積フィールドから計算してしまっている。
このバグにより、独自調査値を入力した物件の査定額が誤った値で計算・表示される。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 「土地（当社調べ）」フィールドに値が入力されている場合 THEN システムは「土地（当社調べ）」の値を無視し、通常の土地面積フィールドの値を使って査定額を計算する

1.2 WHEN 「建物（当社調べ）」フィールドに値が入力されている場合 THEN システムは「建物（当社調べ）」の値を無視し、通常の建物面積フィールドの値を使って査定額を計算する

1.3 WHEN 「土地（当社調べ）」と「建物（当社調べ）」の両方に値が入力されている場合 THEN システムはどちらの値も無視し、通常の面積フィールドから査定額を計算する

### 期待される動作（正しい動作）

2.1 WHEN 「土地（当社調べ）」フィールドに値が入力されている場合 THEN システムは「土地（当社調べ）」の値を優先して査定額計算に使用する

2.2 WHEN 「建物（当社調べ）」フィールドに値が入力されている場合 THEN システムは「建物（当社調べ）」の値を優先して査定額計算に使用する

2.3 WHEN 「土地（当社調べ）」フィールドに値がない場合 THEN システムは通常の土地面積フィールドの値を使って査定額を計算する

2.4 WHEN 「建物（当社調べ）」フィールドに値がない場合 THEN システムは通常の建物面積フィールドの値を使って査定額を計算する

### 変更しない動作（リグレッション防止）

3.1 WHEN 「土地（当社調べ）」「建物（当社調べ）」の両方に値がない場合 THEN システムは引き続き通常の土地面積・建物面積フィールドから査定額を計算する

3.2 WHEN 通常の土地面積・建物面積フィールドのみに値がある場合 THEN システムは引き続きそれらの値を使って査定額を正しく計算する

3.3 WHEN 査定額計算以外の売主情報フィールド（氏名、電話番号、住所など）を操作する場合 THEN システムは引き続き既存の動作を維持する
