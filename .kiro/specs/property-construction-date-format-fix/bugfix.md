# バグ修正要件ドキュメント

## はじめに

物件リスト（売主管理システム）の物件詳細情報セクションにおいて、新築年月フィールドが日本語フォーマットではなく、JavaScriptのDateオブジェクトの生の文字列（例: `Sat Apr 01 1978 00:00:00 GMT+0900 (Japan Standard Time)`）で表示されているバグを修正する。スプレッドシート側では `2008年11月1日` 形式で正しく表示されているため、フロントエンドの表示処理に問題がある。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 物件詳細情報セクションの新築年月フィールドを表示する THEN システムは `Sat Apr 01 1978 00:00:00 GMT+0900 (Japan Standard Time)` のようなJavaScript Dateオブジェクトの生の文字列を表示する

1.2 WHEN 新築年月の値がDateオブジェクトまたはISO形式の日付文字列として渡される THEN システムはフォーマット処理を行わずにそのまま文字列として描画する

### 期待される動作（正常）

2.1 WHEN 物件詳細情報セクションの新築年月フィールドを表示する THEN システムは `2008年11月1日` のような日本語フォーマットで表示しなければならない（SHALL）

2.2 WHEN 新築年月の値がDateオブジェクトまたはISO形式の日付文字列として渡される THEN システムは日本語ロケール（`ja-JP`）を使用してフォーマットし、`YYYY年M月D日` 形式で表示しなければならない（SHALL）

### 変更されない動作（リグレッション防止）

3.1 WHEN 新築年月の値が空またはnullである THEN システムは引き続き空欄または「-」を表示しなければならない（SHALL CONTINUE TO）

3.2 WHEN 物件詳細情報セクションの他のフィールド（所在地、面積、価格など）を表示する THEN システムは引き続き既存の表示形式を維持しなければならない（SHALL CONTINUE TO）

3.3 WHEN スプレッドシートへの同期処理が実行される THEN システムは引き続き新築年月を `YYYY年M月D日` 形式でスプレッドシートに書き込まなければならない（SHALL CONTINUE TO）
