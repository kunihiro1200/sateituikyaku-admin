# 設計書

## 概要

買主リスト詳細画面の「メール・SMS送信履歴」セクションにおいて、送信元ラベルを削除し、表示レイアウトを最適化することで、縦長の表示を解消し、ユーザーが必要な情報を素早く確認できるようにします。

## データモデル

### ActivityLog

```typescript
interface ActivityLog {
  id: string;
  action: 'email' | 'sms' | ...;
  created_at: string;
  employee?: {
    name?: string;
    initials?: string;
  };
  metadata?: {
    source?: 'other_company_distribution' | 'pre_public_price_reduction' | 'buyer_candidate_list' | 'nearby_buyers';
    templateName?: string;
    subject?: string;
    senderName?: string;
    senderEmail?: string;
    phoneNumber?: string;
    propertyNumbers?: string[];
    property_numbers?: string[];
    propertyAddresses?: { [propertyNumber: string]: string };
    body?: string;
  };
}
```

### 変更点

- `metadata.source` は引き続き保存されるが、UIには表示しない
- その他のフィールドは変更なし

## パフォーマンス考慮事項

### 変更前

- 送信元ラベルの生成: `getSourceLabel()` と `getSourceColor()` を各履歴項目ごとに実行
- 表示要素: メール/SMS種別ラベル + 送信元ラベル + 件名 + 送信日時 + 送信者情報 + 物件情報

- 表示要素: メール/SMS種別ラベル + 件名 + 送信日時 + 送信者情報 + 物件情報
