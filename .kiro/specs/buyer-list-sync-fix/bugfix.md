# Bugfix Requirements Document

## Introduction

買主リストの最新状況がスプレッドシートと自動同期されないバグ。

売主リストには `EnhancedPeriodicSyncManager` による自動定期同期（10分ごと）が実装されているが、買主リストには同等の仕組みが存在しない。そのため、スプレッドシートで更新された買主データ（最新状況、次電日、担当者など）がデータベースに反映されず、ブラウザの買主リストに古い情報が表示され続ける。

---

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN サーバーが起動する THEN 買主リストの自動定期同期が開始されない（売主リストとは異なり、`BuyerSyncService` の定期実行がスケジュールされない）

1.2 WHEN スプレッドシートの買主データ（最新状況・次電日・担当者など）が更新される THEN データベースへの反映が行われない（手動同期またはUIボタン操作をしない限り）

1.3 WHEN ユーザーがブラウザで買主リストを閲覧する THEN スプレッドシートの最新状況ではなく、最後に手動同期した時点の古いデータが表示される

### Expected Behavior (Correct)

2.1 WHEN サーバーが起動する THEN 買主リストの自動定期同期が開始され、売主リストと同様に定期的にスプレッドシートからデータベースへ同期が実行される

2.2 WHEN スプレッドシートの買主データが更新される THEN 次回の定期同期タイミング（最大10分以内）にデータベースへ自動的に反映される

2.3 WHEN ユーザーがブラウザで買主リストを閲覧する THEN スプレッドシートの最新状況が表示される（最大10分の遅延は許容）

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーがUIの「スプレッドシートから同期」ボタンを押す THEN 手動同期が引き続き正常に実行される

3.2 WHEN 買主データの同期中に別の同期リクエストが来る THEN 重複同期が防止され、「Sync is already in progress」エラーが返される

3.3 WHEN 売主リストの自動定期同期が実行される THEN 買主リストの同期処理に影響を与えない（独立して動作する）

3.4 WHEN サーバーが起動する THEN 売主リストの自動定期同期（`EnhancedPeriodicSyncManager`）が引き続き正常に起動する
