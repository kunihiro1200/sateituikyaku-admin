# バグ修正要件ドキュメント

## はじめに

買主リスト詳細画面の「他社物件情報」セクションに2つのバグが存在する。

1つ目は、UIに「他社物件」の入力枠が1つしか表示されておらず、「建物名/価格」の入力枠が欠落している問題。
2つ目は、「他社物件」フィールドおよび「建物名/価格」フィールドを保存してもスプレッドシートに即時同期されない問題。

これらのバグにより、担当者が他社物件情報を正しく入力・管理できず、スプレッドシートとのデータ整合性が失われている。

## バグ分析

### 現在の動作（不具合）

**バグ1: UIの入力枠不足**

1.1 WHEN 買主リスト詳細画面の「他社物件情報」セクションを表示する THEN システムは「他社物件」の入力枠を1つのみ表示し、「建物名/価格」の入力枠が表示されない

1.2 WHEN 「他社物件情報」セクションを表示する THEN システムは「こちらは詳細な住所のみにしてください。お客様に物件情報として表示されます。他社名や価格は「建物名/価格」欄に書いてください。」という説明文を表示しない

**バグ2: スプレッドシート即時同期の不具合**

1.3 WHEN 「他社物件」フィールドに値を入力して保存する THEN システムはスプレッドシートのDJ列（カラム名「他社物件」）に値を反映しない

1.4 WHEN 「建物名/価格」フィールドに値を入力して保存する THEN システムはスプレッドシートのH列（カラム名「建物名/価格 内覧物件は赤表示（★は他社物件）」）に値を反映しない

### 期待される動作（正しい状態）

**バグ1の修正**

2.1 WHEN 買主リスト詳細画面の「他社物件情報」セクションを表示する THEN システムは「他社物件」テキストエリアと「建物名/価格」テキストエリアの2つの入力枠を表示しなければならない（SHALL）

2.2 WHEN 「他社物件情報」セクションを表示する THEN システムは「他社物件」テキストエリアの下に「こちらは詳細な住所のみにしてください。お客様に物件情報として表示されます。他社名や価格は「建物名/価格」欄に書いてください。」という説明文を表示しなければならない（SHALL）

2.3 WHEN 「他社物件情報」セクションを表示する THEN システムは「他社物件」テキストエリア（上）→ 説明文 → 「建物名/価格」テキストエリア（下）の順序で表示しなければならない（SHALL）

**バグ2の修正**

2.4 WHEN 「他社物件」フィールドに値を入力して保存する THEN システムはスプレッドシートのDJ列（カラム名「他社物件」）に即時同期しなければならない（SHALL）

2.5 WHEN 「建物名/価格」フィールドに値を入力して保存する THEN システムはスプレッドシートのH列（カラム名「建物名/価格 内覧物件は赤表示（★は他社物件）」）に即時同期しなければならない（SHALL）

### 変更してはならない動作（リグレッション防止）

3.1 WHEN 「他社物件情報」セクション以外の買主詳細フィールドを保存する THEN システムは引き続き既存のスプレッドシート同期処理を正常に実行しなければならない（SHALL CONTINUE TO）

3.2 WHEN 「他社物件」フィールドが空の状態で保存する THEN システムは引き続き他のフィールドの保存処理を正常に完了しなければならない（SHALL CONTINUE TO）

3.3 WHEN 「建物名/価格」フィールドが空の状態で保存する THEN システムは引き続き他のフィールドの保存処理を正常に完了しなければならない（SHALL CONTINUE TO）

3.4 WHEN 買主詳細画面の他のセクション（基本情報、希望条件など）を表示する THEN システムは引き続き既存のレイアウトと機能を正常に表示しなければならない（SHALL CONTINUE TO）
