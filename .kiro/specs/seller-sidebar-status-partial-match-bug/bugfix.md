# バグ修正要件書

## Introduction

売主リスト（SellerListPage）のサイドバーステータス判定において、状況（当社）が「他決→追客」の売主が「当日TEL分」カテゴリに含まれないバグを修正します。

現在の実装では、`isTodayCallBase`関数の`targetStatuses`配列に`'他決→追客'`が含まれているにもかかわらず、実際には「他決→追客」の売主が「当日TEL分」として認識されていません。

このバグにより、次電日が今日以前で追客が必要な売主（AA13755など）が「当日TEL分」リストから漏れ、営業活動に支障をきたしています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 状況（当社）が「他決→追客」であり、次電日が今日以前、コミュニケーション情報が全て空、営担が空の場合 THEN システムは当該売主を「当日TEL分」カテゴリに含めない

1.2 WHEN `targetStatuses`配列に`'他決→追客'`が定義されているにもかかわらず THEN システムは「他決→追客」の売主を正しく判定できない

### Expected Behavior (Correct)

2.1 WHEN 状況（当社）が「他決→追客」であり、次電日が今日以前、コミュニケーション情報が全て空、営担が空の場合 THEN システムは当該売主を「当日TEL分」カテゴリに含むべき

2.2 WHEN 状況（当社）に「追客」という文字列が含まれる場合（「追客中」「除外後追客中」「他決→追客」「専任→追客」など） THEN システムは追客対象として認識すべき

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 状況（当社）が「追客中」であり、次電日が今日以前、コミュニケーション情報が全て空、営担が空の場合 THEN システムは引き続き当該売主を「当日TEL分」カテゴリに含むべき

3.2 WHEN 状況（当社）が「除外後追客中」であり、次電日が今日以前、コミュニケーション情報が全て空、営担が空の場合 THEN システムは引き続き当該売主を「当日TEL分」カテゴリに含むべき

3.3 WHEN 状況（当社）に「追客」が含まれない場合（「他決」「専任」「除外」など） THEN システムは引き続き当該売主を「当日TEL分」カテゴリから除外すべき

3.4 WHEN コミュニケーション情報のいずれかに値が入っている場合 THEN システムは引き続き当該売主を「当日TEL（内容）」カテゴリに分類すべき

3.5 WHEN 営担に値が入っている場合 THEN システムは引き続き当該売主を「当日TEL（担当）」カテゴリに分類すべき

3.6 WHEN 状況（当社）が「追客不要」を含む場合 THEN システムは引き続き当該売主を全ての当日TELカテゴリから除外すべき
