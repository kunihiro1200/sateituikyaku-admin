# Bugfix Requirements Document

## Introduction

買主5641の内覧日・時間をブラウザUIで保存すると、データベースには正常に保存されるが、GASの定期同期（10分ごと、スプシ→DB）が実行されると内覧日・時間が白紙（null）に戻ってしまうバグを修正します。

根本原因は、DB→スプレッドシートの即時同期が機能していないため、スプレッドシートには古いデータ（空欄）が残ったままとなり、GASの定期同期がスプレッドシートの空欄をDBに上書きしてしまうことです。

このバグ修正により、ブラウザUIでの保存後、即座にスプレッドシートにも同期されるようになり、GASの定期同期が実行されても最新データが維持されます。

---

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーがブラウザUIで買主5641の内覧日・時間を保存する THEN システムはデータベースには正常に保存するが、スプレッドシートには同期しない

1.2 WHEN GASの定期同期（10分ごと、スプシ→DB）が実行される THEN システムはスプレッドシートの空欄（古いデータ）をデータベースに上書きし、内覧日・時間がnullに戻る

1.3 WHEN ブラウザUIで保存した内覧日・時間を確認する THEN システムは最初は正しい値を表示するが、10分後には白紙（null）になっている

### Expected Behavior (Correct)

2.1 WHEN ユーザーがブラウザUIで買主5641の内覧日・時間を保存する THEN システムはデータベースに保存し、即座にスプレッドシート（I列「●内覧日(最新）」、BP列「●時間」）にも同期する

2.2 WHEN GASの定期同期（10分ごと、スプシ→DB）が実行される THEN システムはスプレッドシートの最新データ（ブラウザUIで保存した値）をデータベースに反映し、内覧日・時間が維持される

2.3 WHEN ブラウザUIで保存した内覧日・時間を確認する THEN システムは保存直後も10分後も同じ正しい値を表示する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 他の買主フィールド（名前、電話番号など）をブラウザUIで保存する THEN システムは引き続き正常にデータベースとスプレッドシートの両方に同期する

3.2 WHEN GASの定期同期が内覧日・時間以外のフィールドを同期する THEN システムは引き続き正常に動作する

3.3 WHEN 買主5641以外の買主の内覧日・時間を保存する THEN システムは同様に即座にスプレッドシートに同期する

3.4 WHEN スプレッドシートで直接内覧日・時間を編集する THEN システムは引き続きGASの定期同期でデータベースに反映する
