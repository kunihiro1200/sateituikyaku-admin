# 要件定義書

## はじめに

売主管理システムの通話モードページ（CallModePage）の左列にある「メール・SMS履歴」セクションにおいて、各ログエントリをクリックすることで、メール本文またはSMSメッセージ内容を展開表示できる機能を追加する。

現状では、各ログエントリには送信日時・担当者・種別（Email/SMS）のみが表示されており、実際の送信内容（件名・本文）を確認するには別の手段が必要な状態である。本機能により、担当者が通話中に過去の送信内容を素早く確認できるようになる。

## 用語集

- **CallModePage**: 売主管理システムの通話モードページ（`/sellers/:id/call`）
- **メール・SMS履歴セクション**: CallModePageの左列に表示される、過去のメール・SMS送信履歴の一覧
- **ログエントリ**: メール・SMS履歴セクション内の各送信記録（1件の送信に対応する1行）
- **Activity**: バックエンドの`activities`テーブルに保存された活動記録。`type`（email/sms）、`content`（送信内容の概要）、`metadata`（件名・本文などの詳細情報）を持つ
- **metadata**: Activityレコードの詳細情報フィールド。メールの場合は`subject`（件名）と`body`（本文）を含む
- **展開パネル**: ログエントリをクリックしたときに表示される、本文内容を表示するUI領域
- **System**: 売主管理システムのフロントエンド（`frontend/frontend/`）

## 要件

### 要件1: ログエントリのクリックによる本文展開

**ユーザーストーリー:** 担当者として、メール・SMS履歴のログエントリをクリックしたとき、送信した本文内容を確認したい。そうすることで、通話中に過去の送信内容を素早く把握できる。

#### 受け入れ基準

1. WHEN ユーザーがメール・SMS履歴セクションのログエントリをクリックしたとき、THE System SHALL そのエントリの展開パネルを表示する
2. WHEN ユーザーが展開済みのログエントリを再度クリックしたとき、THE System SHALL 展開パネルを折りたたむ
3. THE System SHALL 複数のログエントリの展開状態を独立して管理する（あるエントリを展開しても他のエントリの状態は変わらない）
4. WHEN ログエントリが展開されているとき、THE System SHALL 展開状態を視覚的に示すインジケーター（矢印アイコンなど）を表示する

### 要件2: メール本文の表示

**ユーザーストーリー:** 担当者として、メール送信履歴のエントリを展開したとき、送信したメールの件名と本文を確認したい。そうすることで、売主に何を伝えたかを通話中に確認できる。

#### 受け入れ基準

1. WHEN メール種別（type = 'email'）のログエントリが展開されたとき、THE System SHALL `metadata.subject`フィールドの値を件名として表示する
2. WHEN メール種別のログエントリが展開されたとき、THE System SHALL `metadata.body`フィールドの値を本文として表示する
3. IF `metadata.body`が存在しないとき、THEN THE System SHALL 「本文データなし（旧形式）」というメッセージを表示する
4. IF `metadata.subject`が存在しないとき、THEN THE System SHALL 件名欄を省略して本文のみを表示する
5. WHEN メール本文にHTMLタグが含まれているとき、THE System SHALL HTMLをレンダリングせずプレーンテキストとして表示する（XSS防止）

### 要件3: SMSメッセージ内容の表示

**ユーザーストーリー:** 担当者として、SMS送信履歴のエントリを展開したとき、送信したSMSの内容を確認したい。そうすることで、売主に何を送ったかを通話中に確認できる。

#### 受け入れ基準

1. WHEN SMS種別（type = 'sms'）のログエントリが展開されたとき、THE System SHALL `content`フィールドの値をメッセージ内容として表示する
2. IF `content`が空文字列または未設定のとき、THEN THE System SHALL 「メッセージ内容なし」というメッセージを表示する

### 要件4: 展開パネルの表示形式

**ユーザーストーリー:** 担当者として、展開パネルが読みやすい形式で表示されることを期待する。そうすることで、通話中でも素早く内容を把握できる。

#### 受け入れ基準

1. THE System SHALL 展開パネルを既存のログエントリカードの直下に表示する
2. THE System SHALL 展開パネルの最大高さを制限し（例: 200px）、内容が長い場合はスクロール可能にする
3. THE System SHALL メール本文の改行を保持して表示する（`white-space: pre-wrap`相当）
4. THE System SHALL 展開パネルの背景色をログエントリカードと区別できる色で表示する（例: 白背景）
5. WHERE メール種別のとき、THE System SHALL 件名と本文を視覚的に区別できるラベル（「件名:」「本文:」）を付けて表示する
