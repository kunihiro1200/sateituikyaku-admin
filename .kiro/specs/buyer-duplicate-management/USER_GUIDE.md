# 買主関連表示機能 - ユーザーガイド

## 概要

買主関連表示機能は、同じ電話番号またはメールアドレスを持つ買主を自動的に検出し、関連情報として表示する機能です。この機能により、以下のことが可能になります：

- 同じ人物による複数の物件への問合せを把握
- 同じ物件への重複問合せを検出
- 関連する買主の問合せ履歴を統合して表示

## 重要な注意事項

⚠️ **この機能はデータを自動的に変更しません**

- 買主レコードの自動統合は行いません
- 買主レコードの自動削除は行いません
- 関連情報の表示のみを行います

## 機能の使い方

### 1. 関連買主の確認

買主詳細ページを開くと、以下の情報が表示されます：

#### ヘッダーの通知バッジ

買主名の横に「関連買主」バッジが表示されます：

```
田中太郎 [関連買主: 2件]
```

- バッジをクリックすると、ページ下部の関連買主セクションにスクロールします
- 関連買主がいない場合、バッジは表示されません

#### 関連買主セクション

ページ下部に「関連買主」セクションが表示されます：

**複数問合せの例：**
```
📋 複数問合せ
買主番号: 2065
氏名: 田中太郎
物件番号: AA13130
マッチ理由: 電話番号が一致
```

**重複の可能性の例：**
```
⚠️ 重複の可能性
買主番号: 2066
氏名: 田中太郎
物件番号: AA13129
マッチ理由: 電話番号とメールアドレスが一致

⚠️ 警告: 同じ物件への重複問合せの可能性があります。
スプレッドシートを確認し、必要に応じて重複レコードを削除してください。
```

### 2. 統合問合せ履歴の確認

関連買主セクションの下に「統合問合せ履歴」が表示されます：

```
統合問合せ履歴

買主番号 | 問合せ日 | 物件番号 | 物件住所
---------|----------|----------|----------
2064     | 2024-12-15 | AA13129 | 大分市中央町1-1-1
2065     | 2024-12-10 | AA13130 | 大分市府内町2-2-2
2064     | 2024-12-05 | AA13131 | 大分市金池町3-3-3
```

- 全関連買主の問合せ履歴が統合されて表示されます
- 日付順（新しい順）にソートされます
- 買主番号ごとに色分けされます
- 物件番号をクリックすると、物件詳細ページに移動します

## 関係タイプの理解

### 📋 複数問合せ（Multiple Inquiry）

**定義：** 同じ人物が異なる物件に問合せをしている状態

**特徴：**
- 物件番号が異なる
- 正常な状態（問題なし）
- 青色のラベルで表示

**例：**
- 買主A（物件AA13129）と買主B（物件AA13130）が同じ電話番号
- 同じ人物が複数の物件に興味を持っている

**対応：**
- 特に対応は不要です
- 顧客の興味範囲を把握するための情報として活用できます

### ⚠️ 重複の可能性（Possible Duplicate）

**定義：** 同じ人物が同じ物件に複数回問合せをしている可能性がある状態

**特徴：**
- 物件番号が同じ
- 重複問合せの可能性（要確認）
- 黄色のラベルで表示
- 警告メッセージが表示される

**例：**
- 買主A（物件AA13129）と買主B（物件AA13129）が同じ電話番号
- 同じ人物が同じ物件に2回問合せをした可能性

**対応：**
1. スプレッドシートを確認
2. 本当に重複かどうかを判断
3. 重複の場合、スプレッドシートから1つを削除
4. 次回の同期で自動的にデータベースから削除されます

## マッチ理由の種類

関連買主は以下の条件で検出されます：

### 電話番号が一致
```
マッチ理由: 電話番号が一致
```
- 同じ電話番号を持つ買主が検出されました

### メールアドレスが一致
```
マッチ理由: メールアドレスが一致
```
- 同じメールアドレスを持つ買主が検出されました

### 両方が一致
```
マッチ理由: 電話番号とメールアドレスが一致
```
- 電話番号とメールアドレスの両方が一致する買主が検出されました
- 同一人物である可能性が非常に高いです

## 重複問合せへの対応手順

### ステップ1: 重複の確認

1. 買主詳細ページで「⚠️ 重複の可能性」ラベルを確認
2. 以下の情報を比較：
   - 氏名
   - 電話番号
   - メールアドレス
   - 問合せ日
   - 物件番号

### ステップ2: スプレッドシートの確認

1. Googleスプレッドシート「買主管理」を開く
2. 該当する買主番号を検索
3. 重複レコードを特定

### ステップ3: 重複レコードの削除

**重要：** 削除は慎重に行ってください

1. 保持するレコードを決定：
   - より情報が完全なレコード
   - より早い問合せ日のレコード
   - より詳細なコメントがあるレコード

2. 削除するレコードを選択：
   - スプレッドシートの行全体を選択
   - 右クリック → 「行を削除」

3. 削除の確認：
   - 削除したレコードの買主番号をメモ
   - 削除理由をコメントに記録（推奨）

### ステップ4: 自動同期の確認

1. 次回の自動同期（30分ごと）を待つ
2. または、手動で同期を実行：
   - 管理画面 → 「スプレッドシート同期」
   - 「今すぐ同期」ボタンをクリック

3. 同期後、買主詳細ページを再読み込み
4. 重複が解消されていることを確認

## よくある質問（FAQ）

### Q1: 複数問合せと重複の違いは何ですか？

**A:** 物件番号が異なるかどうかです：
- **複数問合せ**: 異なる物件への問合せ（正常）
- **重複**: 同じ物件への問合せ（要確認）

### Q2: 重複を自動的に削除できませんか？

**A:** いいえ、自動削除は行いません。理由：
- 誤って正しいデータを削除するリスクを避けるため
- 人間の判断が必要な場合があるため
- データの整合性を保つため

### Q3: 関連買主が表示されないのはなぜですか？

**A:** 以下の理由が考えられます：
- 電話番号もメールアドレスも一致する買主がいない
- 電話番号・メールアドレスが空欄
- データベースのインデックスが作成されていない

### Q4: 統合問合せ履歴に表示される順番は？

**A:** 問合せ日の新しい順に表示されます。問合せ日が空欄の場合は最後に表示されます。

### Q5: 買主番号の色分けは何を意味しますか？

**A:** 異なる買主を視覚的に区別するためのものです。色自体に特別な意味はありません。

### Q6: 関連買主の検出は自動ですか？

**A:** はい、買主詳細ページを開くたびに自動的に検出されます。

### Q7: 重複を削除した後、すぐに反映されますか？

**A:** 次回の自動同期（30分ごと）または手動同期の実行後に反映されます。

### Q8: 同じ電話番号でも別人の場合はどうすればいいですか？

**A:** 特に対応は不要です。システムは「可能性」として表示するだけで、自動的にデータを変更することはありません。

## トラブルシューティング

### 問題: 関連買主が表示されない

**原因と対処法：**

1. **電話番号・メールアドレスが空欄**
   - スプレッドシートで該当買主の情報を確認
   - 電話番号またはメールアドレスを入力
   - 同期を実行

2. **インデックスが作成されていない**
   - 管理者に連絡してインデックスの作成を依頼

3. **ブラウザのキャッシュ**
   - ページを再読み込み（Ctrl+F5 または Cmd+Shift+R）
   - ブラウザのキャッシュをクリア

### 問題: 統合問合せ履歴が表示されない

**原因と対処法：**

1. **関連買主がいない**
   - 関連買主セクションを確認
   - 関連買主がいない場合、統合履歴も表示されません

2. **問合せ履歴がない**
   - 該当買主の問合せ履歴を確認
   - スプレッドシートで問合せ日が入力されているか確認

### 問題: パフォーマンスが遅い

**原因と対処法：**

1. **大量の買主データ**
   - 管理者に連絡してパフォーマンス最適化を依頼
   - インデックスの確認を依頼

2. **ネットワークの問題**
   - インターネット接続を確認
   - ページを再読み込み

## ベストプラクティス

### 1. 定期的な重複チェック

- 週に1回程度、重複の可能性がある買主を確認
- 重複が見つかった場合は速やかに対応

### 2. データの正確性

- 電話番号・メールアドレスを正確に入力
- 全角・半角の統一
- ハイフンの有無の統一

### 3. コメントの活用

- 重複を削除した場合、理由をコメントに記録
- 複数問合せの場合、顧客の興味範囲をコメントに記録

### 4. 統合履歴の活用

- 顧客の問合せパターンを把握
- 興味のある物件タイプを分析
- フォローアップの優先順位付けに活用

## サポート

問題が解決しない場合や質問がある場合は、システム管理者に連絡してください。
