# Bugfix Requirements Document

## Introduction

売主リストのサイドバーカテゴリー「当日TEL分」に、AA12674が誤って含まれているバグ。
AA12674の次電日は2026/7/18であり、今日以前ではないため「当日TEL分」の条件を満たさないはず。
このバグには2つの側面がある：
1. スプレッドシートで次電日が更新されたにもかかわらず、GASによるスプレッドシート→DB同期が1時間以上経過しても反映されていない問題
2. サイドバーカテゴリーの計算がGASの同期結果に依存しており、DBで直接値を変更した場合にGASの次回同期まで反映されない問題

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 売主AA12674の次電日がスプレッドシートで2026/7/18（今日より未来）に更新されてから1時間以上経過した後にサイドバーを表示する THEN システムはAA12674を「当日TEL分」カテゴリーに含めて表示する

1.2 WHEN スプレッドシートの次電日フィールドが更新されてから10分以上経過した後にDBの該当売主レコードを確認する THEN システムはDBの`next_call_date`に古い値（今日以前の日付）を保持したままにする

1.3 WHEN DBで売主の`next_call_date`を直接今日より未来の日付に変更した後にサイドバーを表示する THEN システムはGASの次回同期タイミングまでその売主を「当日TEL分」カテゴリーに含めたまま表示する

### Expected Behavior (Correct)

2.1 WHEN 売主AA12674の次電日がスプレッドシートで2026/7/18（今日より未来）に更新されてから最大10分以内にサイドバーを表示する THEN システムはAA12674を「当日TEL分」カテゴリーに含めないこと（次電日が今日以前という条件を満たさないため）

2.2 WHEN スプレッドシートの次電日フィールドが更新されてから最大10分以内にDBの該当売主レコードを確認する THEN システムはDBの`next_call_date`に更新後の値（2026/7/18）を反映していること

2.3 WHEN DBで売主の`next_call_date`を直接今日より未来の日付に変更した後にサイドバーを表示する THEN システムはGASの同期タイミングに関係なく、DBの現在値に基づいてその売主を「当日TEL分」カテゴリーから即時除外すること

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 次電日が今日以前の売主（例：次電日が昨日以前）がサイドバーに表示される THEN システムはその売主を「当日TEL分」カテゴリーに引き続き含めること（他の条件：追客中・コミュニケーション情報が全て空・営担なし、も満たす場合）

3.2 WHEN GASの`syncSellerList`が10分トリガーで実行される THEN システムはスプレッドシートの全売主データをDBに正常に同期し続けること

3.3 WHEN 次電日以外のフィールド（状況、営担など）がスプレッドシートで更新される THEN システムはそれらのフィールドも正常にDBへ同期し続けること
