# Requirements Document

## Introduction

買主リストの「問合時持家ヒアリング」フィールドに「不要」と「未」の選択肢を追加し、これらの選択肢が選択された場合に「持家ヒアリング結果」フィールドを必須ではなくする機能を実装します。

現在、「問合時持家ヒアリング」フィールドにはイニシャル選択肢（Y、I、K など）のみが存在し、「持家ヒアリング結果」の必須判定は「問合時持家ヒアリング」に値があるかどうかで決定されています。この機能により、「不要」または「未」を選択した場合は「持家ヒアリング結果」を入力不要にすることができます。

## Glossary

- **買主リスト**: 買主情報を管理するスプレッドシートおよびデータベーステーブル（`buyers`）
- **問合時持家ヒアリング**: 買主に対して持家の状況をヒアリングしたかどうかを記録するフィールド（データベースカラム: `owned_home_hearing_inquiry`）
- **持家ヒアリング結果**: 持家ヒアリングの結果を記録するフィールド（データベースカラム: `owned_home_hearing_result`）
- **BuyerDetailPage**: 買主詳細ページ（`frontend/frontend/src/pages/BuyerDetailPage.tsx`）
- **条件付き必須**: 特定の条件を満たす場合のみ必須となるバリデーションルール
- **イニシャル選択肢**: Y、I、K などの担当者イニシャルを表す選択肢
- **staffSelect**: イニシャル選択ボタンを表示するフィールドタイプ

## Requirements

### Requirement 1: 「問合時持家ヒアリング」フィールドに選択肢を追加

**User Story:** As a ユーザー, I want 「問合時持家ヒアリング」フィールドに「不要」と「未」の選択肢を追加したい, so that 持家ヒアリングが不要な場合や未実施の場合を明示的に記録できる

#### Acceptance Criteria

1. WHEN 買主詳細ページの「問合時持家ヒアリング」フィールドを表示する THEN システムは既存のイニシャル選択肢（Y、I、K など）に加えて「不要」と「未」の選択肢を表示する

2. WHEN ユーザーが「不要」ボタンをクリックする THEN システムは「問合時持家ヒアリング」フィールドに「不要」を保存し、ボタンを選択状態（強調表示）にする

3. WHEN ユーザーが「未」ボタンをクリックする THEN システムは「問合時持家ヒアリング」フィールドに「未」を保存し、ボタンを選択状態（強調表示）にする

4. WHEN ユーザーが選択済みの「不要」または「未」ボタンを再度クリックする THEN システムは選択を解除し、「問合時持家ヒアリング」フィールドを空欄にする

5. WHEN スプレッドシートの「問合時持家ヒアリング」列に「不要」または「未」が入力されている THEN システムはデータベースに正しく同期し、買主詳細ページで対応するボタンを選択状態で表示する

### Requirement 2: 「持家ヒアリング結果」の条件付き必須ロジックを更新

**User Story:** As a ユーザー, I want 「不要」または「未」を選択した場合に「持家ヒアリング結果」を必須ではなくしたい, so that 不要な入力を強制されずに済む

#### Acceptance Criteria

1. WHEN 「問合時持家ヒアリング」が「不要」の場合 THEN システムは「持家ヒアリング結果」フィールドを必須扱いにせず、赤枠を表示しない

2. WHEN 「問合時持家ヒアリング」が「未」の場合 THEN システムは「持家ヒアリング結果」フィールドを必須扱いにせず、赤枠を表示しない

3. WHEN 「問合時持家ヒアリング」がイニシャル選択肢（Y、I、K など）の場合 THEN システムは「持家ヒアリング結果」フィールドを必須扱いにし、未入力の場合は赤枠を表示する

4. WHEN ユーザーが「問合時持家ヒアリング」を「不要」または「未」に変更する THEN システムは即座に「持家ヒアリング結果」の必須状態を解除し、赤枠を削除する

5. WHEN ユーザーが「問合時持家ヒアリング」をイニシャル選択肢に変更する THEN システムは即座に「持家ヒアリング結果」を必須扱いにし、未入力の場合は赤枠を表示する

6. WHEN ユーザーが保存ボタンを押す THEN システムは「問合時持家ヒアリング」が「不要」または「未」の場合、「持家ヒアリング結果」が未入力でも警告ダイアログに表示しない

### Requirement 3: データベースとスプレッドシートの同期

**User Story:** As a システム, I want 「不要」と「未」の値をデータベースとスプレッドシートで正しく同期したい, so that データの整合性を保つ

#### Acceptance Criteria

1. WHEN 買主詳細ページで「問合時持家ヒアリング」を「不要」に設定して保存する THEN システムはデータベースの `owned_home_hearing_inquiry` カラムに「不要」を保存する

2. WHEN 買主詳細ページで「問合時持家ヒアリング」を「未」に設定して保存する THEN システムはデータベースの `owned_home_hearing_inquiry` カラムに「未」を保存する

3. WHEN データベースの `owned_home_hearing_inquiry` が「不要」または「未」の場合 THEN システムはスプレッドシートの「問合時持家ヒアリング」列に「不要」または「未」を同期する

4. WHEN スプレッドシートの「問合時持家ヒアリング」列が「不要」または「未」の場合 THEN システムはデータベースの `owned_home_hearing_inquiry` カラムに「不要」または「未」を同期する

5. FOR ALL 「不要」または「未」の値において、データベース → スプレッドシート → データベースの往復同期を実行した場合、元の値と同じ値が保持される（Round-trip property）

### Requirement 4: 既存機能の保持

**User Story:** As a ユーザー, I want 既存の「問合時持家ヒアリング」と「持家ヒアリング結果」の動作を維持したい, so that 他の機能に影響を与えない

#### Acceptance Criteria

1. WHEN 「問合時持家ヒアリング」が既存のイニシャル選択肢（Y、I、K など）の場合 THEN システムは引き続き「持家ヒアリング結果」を必須扱いにする

2. WHEN 「問合時持家ヒアリング」が空欄の場合 THEN システムは引き続き「持家ヒアリング結果」を必須扱いにしない

3. WHEN 「問合時持家ヒアリング」が空白文字のみの場合 THEN システムは引き続き「持家ヒアリング結果」を必須扱いにしない

4. WHEN 他の条件付き必須フィールド（初動担当、★最新状況など）を表示・編集する THEN システムは引き続き正しく動作する

5. WHEN 買主詳細ページの他のフィールドを編集・保存する THEN システムは引き続き正しく動作する
