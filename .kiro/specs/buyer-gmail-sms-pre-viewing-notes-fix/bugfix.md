# バグ修正要件ドキュメント

## はじめに

買主リストのGmail送信・SMS送信機能において、「内覧前伝達事項」（`<<内覧前伝達事項v>>`プレースホルダー）のテキストが送信内容に含まれないバグを修正する。

コード調査の結果、根本原因は以下の2点：

1. **Gmail送信（メールテンプレート）**: `EmailTemplateService.mergeAngleBracketPlaceholders()` 内で `<<内覧前伝達事項v>>` が**無条件に空文字へ置換**されている（`backend/src/services/EmailTemplateService.ts` 約340行目）
2. **SMS送信**: `SmsDropdownButton.tsx` のメッセージテンプレート文字列に内覧前伝達事項の差し込みロジックが**存在しない**

---

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 買主の内覧前伝達事項フィールドに値が入力されており、Gmail送信テンプレートに `<<内覧前伝達事項v>>` プレースホルダーが含まれている THEN システムはそのプレースホルダーを空文字に置換し、内覧前伝達事項の内容がメール本文に含まれない

1.2 WHEN 買主の内覧前伝達事項フィールドに値が入力されており、SMS送信テンプレートを選択した THEN システムは内覧前伝達事項の内容をSMSメッセージに含めない

### 期待される動作（正しい動作）

2.1 WHEN 買主の内覧前伝達事項フィールドに値が入力されており、Gmail送信テンプレートに `<<内覧前伝達事項v>>` プレースホルダーが含まれている THEN システムは `<<内覧前伝達事項v>>` をその買主の内覧前伝達事項テキストで置換し、メール本文に含めるものとする（SHALL）

2.2 WHEN 買主の内覧前伝達事項フィールドが空であり、Gmail送信テンプレートに `<<内覧前伝達事項v>>` プレースホルダーが含まれている THEN システムは `<<内覧前伝達事項v>>` を空文字に置換するものとする（SHALL）

2.3 WHEN 買主の内覧前伝達事項フィールドに値が入力されており、SMS送信テンプレートを選択した THEN システムは内覧前伝達事項の内容をSMSメッセージの適切な位置に含めるものとする（SHALL）

2.4 WHEN 買主の内覧前伝達事項フィールドが空であり、SMS送信テンプレートを選択した THEN システムは内覧前伝達事項のセクションを省略し、既存のメッセージ内容を変更しないものとする（SHALL）

### 変更しない動作（リグレッション防止）

3.1 WHEN 内覧前伝達事項フィールドが空の買主に対してGmail送信を行う THEN システムは引き続き他のプレースホルダー（`<<氏名>>`、`<<住居表示>>`、`<<GoogleMap>>`等）を正常に置換し続けるものとする（SHALL CONTINUE TO）

3.2 WHEN 内覧前伝達事項フィールドが空の買主に対してSMS送信を行う THEN システムは引き続き既存のSMSテンプレートメッセージを変更なく送信し続けるものとする（SHALL CONTINUE TO）

3.3 WHEN Gmail送信テンプレートに `<<内覧前伝達事項v>>` プレースホルダーが含まれていない THEN システムは引き続きメール本文を変更なく処理し続けるものとする（SHALL CONTINUE TO）

3.4 WHEN SMS送信履歴の記録処理を行う THEN システムは引き続き既存のSMS履歴記録ロジックを変更なく動作させ続けるものとする（SHALL CONTINUE TO）

3.5 WHEN `mergeAngleBracketPlaceholders()` が呼び出される THEN システムは引き続き `<<SUUMO　URLの表示>>` を空文字に置換し続けるものとする（SHALL CONTINUE TO）
