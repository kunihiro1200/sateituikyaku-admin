# Requirements Document

## Introduction

コールモードページの実績セクションは現在、文字の羅列で表示されており、視覚的に理解しにくい状態です。数値データを視覚的に分かりやすく表示することで、ユーザーが実績を一目で把握できるようにします。

## Glossary

- **実績セクション**: コールモードページに表示される月次パフォーマンス指標のセクション
- **カード形式**: 各指標を独立したカードとして表示するレイアウト
- **プログレスバー**: 目標値に対する達成度を視覚的に示すバー
- **カラーコーディング**: 達成状況に応じて色分けする手法（緑=良好、黄=注意、赤=要改善）

## Requirements

### Requirement 1

**User Story:** ユーザーとして、実績データを視覚的に理解しやすい形式で見たい。そうすることで、パフォーマンスを素早く把握できる。

#### Acceptance Criteria

1. WHEN 実績セクションを表示する THEN システムは各指標をカード形式で表示する
2. WHEN 目標値がある指標を表示する THEN システムはプログレスバーで達成度を視覚化する
3. WHEN 達成度を表示する THEN システムは目標達成状況に応じて色分けする（100%以上=緑、80-99%=黄、80%未満=赤）
4. WHEN 数値を表示する THEN システムは大きく読みやすいフォントサイズで表示する
5. WHEN 複数の担当者データを表示する THEN システムは見やすいテーブル形式で表示する

### Requirement 2

**User Story:** ユーザーとして、現在値と平均値の比較を視覚的に理解したい。そうすることで、今月のパフォーマンスが平均と比べてどうかを把握できる。

#### Acceptance Criteria

1. WHEN 現在値と月平均を表示する THEN システムは両方の値を並べて表示する
2. WHEN 現在値が月平均より高い THEN システムは上向き矢印アイコンと緑色で表示する
3. WHEN 現在値が月平均より低い THEN システムは下向き矢印アイコンと赤色で表示する
4. WHEN 現在値と月平均がほぼ同じ THEN システムは横向き矢印アイコンとグレー色で表示する
5. WHEN 前年度平均がある THEN システムは前年度平均も比較表示する

### Requirement 3

**User Story:** ユーザーとして、担当者別のデータを整理された形式で見たい。そうすることで、各担当者のパフォーマンスを比較しやすくなる。

#### Acceptance Criteria

1. WHEN 担当者別データを表示する THEN システムはテーブル形式で表示する
2. WHEN テーブルを表示する THEN システムは担当者名、件数、割合、月平均の列を含める
3. WHEN テーブルを表示する THEN システムは割合の高い順にソートする
4. WHEN テーブルを表示する THEN システムは合計行を太字で強調表示する
5. WHEN テーブルが空の場合 THEN システムは「データなし」メッセージを表示する

### Requirement 4

**User Story:** ユーザーとして、レスポンシブなデザインで実績を見たい。そうすることで、どのデバイスでも快適に閲覧できる。

#### Acceptance Criteria

1. WHEN モバイルデバイスで表示する THEN システムはカードを縦に並べて表示する
2. WHEN タブレット以上のデバイスで表示する THEN システムはカードを2列のグリッドで表示する
3. WHEN 大画面で表示する THEN システムはカードを適切な最大幅で表示する
4. WHEN テーブルをモバイルで表示する THEN システムはスクロール可能にする
5. WHEN フォントサイズを調整する THEN システムは画面サイズに応じて適切なサイズを使用する
