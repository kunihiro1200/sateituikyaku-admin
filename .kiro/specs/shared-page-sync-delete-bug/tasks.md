- [x] 1. バグ条件の探索テストを書く
  - **Property 1: Bug Condition** - 新規エントリーの不完全保存バグ
  - **重要**: このテストは未修正コードで必ず**失敗**する - 失敗がバグの存在を証明する
  - **修正前にテストを書くこと - テストや実装コードを修正しようとしないこと**
  - **目的**: バグが存在することを示す反例を見つける
  - **スコープ付きPBTアプローチ**: 全フィールドを含むSheetRowを `appendRow` に渡したとき、`objectToRow` の出力配列でH列（インデックス7）以外が空文字になるケースを確認する
  - テスト内容（design.mdのBug Conditionより）:
    - `SharedItemsService.create()` に全フィールド（ID、日付、入力者、共有場、項目、タイトル、内容、共有日）を含むSheetRowを渡す
    - `objectToRow` が返す配列を確認し、H列（共有日）以外のカラムが空文字になっていることを検証する
    - `isBugCondition(input)`: `matchedKeys.size() < headersFromSheet.size()` かつ `matchedKeys.size() > 0` の場合
  - 未修正コードでテストを実行する
  - **期待される結果**: テストが**失敗**する（バグの存在を確認）
  - 見つかった反例を記録して根本原因を理解する（例: `objectToRow` の出力でH列以外が空文字）
  - テストを書き、実行し、失敗を記録したらタスク完了とする
  - _Requirements: 1.1, 1.2_
