# 実装計画: 買主詳細画面の3列レイアウト化

## 概要

`BuyerDetailPage.tsx` の2列レイアウトを3列レイアウト（左28% / 中央36% / 右36%）に変更する。
左列（新規）に通話履歴・メール・SMS送信履歴を配置し、中央列に物件詳細カード、右列に買主情報フィールドと関連買主を配置する。

## タスク

- [x] 1. 3列レイアウトコンテナへの変更
  - `BuyerDetailPage.tsx` の2列レイアウト用 `<Box>` を3列構成に変更する
  - 左列: `flex: '0 0 28%'`、中央列: `flex: '0 0 36%'`、右列: `flex: '1 1 36%'`
  - 各列に `maxHeight: 'calc(100vh - 200px)'`、`overflowY: 'auto'` を設定する
  - 各列にカスタムスクロールバースタイルを適用する
  - レスポンシブ対応: 900px以下で `flexDirection: 'column'` に切り替える
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 2. 左列の新規追加（通話履歴・メール・SMS送信履歴）
  - [x] 2.1 通話履歴セクションを左列に実装する
    - `activities` から `action === 'call' || action === 'phone_call'` でフィルタリングする
    - 日時（`formatDateTime`）・担当者（`getDisplayName`）・メモ（`metadata?.notes || metadata?.memo`）を表示する
    - 通話履歴が空の場合は「通話履歴はありません」メッセージと `PhoneIcon` を表示する
    - _Requirements: 2.1, 2.3, 2.4, 2.5_

  - [ ]* 2.2 通話履歴フィルタリングのプロパティテストを作成する
    - **Property 2: 通話履歴のフィルタリング正確性**
    - **Validates: Requirements 2.1, 2.4**

  - [x] 2.3 メール・SMS送信履歴セクションを右列から左列に移動する
    - 現在の右列にある `activities.filter(a => a.action === 'email' || a.action === 'sms')` の表示ブロックを左列に移動する
    - 表示内容（件名・送信者・物件番号・内覧前伝達事項等）を維持する
    - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

  - [ ]* 2.4 メール・SMS履歴フィルタリングのプロパティテストを作成する
    - **Property 3: メール・SMS履歴のフィルタリング正確性と配置**
    - **Validates: Requirements 3.1, 3.2, 3.3, 5.4**

- [x] 3. 中央列への物件詳細カード移動
  - 現在の左列（`flex: '0 0 42%'`）にある `PropertyInfoCard` の表示ブロックを中央列（`flex: '0 0 36%'`）に移動する
  - 「物件詳細カード」タイトルと紐づき物件数の表示を維持する
  - `linkedProperties` が空の場合の「紐づいた物件はありません」メッセージを維持する
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 4. 右列の整理（メール・SMS送信履歴の削除・関連買主の移動）
  - 右列から「メール・SMS送信履歴」セクションを削除する（左列に移動済み）
  - 現在2列レイアウト外（`<Box sx={{ mt: 3 }}>` 内）にある `RelatedBuyersSection` を右列の最下部に移動する
  - 右列の `BUYER_FIELD_SECTIONS` インライン編集フィールドはそのまま維持する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 5. チェックポイント - 全テストが通ることを確認する
  - 全テストが通ることを確認する。問題があればユーザーに確認する。
