# Bugfix Requirements Document

## Introduction

売主一覧テーブルの「対応中」列に表示されるプレゼンス情報（国、林など）に2つの問題が発生しています。

**問題1: 表示遅延** - 通話モードから一覧に戻った際、自分のイニシャルが「対応中」列に表示されるまでに数秒の遅延が発生します。

**問題2: 早期消失** - ユーザーがページを離れた後、プレゼンス情報がすぐに消えてしまうため、他のスタッフが「この案件は対応済み」と判断する前に消失し、重複電話が発生します。

これらの問題により、スタッフ間の連携が取りにくく、重複電話のリスクが高まっています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーが通話モードで売主を開いた後、売主一覧に戻る THEN 自分のイニシャルが「対応中」列に表示されるまでに数秒の遅延が発生する

1.2 WHEN ユーザーがページを離れる（別の売主を開く、または別のページに移動する） THEN プレゼンス情報が即座に消えてしまう

1.3 WHEN 他のスタッフが売主一覧を見ている THEN 対応中の売主を判断する前にプレゼンス情報が消失し、重複電話が発生する可能性がある

### Expected Behavior (Correct)

2.1 WHEN ユーザーが通話モードで売主を開いた後、売主一覧に戻る THEN 自分のイニシャルが「対応中」列に即座に（遅延なく）表示される

2.2 WHEN ユーザーがページを離れる（別の売主を開く、または別のページに移動する） THEN プレゼンス情報は5秒間表示を維持してから消える

2.3 WHEN 他のスタッフが売主一覧を見ている THEN 対応中の売主を5秒間確認でき、重複電話を防止できる

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 他のユーザーが売主を開いている THEN その売主の「対応中」列に他のユーザーのイニシャル（国、林など）が表示され続ける

3.2 WHEN 複数のユーザーが同じ売主を開いている THEN 全てのユーザーのイニシャルが「対応中」列に表示される

3.3 WHEN ユーザーが売主を開いていない THEN その売主の「対応中」列は空白のままである

3.4 WHEN プレゼンス情報が更新される THEN `formatPresenceLabel`関数により「国」「林」などのラベル形式で表示される
