# Bugfix Requirements Document

## Introduction

売主リストページにおいて、固定資産税路線価を編集しても査定額1/2/3が自動再計算されない問題を修正します。

現在、固定資産税路線価を最初に入力すると査定額が自動計算されますが、その後に固定資産税路線価を編集（例: 50000 → 60000）しても査定額が更新されません。また、リロード後に査定額1だけが再計算され、査定額2/3が空欄になるという不整合も発生しています。

この問題により、ユーザーは固定資産税路線価を変更するたびにページをリロードする必要があり、さらにリロード後も査定額2/3が消えてしまうため、正確な査定額を確認できない状態になっています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 固定資産税路線価を最初に入力した後、その値を編集する（例: 50000 → 60000）THEN システムは査定額1/2/3を再計算しない

1.2 WHEN 固定資産税路線価を編集した後、ページをリロードする THEN システムは査定額1のみを再計算し、査定額2/3を空欄にする

1.3 WHEN 固定資産税路線価の編集中に手入力査定額保存ボタンを押していない THEN システムは査定額の再計算をトリガーしない

### Expected Behavior (Correct)

2.1 WHEN 固定資産税路線価を編集する（例: 50000 → 60000）THEN システムは即座に査定額1/2/3を再計算する

2.2 WHEN 固定資産税路線価を編集した後、ページをリロードする THEN システムは査定額1/2/3の全てを正しく再計算して表示する

2.3 WHEN 固定資産税路線価が変更される THEN システムは自動的に査定額の再計算をトリガーする

2.4 WHEN 査定額の自動計算が実行される THEN システムは計算結果をスプレッドシートのBC/BD/BE列（自動計算用）に保存する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 固定資産税路線価を最初に入力する THEN システムは引き続き査定額1/2/3を自動計算する

3.2 WHEN 手入力査定額保存ボタンを押す THEN システムは引き続き手入力査定額をCB/CC/CD列に保存し、自動計算値より優先する

3.3 WHEN 固定資産税路線価以外のフィールド（土地面積、建物面積等）を編集する THEN システムは引き続き既存の動作を維持する

3.4 WHEN 査定額の優先順位ロジック（手動入力優先、なければ自動計算）を適用する THEN システムは引き続き既存のロジックを維持する
