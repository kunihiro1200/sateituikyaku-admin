# バグ修正要件ドキュメント

## はじめに

売主リストの通話モードページ（`/sellers/:id/call`）において、ステータスセクションのフィールドを編集して「ステータスを更新」ボタンを押しても、1回目の押下では保存が完了せず、2回押さないと保存されないバグが発生している。

このバグにより、ユーザーは保存操作を2回行う必要があり、業務効率が低下している。また、1回目の押下後に保存されたと誤認して画面を離れると、変更が失われるリスクがある。

対象ファイル：
- フロントエンド: `frontend/frontend/src/pages/CallModePage.tsx`
- バックエンド: `backend/src/routes/sellers.ts`、`backend/src/services/SellerService.supabase.ts`

---

## バグ分析

### 現在の動作（不具合）

ステータスセクションのフィールド（状況（当社）、次電日、確度、Pinrichステータス、除外アクション等）を編集して「ステータスを更新」ボタンを押した際の不正な動作。

1.1 WHEN ユーザーがステータスセクションのフィールドを変更して保存ボタンを1回押す THEN システムはAPIを呼び出して保存処理を実行するが、保存ボタンが再び有効化（オレンジ色のパルスアニメーション状態）に戻る

1.2 WHEN 保存ボタンを1回押した後に保存ボタンが再び有効化される THEN システムは `statusChanged` フラグが `false` にリセットされた後、何らかの原因で再び `true` に設定される

1.3 WHEN ユーザーが保存ボタンを2回目に押す THEN システムは同じAPIを再度呼び出して保存処理を実行し、今度は保存ボタンが無効化（グレー状態）のまま維持される

### 期待される動作（正常）

2.1 WHEN ユーザーがステータスセクションのフィールドを変更して保存ボタンを1回押す THEN システムはAPIを呼び出して保存処理を実行し、成功後に保存ボタンが無効化（グレー状態）に変わり、「ステータスを更新しました」の成功メッセージが表示される

2.2 WHEN 保存ボタンを1回押して保存が成功する THEN システムは `statusChanged` フラグを `false` にリセットし、その後 `true` に戻ることなく保存ボタンが無効化状態を維持する

2.3 WHEN 保存が成功した後にユーザーが何もフィールドを変更しない THEN システムは保存ボタンを無効化（グレー状態）のまま維持し、再度押せない状態にする

### 変更されない動作（リグレッション防止）

3.1 WHEN ユーザーがステータスセクションのフィールドを変更する THEN システムは引き続き `statusChanged` フラグを `true` に設定し、保存ボタンをオレンジ色のパルスアニメーション状態で有効化する

3.2 WHEN ページを初回ロードする THEN システムは引き続き売主データを取得してステータスフィールドを初期化し、`statusChanged` フラグを `false` に設定する

3.3 WHEN 保存処理中（`savingStatus=true`）である THEN システムは引き続き保存ボタンを無効化してローディングインジケーターを表示する

3.4 WHEN バリデーションエラーが発生する（確度未入力、専任他決の必須フィールド未入力等） THEN システムは引き続きエラーメッセージを表示してAPIを呼び出さない

3.5 WHEN 保存が成功した後にユーザーが再度フィールドを変更する THEN システムは引き続き `statusChanged` フラグを `true` に設定し、保存ボタンを有効化する

3.6 WHEN キャッシュヒット時にバックグラウンドで最新データを取得する THEN システムは引き続き `setSeller(freshData)` を呼び出して売主データを更新する（ただし `statusChanged` フラグに影響を与えない）
