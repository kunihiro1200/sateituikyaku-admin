# Bugfix Requirements Document

## Introduction

買主リストおよび売主リストにおいて、ブラウザUIからデータベースを更新した際、サイドバーカテゴリーのカウントが即座に反映されない問題を修正します。

現在、サイドバーカウントは`buyer_sidebar_counts`テーブル（買主）と`seller_sidebar_counts`テーブル（売主）に保存され、GASが10分ごとに更新しています。そのため、ブラウザUIからデータベースを直接更新しても、GASの次回同期（最大10分後）まで反映されません。

この問題により、ユーザーは更新内容がサイドバーに即座に反映されず、混乱を招いています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーがブラウザUIから買主データを更新する（例: 買主番号5641の「次電日」を今日に設定） THEN データベースには即座に保存されるが、サイドバーの「当日TEL」カテゴリーには反映されない

1.2 WHEN ユーザーがブラウザUIから売主データを更新する（例: 売主番号AA13501の「次電日」を今日に設定） THEN データベースには即座に保存されるが、サイドバーの「当日TEL分」カテゴリーには反映されない

1.3 WHEN データベース更新後にサイドバーを確認する THEN GASの10分同期を待たないと更新内容が反映されない

1.4 WHEN GASの10分同期が実行される THEN サイドバーカウントが更新され、正しいカウントが表示される

### Expected Behavior (Correct)

2.1 WHEN ユーザーがブラウザUIから買主データを更新する（例: 買主番号5641の「次電日」を今日に設定） THEN データベースに保存され、サイドバーの「当日TEL」カテゴリーに即座に反映される

2.2 WHEN ユーザーがブラウザUIから売主データを更新する（例: 売主番号AA13501の「次電日」を今日に設定） THEN データベースに保存され、サイドバーの「当日TEL分」カテゴリーに即座に反映される

2.3 WHEN データベース更新後にサイドバーを確認する THEN 更新内容が即座に反映されている

2.4 WHEN GASの10分同期が実行される THEN サイドバーカウントが正しく維持される（既存の動作を保持）

### Unchanged Behavior (Regression Prevention)

3.1 WHEN GASの10分同期が実行される THEN サイドバーカウントが正しく更新される（既存の動作を保持）

3.2 WHEN サイドバーカテゴリーをクリックする THEN 該当する買主/売主の一覧が正しく表示される（既存の動作を保持）

3.3 WHEN ページをリロードする THEN サイドバーカウントが正しく表示される（既存の動作を保持）

3.4 WHEN 複数のユーザーが同時にデータを更新する THEN 各ユーザーのサイドバーカウントが正しく更新される（既存の動作を保持）
