# Bugfix Requirements Document

## Introduction

買主リストページのサイドバーで「②内覧済み」「担当(Y)」などのカテゴリをクリックしても、「買主データが見つかりませんでした」と表示される問題を修正します。

サイドバーのカウント表示は正しく動作しており（例：内覧済み 1546件、担当(Y) 210件）、データベースにも該当する買主が存在しますが、フィルタリング処理が正しく機能していません。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーが「②内覧済み」カテゴリをクリックした場合 THEN システムは「買主データが見つかりませんでした」と表示する

1.2 WHEN ユーザーが「担当(Y)」カテゴリをクリックした場合 THEN システムは「買主データが見つかりませんでした」と表示する

1.3 WHEN ユーザーが「担当(林)」などの他の担当者カテゴリをクリックした場合 THEN システムは「買主データが見つかりませんでした」と表示する

1.4 WHEN `BuyerStatusCalculator.calculateBuyerStatus()`が呼び出された場合 THEN システムは優先度の高いステータス（査定アンケート、業者問合せ、内覧日前日など）のみを返し、「内覧済み」「担当」などの基本カテゴリのステータスを返さない

1.5 WHEN 買主の`calculated_status`が空文字列または担当者別のステータス（例：`担当(林)`）の場合 THEN フロントエンドのフィルタリングロジックがサイドバーのカテゴリキー（例：`visitCompleted`）を日本語の表示名（例：`内覧済み`）に変換してから`calculated_status`と比較するが、マッチしない

### Expected Behavior (Correct)

2.1 WHEN ユーザーが「②内覧済み」カテゴリをクリックした場合 THEN システムは内覧済みの買主（1546件）を一覧表示する

2.2 WHEN ユーザーが「担当(Y)」カテゴリをクリックした場合 THEN システムは担当Yの買主（210件）を一覧表示する

2.3 WHEN ユーザーが「担当(林)」などの他の担当者カテゴリをクリックした場合 THEN システムはその担当者の買主を一覧表示する

2.4 WHEN `BuyerStatusCalculator.calculateBuyerStatus()`が呼び出された場合 THEN システムは優先度の高いステータスに該当しない買主に対して「内覧済み」「担当」などの基本カテゴリのステータスを返す

2.5 WHEN 買主の`calculated_status`が「内覧済み」または「担当(林)」などの場合 THEN フロントエンドのフィルタリングロジックが正しくマッチングして該当する買主を表示する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーが「①内覧日前日」カテゴリをクリックした場合 THEN システムは引き続き内覧日前日の買主（2件）を正しく表示する

3.2 WHEN サイドバーのカウント表示が更新された場合 THEN システムは引き続き正しいカウント数（例：内覧済み 1546件、担当(Y) 210件）を表示する

3.3 WHEN 優先度の高いステータス（査定アンケート、業者問合せ、内覧日前日など）に該当する買主が存在する場合 THEN システムは引き続きそれらのステータスを優先して表示する

3.4 WHEN ユーザーが買主番号、氏名、電話番号で検索した場合 THEN システムは引き続き正しく検索結果を表示する

3.5 WHEN ユーザーがページネーションを操作した場合 THEN システムは引き続き正しくページ遷移する
