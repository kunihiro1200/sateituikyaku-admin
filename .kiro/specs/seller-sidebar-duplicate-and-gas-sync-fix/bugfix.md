# Bugfix Requirements Document

## Introduction

売主リストのサイドバーに関する2つのバグを修正する。

**絶対ルール**: サイドバーカテゴリーはいかなる場合も重複を避け、優先順位が高いカテゴリーにのみ表示すること。1人の売主は必ず1つのカテゴリーにのみカウントされる。

**バグ1**: サイドバーカテゴリーの重複問題。「未着手」カテゴリーに含まれる売主が「当日TEL」カテゴリーにも重複して表示されている。「未着手」は全カテゴリーの中で最高優先順位を持ち、「未着手」条件を満たす売主は他のどのカテゴリーの条件を満たしていても必ず「未着手」にのみカウントされる。

**バグ2**: GASの次電日更新問題。スプレッドシートで更新された次電日（例：AA13950の4/14）がDBに反映されず、古い値（4/12）のままになっている。ハッシュキャッシュが正しく更新されていないか、差分検知ロジックに問題がある可能性がある。

---

## Bug Analysis

### Current Behavior (Defect)

**バグ1: サイドバーカテゴリー重複**

1.1 WHEN 売主が「未着手」カテゴリーの条件（`status === '追客中'` かつ `next_call_date <= today` かつ連絡先情報なし かつ `unreachable_status`なし かつ `inquiry_date >= '2026-01-01'`）を満たす THEN システムはその売主を「当日TEL」カテゴリーにも重複して表示する

1.2 WHEN 売主が複数のサイドバーカテゴリーの条件を同時に満たす THEN システムは優先順位を考慮せず、全ての該当カテゴリーにその売主をカウントする

**バグ2: GAS次電日更新**

1.3 WHEN スプレッドシートの次電日が更新されてから5時間以上経過している THEN システム（GAS）はその変更をDBに反映しない場合がある

1.4 WHEN `_seller_hashes` シートのハッシュキャッシュが古い状態のまま残っている THEN システムは差分なしと判断してSupabaseへのPATCHをスキップする

### Expected Behavior (Correct)

**バグ1: サイドバーカテゴリー重複**

2.1 WHEN 売主が「未着手」カテゴリーの条件を満たす THEN システムは「当日TEL」カテゴリーからその売主を除外し、「未着手」カテゴリーのみにカウントする（「未着手」は全カテゴリーの中で最高優先順位）

2.2 WHEN 売主が複数のサイドバーカテゴリーの条件を同時に満たす THEN システムは定義された優先順位に従い、最も優先度の高いカテゴリーにのみその売主をカウントする（いかなる条件の組み合わせでも1人の売主は1つのカテゴリーにのみカウントされる）

2.3 WHEN 「未着手」条件を満たす売主が「当日TEL」「専任」「一般媒介」など他のカテゴリーの条件も同時に満たす THEN システムは他のカテゴリーの条件を無視し、「未着手」カテゴリーにのみカウントする（「未着手」の優先順位は絶対）

**バグ2: GAS次電日更新**

2.4 WHEN スプレッドシートの次電日が更新される THEN システム（GAS）は次回の10分トリガー実行時にその変更を検知し、Supabaseに正しく反映する

2.5 WHEN `buildRowHash_()` 関数がハッシュを計算する THEN システムは `次電日` フィールドの変更を確実に検知できるハッシュを生成する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 売主が「未着手」カテゴリーの条件を満たさない（連絡先情報あり、または `unreachable_status` あり、または `inquiry_date < '2026-01-01'` など） THEN システムは引き続き「当日TEL」カテゴリーに正しくカウントする

3.2 WHEN 売主が「当日TEL（内容）」カテゴリーの条件を満たす（連絡先情報あり） THEN システムは引き続き「当日TEL（内容）」カテゴリーに正しくカウントする

3.3 WHEN スプレッドシートの行データに変更がない THEN システムは引き続き差分なしと判断してSupabaseへのPATCHをスキップする（不要な更新を行わない）

3.4 WHEN GASの `onEditTrigger` が発火する THEN システムは引き続き即時同期を正しく実行する

3.5 WHEN 売主が「専任」「一般媒介」「訪問後他決」「未訪問他決」などの他のカテゴリーの条件を満たす（かつ「未着手」条件を満たさない） THEN システムは引き続きそれらのカテゴリーに正しくカウントする
