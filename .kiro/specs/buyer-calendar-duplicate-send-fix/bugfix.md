# バグ修正要件書

## Introduction

買主リストの内覧ページ（`BuyerViewingResultPage`）で「📅 カレンダーで開く」ボタンを押すと、後続担当のGoogleカレンダーに同じイベントが2件重複して送信される問題と、`tenant@ifoo-oita.com`（テナント系カレンダー）にも誤って送信される問題を修正する。

正しい動作は「後続担当のカレンダーに1件だけイベントが表示される」ことである。

現在の実装では `handleCalendarButtonClick` 関数内で以下の2つのパラメータが同時に使用されており、これが重複の原因となっている：
- `add={assignedEmail}` → 後続担当をゲストとして招待（1件目）
- `src={assignedEmail}` → 後続担当のカレンダーに直接作成（2件目）

また、後続担当が「業者」の場合に `tenant@ifoo-oita.com` をハードコードして送信しているため、テナント系カレンダーへの誤送信が発生している。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 後続担当が設定された状態で「📅 カレンダーで開く」ボタンを押す THEN システムは `add` パラメータと `src` パラメータの両方を同時にGoogleカレンダーURLに付与し、後続担当のカレンダーに同じイベントが2件重複して登録される

1.2 WHEN 後続担当が「業者」に設定された状態で「📅 カレンダーで開く」ボタンを押す THEN システムは `tenant@ifoo-oita.com` をカレンダー送信先として使用し、テナント系カレンダーにイベントが送信される

### Expected Behavior (Correct)

2.1 WHEN 後続担当が設定された状態で「📅 カレンダーで開く」ボタンを押す THEN システムは後続担当のカレンダーに1件のみイベントを登録し、重複は発生しない

2.2 WHEN 後続担当が「業者」に設定された状態で「📅 カレンダーで開く」ボタンを押す THEN システムはテナント系カレンダー（`tenant@ifoo-oita.com`）への送信を行わず、カレンダー送信をスキップするかエラーメッセージを表示する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 後続担当が従業員マスタに存在するイニシャルに設定されている THEN システムは引き続き従業員マスタからメールアドレスを検索してカレンダー送信先として使用する

3.2 WHEN 後続担当が従業員マスタに存在しない場合 THEN システムは引き続きエラーメッセージを表示してカレンダー送信を中断する

3.3 WHEN 後続担当のイニシャルが複数の社員に一致する場合 THEN システムは引き続き「複数の社員に一致します」エラーメッセージを表示してカレンダー送信を中断する

3.4 WHEN カレンダーボタンをクリックする THEN システムは引き続きGoogleカレンダーの新規イベント作成画面を新しいタブで開く

3.5 WHEN カレンダーボタンをクリックする THEN システムは引き続き `calendarOpened` フラグを `true` に設定し、離脱ガードを解除する

3.6 WHEN カレンダーボタンをクリックする THEN システムは引き続き `/api/buyer-appointments` へのメール通知送信を試みる（失敗しても無視）
