# Bugfix Requirements Document

## Introduction

売主リストページ（通話モードページ）で「不通・１番電話を保存」ボタンと「ステータスを更新」ボタンを押すと、保存中に一瞬画面が白くなって保存される問題を修正する。

コメントの保存ボタンのように、画面を維持したまま保存作業を行うようにする。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーが「不通・１番電話を保存」ボタンをクリックする THEN システムは保存処理後に`loadAllData()`を呼び出し、画面が一瞬白くなる（ページ全体が再レンダリングされる）

1.2 WHEN ユーザーが「ステータスを更新」ボタンをクリックする THEN システムは保存処理後に`loadAllData()`を呼び出し、画面が一瞬白くなる（ページ全体が再レンダリングされる）

1.3 WHEN 画面が白くなる THEN ユーザー体験が悪化し、保存処理が完了したかどうかが分かりにくくなる

### Expected Behavior (Correct)

2.1 WHEN ユーザーが「不通・１番電話を保存」ボタンをクリックする THEN システムは保存処理後に画面を維持したまま、必要な状態のみを更新する（画面が白くならない）

2.2 WHEN ユーザーが「ステータスを更新」ボタンをクリックする THEN システムは保存処理後に画面を維持したまま、必要な状態のみを更新する（画面が白くならない）

2.3 WHEN 保存処理が完了する THEN システムは保存完了メッセージを表示し、ローディング表示のみで画面フラッシュは発生しない

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーが「コメント保存」ボタンをクリックする THEN システムは CONTINUE TO 画面を維持したまま保存処理を行う（現在の正常な動作を維持）

3.2 WHEN 保存処理が成功する THEN システムは CONTINUE TO データベースに正しく保存し、スプレッドシートに同期する

3.3 WHEN 保存処理が失敗する THEN システムは CONTINUE TO エラーメッセージを表示し、ユーザーに通知する
