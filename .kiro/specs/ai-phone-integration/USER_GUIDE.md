# User Guide - AI電話統合機能

## 概要

AI電話統合機能は、売主管理システムに電話発信・着信、通話録音、文字起こし、感情分析などの機能を追加します。このガイドでは、各機能の使い方を説明します。

## 1. 発信機能

### 1.1 売主詳細ページから発信

売主詳細ページから直接電話をかけることができます。

**手順**:
1. 売主一覧ページから売主を選択
2. 売主詳細ページを開く
3. 電話番号の横にある「電話をかける」ボタンをクリック
4. 発信が開始されます

**注意事項**:
- 電話番号が登録されていない売主には発信できません
- 発信中は「発信中...」と表示されます
- 発信が成功すると、通話ログが自動的に作成されます

### 1.2 発信後の動作

発信が成功すると：
- 通話ログが自動的に作成されます
- 活動履歴に「電話発信」が記録されます
- 通話終了後、録音ファイルが自動的に保存されます
- 数分後、文字起こしと感情分析が自動的に実行されます

## 2. 通話履歴の確認

### 2.1 通話履歴ページへのアクセス

**手順**:
1. メニューから「通話履歴」をクリック
2. 通話履歴ページが表示されます

### 2.2 通話履歴の表示内容

各通話ログには以下の情報が表示されます：

- **方向**: 着信 / 発信
- **ステータス**: 完了 / 不在着信 / 失敗 / 話中
- **感情**: ポジティブ / ネガティブ / 中立 / 混合
- **日時**: 通話開始日時
- **通話時間**: 通話の長さ（分:秒）
- **売主名**: 通話相手の売主名（クリックで詳細ページへ）
- **担当者**: 通話を行った担当者名
- **キーワード**: 検出されたキーワード（「査定」「専任媒介」など）

### 2.3 通話履歴のフィルタリング

通話履歴は以下の条件でフィルタリングできます：

**検索**:
- 売主名、電話番号、担当者名で検索

**フィルター**:
- **方向**: すべて / 着信 / 発信
- **ステータス**: すべて / 完了 / 不在着信 / 失敗 / 話中
- **担当者**: すべて / 特定の担当者
- **日付範囲**: 開始日〜終了日

**手順**:
1. 「フィルター」ボタンをクリック
2. 条件を選択
3. 自動的にフィルタリングされます
4. 「フィルターをリセット」でクリア

### 2.4 通話履歴のエクスポート

通話履歴をCSVファイルとしてエクスポートできます。

**手順**:
1. 通話履歴ページで「CSVエクスポート」ボタンをクリック
2. CSVファイルがダウンロードされます

**エクスポート内容**:
- 日時
- 方向（着信/発信）
- 売主名
- 電話番号
- 担当者
- ステータス
- 通話時間

## 3. 録音の再生

### 3.1 録音の読み込み

**手順**:
1. 通話履歴ページで通話ログをクリックして展開
2. 「録音を読み込む」ボタンをクリック
3. 数秒後、音声プレーヤーが表示されます

### 3.2 音声プレーヤーの使い方

音声プレーヤーには以下の機能があります：

**基本操作**:
- **再生/一時停止**: 再生ボタンをクリック
- **シーク**: プログレスバーをクリックまたはドラッグ
- **10秒スキップ**: 「-10秒」「+10秒」ボタンをクリック

**音量調整**:
- **音量スライダー**: スライダーをドラッグして音量を調整
- **ミュート**: スピーカーアイコンをクリック

**再生速度調整**:
- **速度選択**: 0.5x / 0.75x / 1x / 1.25x / 1.5x / 2x から選択
- デフォルトは1x（通常速度）

**文字起こしとの同期**:
- 再生中、現在の位置に対応する文字起こしセグメントが自動的にハイライトされます
- 文字起こしセグメントをクリックすると、その位置にジャンプします

### 3.3 話者識別

文字起こしには話者識別機能があります：

- **担当者**: 青色で表示
- **顧客**: 緑色で表示

各セグメントには以下の情報が表示されます：
- 話者（担当者/顧客）
- 発言内容
- タイムスタンプ（開始時刻〜終了時刻）
- 信頼度スコア

## 4. 文字起こしの確認

### 4.1 文字起こしの表示

**手順**:
1. 通話履歴ページで通話ログをクリックして展開
2. 「文字起こし」セクションに文字起こしテキストが表示されます

### 4.2 文字起こしのステータス

文字起こしには以下のステータスがあります：

- **処理待ち**: 文字起こしジョブが開始されていません
- **処理中**: 文字起こし処理が実行中です（数分かかります）
- **完了**: 文字起こしが完了しました
- **失敗**: 文字起こしに失敗しました

**注意事項**:
- 文字起こしは通話終了後、自動的に開始されます
- 処理には通話時間に応じて数分かかります
- 処理中は「文字起こし処理中...」と表示されます

### 4.3 文字起こしの精度

文字起こしの精度は以下の要因に影響されます：

- **音声品質**: クリアな音声ほど精度が高くなります
- **話し方**: ゆっくり明瞭に話すと精度が向上します
- **専門用語**: カスタム語彙に登録された用語は正確に認識されます
- **背景ノイズ**: ノイズが少ないほど精度が高くなります

## 5. 感情分析の確認

### 5.1 感情分析結果の表示

通話ログには感情分析結果が表示されます：

**感情の種類**:
- **ポジティブ**: 前向きな感情（満足、興味、期待など）
- **ネガティブ**: 否定的な感情（不満、不安、怒りなど）
- **中立**: 中立的な感情
- **混合**: 複数の感情が混在

**表示場所**:
- 通話ログカードのバッジ（アイコン付き）
- 展開時の感情分析スコア

### 5.2 感情分析スコアの見方

展開時に表示される感情分析スコアは、各感情の割合を示します：

```
ポジティブ: 85.0%
ネガティブ: 5.0%
中立: 10.0%
混合: 0.0%
```

**解釈**:
- 合計は100%になります
- 最も高いスコアの感情が全体の感情として表示されます
- 複数の感情が混在する場合は「混合」と表示されます

### 5.3 感情分析の活用方法

感情分析結果は以下のように活用できます：

**ポジティブな通話**:
- 売主の興味が高い可能性があります
- フォローアップを優先的に行いましょう

**ネガティブな通話**:
- 売主に不安や不満がある可能性があります
- 丁寧なフォローアップが必要です
- 上司に報告して対応を相談しましょう

**中立な通話**:
- 情報収集段階の可能性があります
- 継続的なフォローアップを行いましょう

## 6. 統計情報の確認

### 6.1 統計ページへのアクセス

**手順**:
1. メニューから「通話統計」をクリック
2. 統計ページが表示されます

### 6.2 サマリー統計

ページ上部に以下のサマリー統計が表示されます：

- **総通話数**: 期間内の総通話数
- **着信数**: 着信の数
- **発信数**: 発信の数
- **平均通話時間**: 平均的な通話の長さ
- **総通話時間**: すべての通話の合計時間

### 6.3 グラフ表示

以下のグラフが表示されます：

**ステータス別通話数**（棒グラフ）:
- 完了、不在着信、失敗、話中の件数

**感情分析分布**（円グラフ）:
- ポジティブ、ネガティブ、中立、混合の割合

**担当者別通話数**（棒グラフ）:
- 各担当者の通話数

### 6.4 担当者別統計テーブル

各担当者の詳細な統計が表示されます：

- 担当者名
- 通話数
- 平均通話時間
- 総通話時間

### 6.5 頻出キーワード

通話内容から検出された頻出キーワードが表示されます：

- キーワード名
- 出現回数

**活用方法**:
- 売主の関心事を把握できます
- よく話題になるトピックを確認できます

### 6.6 期間フィルター

統計は期間でフィルタリングできます：

**手順**:
1. 「開始日」と「終了日」を選択
2. 自動的に統計が更新されます

### 6.7 統計のエクスポート

統計データをCSVファイルとしてエクスポートできます。

**手順**:
1. 統計ページで「CSVエクスポート」ボタンをクリック
2. CSVファイルがダウンロードされます

## 7. 設定管理（管理者のみ）

### 7.1 設定ページへのアクセス

**手順**:
1. メニューから「設定」→「電話設定」をクリック
2. 設定ページが表示されます

**注意**: この機能は管理者（admin）のみアクセス可能です。

### 7.2 AWS基本設定

以下の設定を管理できます：

- **AWSリージョン**: AWSサービスのリージョン（デフォルト: ap-northeast-1）
- **AWSアクセスキーID**: AWS認証情報
- **AWSシークレットアクセスキー**: AWS認証情報（暗号化して保存）

### 7.3 Amazon Connect設定

- **インスタンスID**: Amazon ConnectインスタンスのID
- **インスタンスARN**: Amazon ConnectインスタンスのARN
- **コンタクトフローID**: 使用するコンタクトフローのID
- **発信元電話番号**: 発信時に表示される電話番号

### 7.4 S3・Transcribe設定

- **録音ファイル保存バケット**: 録音ファイルを保存するS3バケット名
- **カスタム語彙名**: 文字起こしで使用するカスタム語彙（オプション）

### 7.5 機能設定

以下の機能を有効/無効にできます：

- **発信機能を有効化**: 発信機能のオン/オフ
- **着信機能を有効化**: 着信機能のオン/オフ
- **感情分析を有効化**: 感情分析機能のオン/オフ

### 7.6 接続テスト

設定を保存する前に、AWS接続テストを実行できます。

**手順**:
1. 設定を入力
2. 「接続テスト」ボタンをクリック
3. 各AWSサービスへの接続が確認されます

**テスト対象**:
- Amazon Connect
- Amazon Transcribe
- Amazon S3
- Amazon Comprehend（感情分析が有効な場合）

**結果の見方**:
- ✓ 緑色: 接続成功
- ✗ 赤色: 接続失敗

### 7.7 設定の保存

**手順**:
1. 設定を入力
2. 「接続テスト」で接続を確認（推奨）
3. 「設定を保存」ボタンをクリック
4. 「設定を保存しました」と表示されれば成功

**注意事項**:
- 誤った設定を保存すると、電話機能が正常に動作しなくなります
- 必ず接続テストを実行してから保存してください
- AWS認証情報は暗号化して保存されます

## 8. トラブルシューティング

### 8.1 発信できない

**症状**: 「電話をかける」ボタンをクリックしても発信できない

**原因と解決策**:

1. **電話番号が登録されていない**
   - 売主詳細ページで電話番号を登録してください

2. **電話番号の形式が正しくない**
   - 日本の電話番号形式（+81-xx-xxxx-xxxx または 0x-xxxx-xxxx）で入力してください

3. **発信機能が無効になっている**
   - 管理者に連絡して、設定ページで発信機能を有効にしてもらってください

4. **AWS設定が正しくない**
   - 管理者に連絡して、設定ページで接続テストを実行してもらってください

### 8.2 文字起こしが表示されない

**症状**: 通話終了後、文字起こしが「処理中...」のまま

**原因と解決策**:

1. **処理に時間がかかっている**
   - 文字起こしには数分かかります。しばらく待ってからページを更新してください

2. **録音ファイルが保存されていない**
   - 管理者に連絡して、S3バケットに録音ファイルが保存されているか確認してもらってください

3. **文字起こしジョブが失敗した**
   - 管理者に連絡して、ジョブキューの状態を確認してもらってください

### 8.3 録音が再生できない

**症状**: 「録音を読み込む」ボタンをクリックしても再生できない

**原因と解決策**:

1. **録音ファイルが見つからない**
   - 通話が正常に終了しなかった可能性があります
   - 管理者に連絡して、S3バケットを確認してもらってください

2. **Presigned URLの有効期限が切れた**
   - 「録音を読み込む」ボタンを再度クリックして、新しいURLを取得してください

3. **ブラウザの問題**
   - ブラウザのキャッシュをクリアしてください
   - 別のブラウザで試してください

### 8.4 感情分析が表示されない

**症状**: 文字起こしは完了しているが、感情分析結果が表示されない

**原因と解決策**:

1. **感情分析機能が無効になっている**
   - 管理者に連絡して、設定ページで感情分析機能を有効にしてもらってください

2. **感情分析ジョブが失敗した**
   - 管理者に連絡して、ジョブキューの状態を確認してもらってください

3. **文字起こしテキストが短すぎる**
   - 感情分析には一定量のテキストが必要です
   - 通話時間が短すぎる場合、感情分析が実行されない可能性があります

### 8.5 通話履歴が表示されない

**症状**: 通話履歴ページに何も表示されない

**原因と解決策**:

1. **まだ通話を行っていない**
   - 最初の通話を行ってください

2. **フィルターが適用されている**
   - 「フィルターをリセット」ボタンをクリックしてください

3. **データベース接続エラー**
   - ページを更新してください
   - 問題が続く場合は管理者に連絡してください

## よくある質問（FAQ）

### Q1: 通話は自動的に録音されますか？

**A**: はい、すべての通話は自動的に録音されます。録音ファイルはAmazon S3に安全に保存されます。

### Q2: 録音ファイルはいつまで保存されますか？

**A**: デフォルトでは90日間保存されます。その後、自動的にアーカイブストレージに移行され、365日後に削除されます。保持期間は管理者が変更できます。

### Q3: 文字起こしの精度を向上させるには？

**A**: 以下の点に注意してください：
- クリアな音声で通話する
- ゆっくり明瞭に話す
- 背景ノイズを減らす
- 専門用語はカスタム語彙に登録する（管理者に依頼）

### Q4: 感情分析は正確ですか？

**A**: Amazon Comprehendの感情分析は高精度ですが、100%正確ではありません。参考情報として活用してください。

### Q5: 通話履歴を削除できますか？

**A**: 現在、通話履歴の削除機能は実装されていません。必要な場合は管理者に連絡してください。

### Q6: 複数の担当者で同時に発信できますか？

**A**: はい、複数の担当者が同時に発信できます。Amazon Connectは複数の同時通話をサポートしています。

### Q7: 着信時に売主情報は自動的に表示されますか？

**A**: はい、着信時に電話番号から売主を自動的に検索し、マッチした場合は売主情報が表示されます。

### Q8: 通話中にメモを取ることはできますか？

**A**: 現在、通話中のメモ機能は実装されていません。通話後、売主詳細ページの活動履歴にコメントを追加してください。

1. まず[トラブルシューティング](#8-トラブルシューティング)セクションを確認
2. [よくある質問](#よくある質問faq)を確認
3. それでも解決しない場合は、管理者または開発チームに連絡

- 発生した問題の詳細
- エラーメッセージ（表示されている場合）
- 問題が発生した日時
- 使用しているブラウザとバージョン
- 問題を再現する手順
