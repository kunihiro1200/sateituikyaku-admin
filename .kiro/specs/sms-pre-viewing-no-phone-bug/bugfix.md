# バグ修正要件ドキュメント

## はじめに

内覧ページ（`BuyerViewingResultPage`）において、電話番号が未登録の買主に対して「内覧前日メッセージ」のSMS送信ボタンが表示されないバグを修正する。

買主番号7185が具体的な再現例。電話番号がない買主では、内覧日前日であっても内覧前日SMSボタンが一切表示されず、内覧前日メッセージを送信できない状態になっている。

コード調査の結果、根本原因は以下の通り：

`BuyerViewingResultPage.tsx` の内覧前日ボタン表示ロジックが `!buyer.email && buyer.phone_number` という条件になっており、電話番号がない買主はこの条件を満たさないため、SMSボタンが表示されない。

---

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 電話番号が未登録の買主の内覧ページを開き、内覧日前日である THEN システムは内覧前日SMSボタンを表示しない

1.2 WHEN 電話番号が未登録の買主の内覧ページで、SMS送信ボタンをクリックしようとする THEN システムはSMS送信ボタン自体が存在しないため、内覧前日メッセージを送信できない

### 期待される動作（正しい動作）

2.1 WHEN 電話番号が未登録かつメールアドレスも未登録の買主の内覧ページを開き、内覧日前日である THEN システムは内覧前日SMSボタンを表示しないものとする（SHALL）（送信手段がないため非表示が正しい）

2.2 WHEN 電話番号が登録済みかつメールアドレスが未登録の買主の内覧ページを開き、内覧日前日である THEN システムは内覧前日SMSボタンを表示するものとする（SHALL）

2.3 WHEN 電話番号が登録済みかつメールアドレスも登録済みの買主の内覧ページを開き、内覧日前日である THEN システムはEメールボタンとSMSボタンの両方を表示するものとする（SHALL）

2.4 WHEN 電話番号が未登録かつメールアドレスが登録済みの買主の内覧ページを開き、内覧日前日である THEN システムはEメールボタンのみを表示するものとする（SHALL）

### 変更しない動作（リグレッション防止）

3.1 WHEN メールアドレスが登録済みの買主の内覧ページを開き、内覧日前日である THEN システムは引き続きEメールボタンを表示し続けるものとする（SHALL CONTINUE TO）

3.2 WHEN 内覧日前日ではない買主の内覧ページを開く THEN システムは引き続き内覧前日ボタン群を表示しないものとする（SHALL CONTINUE TO）

3.3 WHEN `broker_inquiry === '業者問合せ'` の買主の内覧ページを開く THEN システムは引き続き内覧前日ボタン群を表示しないものとする（SHALL CONTINUE TO）

3.4 WHEN `notification_sender` が入力済みの買主の内覧ページを開く THEN システムは引き続き内覧前日ボタン群を表示しないものとする（SHALL CONTINUE TO）

3.5 WHEN SMS送信ボタンをクリックする THEN システムは引き続き既存のSMS履歴記録ロジック（`/api/buyers/:buyerNumber/sms-history`）を変更なく動作させ続けるものとする（SHALL CONTINUE TO）
