# Bugfix Requirements Document

## Introduction

通話モードページ（`/sellers/:id/call`）で以下の2つの保存エラーが発生しています：

1. **コミュニケーション情報フィールドの保存エラー**: 3つのフィールド（電話担当（任意）、連絡取りやすい日、連絡方法）を編集しても、データベースに保存されず、スプレッドシートにも即時同期されない

2. **訪問予約の保存エラー**: 訪問予約を保存する際に400エラーが発生し、保存できない

これらの問題により、ユーザーが入力した情報が失われ、業務に支障をきたしています。

## Bug Analysis

### Current Behavior (Defect)

#### コミュニケーション情報フィールドの保存エラー

1.1 WHEN ユーザーが通話モードページで「電話担当（任意）」フィールドを編集する THEN システムはデータベースに保存しない

1.2 WHEN ユーザーが通話モードページで「連絡取りやすい日」フィールドを編集する THEN システムはデータベースに保存しない

1.3 WHEN ユーザーが通話モードページで「連絡方法」フィールドを編集する THEN システムはデータベースに保存しない

1.4 WHEN ユーザーが通話モードページでコミュニケーション情報フィールドを編集する THEN システムはスプレッドシートに即時同期しない

#### 訪問予約の保存エラー

1.5 WHEN ユーザーが通話モードページで営担「K」を選択して訪問予約を保存する THEN システムは400エラー「無効な営担です」を返す

1.6 WHEN 営担「K」が`employees`テーブルで`is_normal=false`または`is_active=false`の場合 THEN システムは訪問予約の保存を拒否する

1.7 WHEN フロントエンドの営担選択UIに無効な従業員（`is_normal=false`または`is_active=false`）が表示されている THEN ユーザーは無効な営担を選択できてしまう

### Expected Behavior (Correct)

#### コミュニケーション情報フィールドの保存

2.1 WHEN ユーザーが通話モードページで「電話担当（任意）」フィールドを編集する THEN システムは1秒後にデータベースに自動保存する

2.2 WHEN ユーザーが通話モードページで「連絡取りやすい日」フィールドを編集する THEN システムは1秒後にデータベースに自動保存する

2.3 WHEN ユーザーが通話モードページで「連絡方法」フィールドを編集する THEN システムは1秒後にデータベースに自動保存する

2.4 WHEN ユーザーが通話モードページでコミュニケーション情報フィールドを編集してデータベースに保存される THEN システムはスプレッドシートに即時同期する

#### 訪問予約の保存

2.5 WHEN ユーザーが通話モードページで有効な営担（`is_normal=true`かつ`is_active=true`）を選択して訪問予約を保存する THEN システムは正常に保存する

2.6 WHEN フロントエンドの営担選択UIが表示される THEN システムは有効な従業員（`is_normal=true`かつ`is_active=true`）のみを選択肢に表示する

2.7 WHEN 営担「K」が`employees`テーブルで`is_normal=false`または`is_active=false`の場合 THEN システムは営担選択UIに「K」を表示しない

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーが通話モードページで他のフィールド（ステータス、コメント等）を編集する THEN システムは既存の保存処理を継続して実行する

3.2 WHEN ユーザーが通話モードページで「1番電話」フィールドを編集する THEN システムは既存の自動保存処理を継続して実行する

3.3 WHEN ユーザーが売主詳細ページでコミュニケーション情報フィールドを編集する THEN システムは既存の保存処理を継続して実行する

3.4 WHEN ユーザーがスプレッドシートでコミュニケーション情報フィールドを編集する THEN システムは既存の同期処理を継続して実行する

3.5 WHEN ユーザーが有効な営担で訪問予約を保存する THEN システムは既存の保存処理を継続して実行する
