# Bugfix Requirements Document

## Introduction

買主リストの希望条件ページ（`BuyerDesiredConditionsPage.tsx`）において、希望エリアフィールド（`desired_area`）を変更して保存ボタンを押してもDBに保存されない問題を修正します。

この問題により、ユーザーが希望エリアを変更しても、その変更がデータベースに反映されず、スプレッドシートにも同期されません。他のフィールド（希望時期、希望種別など）は正常に保存されるため、希望エリアフィールド特有の実装に問題があります。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーが希望条件ページで希望エリアフィールドのドロップダウンを開き、エリアを選択し、ドロップダウンを閉じずに保存ボタンを押した THEN システムは`onClose`イベントが発火せず、`pendingChanges`に希望エリアの変更が反映されないため、DBに保存されない

1.2 WHEN ユーザーが希望エリアフィールドのチップ削除ボタンでエリアを削除し、ドロップダウンを開かずに保存ボタンを押した THEN システムは`pendingChanges`に変更が反映されているが、`selectedAreasRef.current`の値が古い可能性があり、正しく保存されない場合がある

1.3 WHEN ユーザーが希望エリアフィールドを変更し、ドロップダウンを閉じた後に保存ボタンを押した THEN システムは`onClose`イベントで`pendingChanges`に反映されるため、正常に保存される（このケースは正常動作）

### Expected Behavior (Correct)

2.1 WHEN ユーザーが希望エリアフィールドのドロップダウンで選択を変更した THEN システムは`onChange`イベントで即座に`pendingChanges`に反映し、ドロップダウンを閉じなくても保存ボタン押下時にDBに保存される

2.2 WHEN ユーザーが希望エリアフィールドのチップ削除ボタンでエリアを削除した THEN システムは削除操作を即座に`pendingChanges`に反映し、保存ボタン押下時にDBに保存される

2.3 WHEN 希望エリアフィールドが保存された THEN システムはスプレッドシートに即時同期する（`buyerApi.update(..., { sync: true })`）

### Unchanged Behavior (Regression Prevention)

3.1 WHEN ユーザーが希望エリア以外のフィールド（希望時期、希望種別、価格帯など）を変更して保存ボタンを押した THEN システムは引き続き正常にDBに保存し、スプレッドシートに同期する

3.2 WHEN ユーザーが希望エリアフィールドを変更せずに保存ボタンを押した THEN システムは`pendingChanges`が空のため、保存処理を実行しない（現在の動作を維持）

3.3 WHEN ユーザーが配信メール「要」の買主の希望エリアを空にして保存しようとした THEN システムは必須バリデーションエラーを表示し、保存を拒否する（現在の動作を維持）

3.4 WHEN ユーザーが希望エリアフィールドのドロップダウンを開いて選択を変更し、ドロップダウンを閉じた後に保存ボタンを押した THEN システムは引き続き正常にDBに保存する（現在の正常動作を維持）
