# 実装計画: 業者フィルターボタングループ化

## 概要

`NearbyBuyersList.tsx` の業者フィルターボタン（「業者_土地」「業者_戸建」「業者_マンション」）を、背景色付きの `Box` コンテナで視覚的にグループ化する。フロントエンドのUI変更のみ。

## タスク

- [x] 1. 業者フィルターボタングループのUI実装
  - `frontend/frontend/src/components/NearbyBuyersList.tsx` を開き、`{/* 業者フィルターボタン */}` セクションを特定する
  - 親の `Box`（アクションボタン行）に `alignItems: 'center'` を追加する
  - 既存の業者フィルターボタン群（`showLandAndHouseButtons` と `showApartmentButton` の条件分岐）を、新しいグループコンテナ `Box` で囲む
  - グループコンテナに `backgroundColor: '#e8f5e9'`、`borderRadius: 1`、`border: '1px solid #a5d6a7'`、`p: 1`、`display: 'flex'`、`flexDirection: 'column'`、`gap: 0.5` を適用する
  - グループコンテナ内の先頭に `Typography variant="caption"` で「業者フィルター」ラベルを追加し、`color: '#2e7d32'`、`fontWeight: 600`、`fontSize: '0.7rem'` を適用する
  - ラベルの下にボタンを横並びにする内側 `Box`（`display: 'flex'`、`gap: 0.5`）を追加し、既存のボタンをその中に移動する
  - グループコンテナの表示条件を `(showLandAndHouseButtons || showApartmentButton)` の単一条件に統合する
  - _要件: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 4.1, 4.2, 4.3_

  - [ ]* 1.1 業者ボタングループの表示制御プロパティテストを作成する
    - **Property 1: 物件種別に応じた業者ボタングループの表示制御**
    - fast-check を使用し、`effectivePropertyType` の任意の値に対して土地・戸建て系では「業者_土地」「業者_戸建」が表示され、マンション系では「業者_マンション」のみ表示され、それ以外では非表示になることを検証する
    - **検証対象: 要件 1.1, 1.5, 1.6, 1.7, 3.3**

  - [ ]* 1.2 業者フィルタートグル動作のプロパティテストを作成する
    - **Property 2: 業者フィルタートグル動作の維持**
    - fast-check を使用し、任意の `activeAgencyFilter` 状態と任意のクリック操作に対して、同じフィルターをクリックすると `null`（解除）、異なるフィルターをクリックすると排他切り替えになることを検証する
    - **検証対象: 要件 3.1, 3.2**

  - [ ]* 1.3 業者フィルターと価格帯フィルターのAND結合プロパティテストを作成する
    - **Property 3: 業者フィルターと価格帯フィルターのAND結合維持**
    - fast-check を使用し、任意の買主リスト・業者フィルター状態・価格帯フィルター状態の組み合わせに対して、`filteredBuyers` が両フィルターのAND結合結果のみを含むことを検証する
    - **検証対象: 要件 3.4**

- [x] 2. チェックポイント - 動作確認
  - 既存のフィルタリング動作（トグル、排他切り替え、AND結合）が変わっていないことを確認する
  - 全テストが通ることを確認し、疑問点があればユーザーに確認する
