# 物件リスト営業担当同期バグ修正デザイン

## Overview

物件番号AA12497を含む約5-6件の物件で、スプレッドシートの「担当名（営業）」列に値が入力されているにもかかわらず、データベースの`property_listings.sales_assignee`フィールドが`null`のまま同期されない問題を修正します。

**影響範囲**: 物件リスト全体の営業担当フィールド（ほとんどの物件は正常に同期されているが、一部の物件で同期されていない）

**修正アプローチ**: 
1. GASの同期処理でスプレッドシートのヘッダー名を正確に認識しているか確認
2. 特定の条件下で同期がスキップされる可能性を調査
3. AA12497を含む問題のある物件を特定し、共通パターンを発見
4. 根本原因を修正し、手動で問題のある物件を再同期

## Glossary

- **Bug_Condition (C)**: スプレッドシートの「担当名（営業）」列に値が入力されているが、データベースの`sales_assignee`が`null`の状態
- **Property (P)**: スプレッドシートの「担当名（営業）」列の値が、データベースの`sales_assignee`フィールドに正しく同期される
- **Preservation**: 他の物件フィールド（物件番号、種別、契約日など）の同期は引き続き正常に動作する
- **GAS (Google Apps Script)**: スプレッドシートに紐づいた同期スクリプト（10分ごとのトリガー）
- **COLUMN_MAPPING**: GASとバックエンドのカラムマッピング定義（`'担当名（営業）': 'sales_assignee'`）

## Bug Details

### Bug Condition

バグは、スプレッドシートの「担当名（営業）」列に値が入力されているにもかかわらず、GASの10分トリガーが実行されても、データベースの`property_listings.sales_assignee`フィールドが`null`のまま更新されない場合に発生します。

**Formal Specification:**
```
FUNCTION isBugCondition(property)
  INPUT: property of type PropertyListing
  OUTPUT: boolean
  
  RETURN property.property_number IN ['AA12497', ...他の約5件]
         AND spreadsheet['担当名（営業）'] IS NOT NULL
         AND spreadsheet['担当名（営業）'] != ''
         AND database.property_listings.sales_assignee IS NULL
END FUNCTION
```

### Examples

- **AA12497**: スプレッドシートの「担当名（営業）」列に「裏」と入力されているが、データベースの`sales_assignee`は`null`
  - **期待される動作**: `sales_assignee = '裏'`
  - **実際の動作**: `sales_assignee = null`

- **他の約5件の物件**: 同様にスプレッドシートに値が入力されているが、データベースは`null`
  - **期待される動作**: スプレッドシートの値がデータベースに同期される
  - **実際の動作**: `sales_assignee = null`のまま

- **正常に同期されている物件**: ほとんどの物件は正常に同期されている
  - **期待される動作**: スプレッドシートの値がデータベースに同期される
  - **実際の動作**: 正常に同期されている

- **エッジケース**: 「担当名（営業）」列が空欄の物件
  - **期待される動作**: `sales_assignee = null`
  - **実際の動作**: `sales_assignee = null`（正常）

## Expected Behavior

### Preservation Requirements

**Unchanged Behaviors:**
- 他の物件フィールド（物件番号、種別、契約日など）の同期は引き続き正常に動作する
- 「事務担当名」フィールドの同期は引き続き正常に動作する
- 既存の物件データは保持され、営業担当フィールドのみが更新される

**Scope:**
「担当名（営業）」列以外のフィールドの同期は、完全に影響を受けないようにする必要があります。これには以下が含まれます：
- 物件番号、種別、契約日などの基本情報
- 事務担当名、売主情報などの他の担当者情報
- 価格、面積などの物件詳細情報

## Hypothesized Root Cause

バグの説明に基づいて、最も可能性の高い原因は以下の通りです：

1. **ヘッダー名の微妙な違い**: スプレッドシートのヘッダー名が「担当名（営業）」ではなく、全角/半角、スペース、特殊文字などが微妙に異なる可能性
   - 例: 「担当名（営業）」vs「担当名（営業）」（全角スペース）
   - 例: 「担当名（営業）」vs「担当名(営業)」（半角括弧）

2. **GASの同期処理でのヘッダー認識エラー**: `COLUMN_MAPPING`に定義されているヘッダー名と、実際のスプレッドシートのヘッダー名が一致していない
   - GASの`mapRowToRecord`関数でヘッダー名が正確にマッチしていない

3. **特定の行でのデータ型エラー**: 「担当名（営業）」列の値が特定の形式（例: 数値、日付）で入力されている場合、型変換エラーが発生している可能性

4. **GASの同期トリガーのタイミング問題**: 特定の条件下（例: スプレッドシートが編集中）でGASの同期がスキップされている可能性

## Correctness Properties

Property 1: Bug Condition - 営業担当フィールドの同期

_For any_ 物件で、スプレッドシートの「担当名（営業）」列に値が入力されている場合、GASの10分トリガーが実行された後、データベースの`property_listings.sales_assignee`フィールドにその値が正しく同期されるべきです。

**Validates: Requirements 2.1, 2.2, 2.3**

Property 2: Preservation - 他のフィールドの同期

_For any_ 物件で、「担当名（営業）」列以外のフィールド（物件番号、種別、契約日など）がスプレッドシートに入力されている場合、修正後もこれらのフィールドは引き続き正しく同期され、既存の動作が保持されるべきです。

**Validates: Requirements 3.1, 3.2, 3.3**

## Fix Implementation

### Changes Required

根本原因分析が正しいと仮定した場合：

**File**: `gas/property-listing-sync/PropertyListingSync.gs`

**Function**: `mapRowToRecord`

**Specific Changes**:
1. **ヘッダー名の正規化**: スプレッドシートから取得したヘッダー名を正規化（トリム、全角/半角統一）してから`COLUMN_MAPPING`と照合する
   - `header.trim()`でスペースを除去
   - 全角括弧と半角括弧を統一

2. **ヘッダー名のログ出力**: デバッグ用に、スプレッドシートから取得したヘッダー名をログに出力
   - 「担当名（営業）」列のインデックスと実際のヘッダー名を確認

3. **型変換の改善**: `convertValue`関数で、文字列型のフィールドに対して適切な型変換を行う
   - 数値や日付が入力されている場合でも、文字列として扱う

4. **エラーハンドリングの追加**: `mapRowToRecord`関数で、特定のフィールドの変換エラーが発生した場合でも、他のフィールドの同期を継続する
   - エラーログに詳細情報を出力

5. **手動同期スクリプトの作成**: AA12497を含む問題のある物件を手動で再同期するスクリプトを作成
   - `syncSingleProperty('AA12497')`を実行

## Testing Strategy

### Validation Approach

テスト戦略は2段階のアプローチに従います：まず、未修正コードでバグを再現する反例を表面化させ、次に修正が正しく動作し、既存の動作を保持することを検証します。

### Exploratory Bug Condition Checking

**Goal**: 未修正コードでバグを再現する反例を表面化させる。根本原因分析を確認または反証する。反証された場合は、再仮説を立てる必要があります。

**Test Plan**: スプレッドシートのヘッダー名を確認し、GASの同期処理でヘッダー名が正確に認識されているかをテストします。未修正コードでテストを実行し、失敗を観察して根本原因を理解します。

**Test Cases**:
1. **ヘッダー名確認テスト**: スプレッドシートのヘッダー行を取得し、「担当名（営業）」列のインデックスと実際のヘッダー名をログに出力（未修正コードで失敗する可能性）
2. **AA12497同期テスト**: AA12497の「担当名（営業）」列の値を取得し、GASの`mapRowToRecord`関数で正しくマッピングされるかをテスト（未修正コードで失敗）
3. **他の物件同期テスト**: 正常に同期されている物件で、「担当名（営業）」列の値が正しくマッピングされるかをテスト（未修正コードで成功）
4. **空欄テスト**: 「担当名（営業）」列が空欄の物件で、`sales_assignee`が`null`になるかをテスト（未修正コードで成功する可能性）

**Expected Counterexamples**:
- スプレッドシートのヘッダー名が「担当名（営業）」ではなく、微妙に異なる可能性（全角/半角、スペース、特殊文字）
- Possible causes: ヘッダー名の不一致、型変換エラー、GASの同期トリガーのタイミング問題

### Fix Checking

**Goal**: バグ条件が成立するすべての入力に対して、修正後の関数が期待される動作を生成することを検証します。

**Pseudocode:**
```
FOR ALL property WHERE isBugCondition(property) DO
  result := syncPropertyListing_fixed(property)
  ASSERT expectedBehavior(result)
END FOR
```

### Preservation Checking

**Goal**: バグ条件が成立しないすべての入力に対して、修正後の関数が元の関数と同じ結果を生成することを検証します。

**Pseudocode:**
```
FOR ALL property WHERE NOT isBugCondition(property) DO
  ASSERT syncPropertyListing_original(property) = syncPropertyListing_fixed(property)
END FOR
```

**Testing Approach**: プロパティベーステストは、保存チェックに推奨されます。理由は以下の通りです：
- 入力ドメイン全体で多くのテストケースを自動的に生成します
- 手動の単体テストでは見逃す可能性のあるエッジケースをキャッチします
- バグのない入力に対して動作が変更されていないことを強力に保証します

**Test Plan**: 未修正コードで他のフィールドの同期動作を観察し、その動作をキャプチャするプロパティベーステストを作成します。

**Test Cases**:
1. **他のフィールド同期保存テスト**: 未修正コードで物件番号、種別、契約日などのフィールドが正しく同期されることを観察し、修正後もこれが継続することを検証するテストを作成
2. **事務担当名同期保存テスト**: 未修正コードで「事務担当名」フィールドが正しく同期されることを観察し、修正後もこれが継続することを検証するテストを作成
3. **既存データ保持テスト**: 未修正コードで既存の物件データが保持されることを観察し、修正後もこれが継続することを検証するテストを作成

### Unit Tests

- スプレッドシートのヘッダー名を取得し、「担当名（営業）」列のインデックスを確認するテスト
- AA12497の「担当名（営業）」列の値を取得し、正しくマッピングされるかをテスト
- エッジケース（空欄、数値、日付）のテスト

### Property-Based Tests

- ランダムな物件データを生成し、「担当名（営業）」列の値が正しく同期されることを検証
- ランダムな物件構成を生成し、他のフィールドの同期動作が保持されることを検証
- 多くのシナリオで、バグのない入力に対して動作が変更されていないことをテスト

### Integration Tests

- GASの10分トリガーを手動で実行し、AA12497を含む問題のある物件が正しく同期されることを検証
- 複数の物件を同時に同期し、すべての物件が正しく同期されることを検証
- スプレッドシートを編集してから同期を実行し、変更が正しく反映されることを検証
