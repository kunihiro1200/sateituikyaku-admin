# Bugfix Requirements Document

## Introduction

物件番号AA12497を含む約5-6件の物件で、営業担当フィールド(`sales_assignee`)がスプレッドシートからデータベースに同期されていない問題を修正します。

**影響範囲**: 物件リスト全体の営業担当フィールド（ほとんどの物件は正常に同期されているが、一部の物件で同期されていない）

**具体例**:
- AA12497: スプレッドシートの「担当名（営業）」列に「裏」と入力されているが、データベースの`sales_assignee`は`null`
- 他に約5件の物件で同様の問題が発生

**根本原因の仮説**: 
- スプレッドシートのカラム名「担当名（営業）」は存在する（ユーザー確認済み）
- GASの同期処理とバックエンドのカラムマッピングは正しく定義されている
- しかし、一部の物件でのみ同期が失敗している
- 可能性: GASの同期トリガーが正常に動作していない、または特定の条件下で同期がスキップされている

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN スプレッドシートの「担当名（営業）」列に値が入力されている THEN データベースの`property_listings.sales_assignee`が`null`のまま同期されない

1.2 WHEN GASの10分トリガーが実行される THEN 「担当名（営業）」列のデータがデータベースに反映されない

1.3 WHEN 物件詳細画面で営業担当を確認する THEN 営業担当が表示されない（`null`）

### Expected Behavior (Correct)

2.1 WHEN スプレッドシートの「担当名（営業）」列に値が入力されている THEN データベースの`property_listings.sales_assignee`に正しく同期される

2.2 WHEN GASの10分トリガーが実行される THEN 「担当名（営業）」列のデータがデータベースに正しく反映される

2.3 WHEN 物件詳細画面で営業担当を確認する THEN 営業担当が正しく表示される

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 他の物件フィールド（物件番号、種別、契約日など）がスプレッドシートに入力されている THEN これらのフィールドは引き続き正しく同期される

3.2 WHEN 「事務担当名」フィールドがスプレッドシートに入力されている THEN `admin_assignee`フィールドは引き続き正しく同期される

3.3 WHEN 既存の物件データがデータベースに存在する THEN 既存データは保持され、営業担当フィールドのみが更新される
