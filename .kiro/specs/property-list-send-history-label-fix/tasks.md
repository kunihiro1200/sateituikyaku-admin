# Implementation Plan

- [x] 1. バグ条件の探索テストを作成する
  - **Property 1: Bug Condition** - セクションラベル誤表示バグ
  - **重要**: このテストは修正前のコードで必ず FAIL すること — FAIL がバグの存在を証明する
  - **修正やコードを直そうとしないこと（FAIL が正しい結果）**
  - **目的**: バグが存在することを示すカウンターエグザンプルを発見する
  - **スコープ**: `SellerSendHistory` コンポーネントをレンダリングし、ラベルテキストを検査する
  - `SellerSendHistory` をレンダリングして「売主への送信履歴」が存在することを確認（isBugCondition = true）
  - 「売主・物件の送信履歴」が存在しないことを確認（修正前は null を返す）
  - 修正前のコードでテストを実行する
  - **期待される結果**: テストが FAIL する（バグの存在を確認）
  - カウンターエグザンプルを記録する（例: `getByText('売主への送信履歴')` が要素を返す）
  - テストを作成・実行し、FAIL を確認したらタスク完了とする
  - _Requirements: 1.1_

  - 観察: 履歴アイテムクリック時にモーダルが開く
  - プロパティベーステスト: 任意の `propertyNumber` に対してラベル以外の動作が変わらないことを検証
  - 修正前のコードでテストを実行する
  - **期待される結果**: テストが PASS する（ベースライン動作を確認）
  - テストを作成・実行し、PASS を確認したらタスク完了とする
