# Bugfix Requirements Document

## Introduction

物件リスト画面（PropertyListingsPage）のサイドバーに2つのバグが存在する。

1. **サイドバーの枠線表示バグ**: サイドバーのカテゴリーリストを囲む `Paper` コンポーネントに枠線（border/outline）が表示されている。スクロールバーは維持したまま、枠線のみ解除する必要がある。

2. **「林専任公開中」カテゴリーフィルターバグ**: サイドバーに「林専任公開中」カテゴリーが表示されるが、クリックしても「物件データが見つかりませんでした」と表示される。`PropertyListingsPage.tsx` のフィルタリングロジックが `'林専任公開中'` に対応していないことが原因。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 物件リスト画面のサイドバーが表示される THEN サイドバー全体を囲む枠線（border）が表示される

1.2 WHEN サイドバーの「林専任公開中」カテゴリーをクリックする THEN 物件リストに「物件データが見つかりませんでした」と表示される（担当者「林田」の物件が存在するにもかかわらず）

### Expected Behavior (Correct)

2.1 WHEN 物件リスト画面のサイドバーが表示される THEN サイドバーの枠線は表示されず、スクロールバーのみ維持される

2.2 WHEN サイドバーの「林専任公開中」カテゴリーをクリックする THEN 担当者「林田」に紐づく物件が正しくフィルタリングされて表示される

### Unchanged Behavior (Regression Prevention)

3.1 WHEN サイドバーの「Y専任公開中」「生・専任公開中」「久・専任公開中」「U専任公開中」「林・専任公開中」「K専任公開中」「R専任公開中」「I専任公開中」カテゴリーをクリックする THEN 各担当者に紐づく物件が引き続き正しくフィルタリングされて表示される

3.2 WHEN サイドバーの「すべて」「未完了」「要値下げ」「未報告」などその他のカテゴリーをクリックする THEN 各カテゴリーのフィルタリングが引き続き正しく動作する

3.3 WHEN サイドバーのカテゴリーリストの件数が画面高さを超える THEN スクロールバーが引き続き表示されてスクロール操作が可能である
