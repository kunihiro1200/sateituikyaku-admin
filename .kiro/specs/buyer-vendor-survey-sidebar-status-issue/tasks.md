# Implementation Plan

- [x] 1. バグ条件探索テストを作成
  - **Property 1: Bug Condition** - スプレッドシート「業者向けアンケート」とデータベース`broker_survey`の同期不整合
  - **重要**: このテストは未修正コードで実行し、失敗することを確認する（失敗がバグの存在を証明）
  - **修正しようとしない**: テストが失敗しても、テストやコードを修正しない
  - **目的**: バグの存在を実証する反例を発見する
  - **スコープ付きPBTアプローチ**: 買主7260の具体的な失敗ケースにプロパティをスコープして再現性を確保
  - テスト実装の詳細（Bug Conditionから）:
    - 買主7260のスプレッドシート「業者向けアンケート」= "確認済み"
    - データベース`broker_survey` = "未"（または`vendor_survey` = "未"）
    - サイドバーステータス = "業者問合せあり"（誤り）
  - テストアサーションは設計のExpected Behavior Propertiesと一致させる:
    - スプレッドシート「業者向けアンケート」= "確認済み"の場合、データベース`broker_survey`も"確認済み"であるべき
    - `broker_survey` = "確認済み"の場合、サイドバーに「業者問合せあり」と表示されないべき
  - 未修正コードでテストを実行
  - **期待される結果**: テストが失敗（これは正しい - バグの存在を証明）
  - 反例を記録して根本原因を理解する
  - テストを作成・実行し、失敗を記録したらタスク完了とマークする
  - _Requirements: 1.1, 1.2_

- [x] 2. 保全プロパティテストを作成（修正実装前）
  - **Property 2: Preservation** - 非バグ条件での買主ステータス表示
  - **重要**: 観察優先方法論に従う
  - 未修正コードで非バグ入力（`broker_survey` = "未"、空欄など）の動作を観察
  - Preservation Requirementsから観察された動作パターンを捉えるプロパティベーステストを作成:
    - `broker_survey = "未"`の買主は「業者問合せあり」カテゴリに表示される
    - `broker_survey`が空欄の買主は「業者問合せあり」カテゴリに表示されない
    - 他のサイドバーカテゴリ（「内覧日前日」「当日TEL」など）は影響を受けない
  - プロパティベーステストは多数のテストケースを生成し、より強力な保証を提供
  - 未修正コードでテストを実行
  - **期待される結果**: テストが成功（ベースライン動作を確認）
  - テストを作成・実行し、未修正コードで成功したらタスク完了とマークする
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 3. 買主7260の「業者向けアンケート」同期不整合の修正

  - [x] 3.1 根本原因の調査と特定
    - 買主7260のデータ確認:
      - スプレッドシートの「業者向けアンケート」列の値を確認
      - データベースの`broker_survey`フィールドの値を確認
      - データベースの`vendor_survey`フィールドの値を確認（存在する場合）
    - GASカラムマッピング確認:
      - `gas_buyer_complete_code.js`の`BUYER_COLUMN_MAPPING`に「業者向けアンケート」→`broker_survey`のマッピングが存在するか確認
      - マッピングが正しいか確認
    - 根本原因を特定（GASマッピング不足、データ移行不完全、フィールド名混同のいずれか）
    - _Bug_Condition: isBugCondition(buyer) where spreadsheet.業者向けアンケート = "確認済み" AND database.broker_survey = "未"_
    - _Expected_Behavior: スプレッドシートとデータベースの`broker_survey`が常に同期される_
    - _Preservation: 他の買主のステータス計算とサイドバー表示が影響を受けない_
    - _Requirements: 1.1, 1.2, 2.1, 2.2, 3.1, 3.2, 3.3_

  - [x] 3.2 修正の実装

      - 修正したGASコードをスプレッドシートに紐づいたGASエディタにデプロイ
      - `syncSingleBuyer('7260')`を実行して買主7260のデータを同期
    - **ケース2: データ移行が不完全な場合**

      - 買主7260の`vendor_survey`の値を`broker_survey`にコピーするスクリプトを実装
      - スクリプトを実行
    - **ケース3: 両方のフィールドが存在し、混同されている場合**
      - `backend/src/services/BuyerStatusCalculator.ts`の`calculateBuyerStatus`にフォールバックロジックを追加
      - `broker_survey`が空の場合、`vendor_survey`を参照
      - 全買主のデータ移行スクリプトを実行
      - データ移行完了後、フォールバックロジックを削除
    - _Bug_Condition: isBugCondition(buyer) from design_
    - _Expected_Behavior: expectedBehavior(result) from design_
    - _Preservation: Preservation Requirements from design_
    - _Requirements: 1.1, 1.2, 2.1, 2.2, 3.1, 3.2, 3.3_
