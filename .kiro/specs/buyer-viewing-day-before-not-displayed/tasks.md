## タスク一覧

- [x] 1. GASの時間トリガー設定
  - Google スプレッドシート（買主リスト）を開く
  - 「拡張機能」→「Apps Script」を選択
  - GASエディタで `setupBuyerSyncTrigger` 関数を選択
  - 「実行」ボタンをクリック
  - 権限を承認（初回のみ）
  - ログに「✅ トリガーを設定しました: 10分ごと」と表示されることを確認
  - GASのトリガー管理画面で `syncBuyerList` の10分トリガーが表示されることを確認
  - _Requirements: FR-3_

- [x] 2. 手動実行してテスト
  - GASエディタで `testBuyerSync` 関数を選択
  - 「実行」ボタンをクリック
  - 実行ログを確認
  - 「内覧日前日カテゴリに追加」のログが表示されることを確認
  - 「buyer_sidebar_counts INSERT成功」のログが表示されることを確認
  - _Requirements: FR-1, AC-1_

- [x] 3. データベース確認
  - Supabaseダッシュボードを開く
  - `buyer_sidebar_counts` テーブルを確認
  - 以下のSQLを実行: `SELECT * FROM buyer_sidebar_counts WHERE category = 'viewingDayBefore';`
  - `viewingDayBefore` カテゴリのデータが存在することを確認
  - カウント数が正しいことを確認（例: 2件）
  - _Requirements: AC-1_

- [x] 4. フロントエンド確認
  - ブラウザで買主リストページ（`/buyers`）を開く
  - サイドバーに「内覧日前日」カテゴリが表示されることを確認
  - カウント数が正しいことを確認（例: 2件）
  - 「内覧日前日」カテゴリをクリック
  - 該当する買主のみが表示されることを確認
  - _Requirements: AC-2, AC-3_

- [x] 5. Checkpoint - 全ての確認が完了
  - GASの時間トリガーが設定されている
  - `buyer_sidebar_counts` テーブルに `viewingDayBefore` カテゴリのデータが存在する
  - サイドバーに「内覧日前日」カテゴリが表示される
  - カウント数が正しい
  - 問題があれば、ユーザーに質問する

## トラブルシューティング

### 問題1: トリガーが設定されない

**症状**: `setupBuyerSyncTrigger` を実行してもトリガーが表示されない

**解決策**:
1. GASエディタの「トリガー」タブを開く
2. 既存のトリガーを削除
3. `setupBuyerSyncTrigger` を再実行

### 問題2: 実行ログにエラーが表示される

**症状**: `testBuyerSync` を実行するとエラーが表示される

**解決策**:
1. エラーメッセージを確認
2. Supabaseの接続情報が正しいか確認
3. スプレッドシートの読み取り権限があるか確認

### 問題3: `buyer_sidebar_counts` テーブルにデータが保存されない

**症状**: GAS実行ログに「INSERT成功」と表示されるが、テーブルにデータがない

**解決策**:
1. Supabaseダッシュボードでテーブルが存在するか確認
2. GASのSupabase接続情報が正しいか確認
3. GASの実行ログで詳細なエラーメッセージを確認

### 問題4: サイドバーに「内覧日前日」カテゴリが表示されない

**症状**: データベースにはデータが存在するが、サイドバーに表示されない

**解決策**:
1. ブラウザのキャッシュをクリア
2. ページをリロード
3. バックエンドの `getSidebarCounts()` メソッドが正しく動作しているか確認
4. APIレスポンスに `viewingDayBefore` カテゴリが含まれているか確認（DevTools）
