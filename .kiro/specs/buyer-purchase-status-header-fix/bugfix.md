# Bugfix Requirements Document

## Introduction

AA18の物件詳細画面（売主管理システム）において、買付が削除された後もヘッダーに「買（専任　両手）」の表示が消えない問題を修正する。

現在はGAS（Google Apps Script）経由でスプレッドシートを通じてDBが更新される仕組みのため、買付削除後の反映に5分以上かかる。ユーザーの要望として、買付削除時はDBを直接更新することでリアルタイムに反映されるようにする。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 買付申し込みが存在する物件の買付を削除した THEN システムはヘッダーの「買（専任　両手）」表示をすぐに消去せず、5分以上表示し続ける

1.2 WHEN 買付が削除された THEN システムはGAS経由でスプレッドシートを通じてDBを更新するため、ヘッダー表示の更新が遅延する

### Expected Behavior (Correct)

2.1 WHEN 買付申し込みが存在する物件の買付を削除した THEN システムはDBを直接更新し、ヘッダーの「買（専任　両手）」表示をリアルタイムに消去する

2.2 WHEN 買付が削除された THEN システムはGASを経由せずDBを直接更新することで、ヘッダー表示に即時反映する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 買付申し込みが存在する物件を表示した THEN システムはヘッダーに「買（専任　両手）」を引き続き正しく表示する

3.2 WHEN 買付申し込みが新規登録された THEN システムはヘッダーの「買（専任　両手）」表示を引き続き正しく更新する

3.3 WHEN 買付以外の物件情報が変更された THEN システムはヘッダー表示を引き続き正しく維持する
