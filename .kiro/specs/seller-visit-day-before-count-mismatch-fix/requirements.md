# 売主リスト「訪問日前日」カテゴリのカウント不一致修正 - 要件定義

## 問題の概要

売主リストページにおいて、サイドバーの「①訪問日前日」カテゴリのカウント（2件）と、一覧に表示される売主の件数（1件）が一致していない。

## 現状の問題

### スクリーンショット分析

**サイドバー**:
- 「①訪問日前日: 2」と表示

**一覧**:
- AA10538（下地 純子）のみ表示（1件）

### 根本原因の仮説

1. **サイドバーのカウント計算ロジック**と**一覧のフィルタリングロジック**が異なる実装になっている
2. サイドバーはAPIから取得したカウント（`/api/sellers/sidebar-counts`）を使用
3. 一覧は`filterSellersByCategory()`関数でフィルタリング
4. 両者の判定条件が微妙に異なる可能性

## 要件

### 機能要件

#### FR-1: カウント計算とフィルタリングロジックの統一

**要件**: サイドバーのカウント計算と一覧のフィルタリングロジックを完全に統一する

**詳細**:
- 「訪問日前日」の判定条件を1つの関数（`isVisitDayBefore()`）に集約
- サイドバーのカウント計算（バックエンド）と一覧のフィルタリング（フロントエンド）の両方で同じロジックを使用
- 判定条件の定義:
  - 営担（`visitAssignee`）に入力がある
  - 今日が訪問日の「前営業日」である
    - 通常: 訪問日の**1日前**
    - 木曜訪問の場合のみ: **2日前**（水曜が定休日のため火曜に通知）
    - 日曜訪問の場合: **1日前**（土曜に通知）

#### FR-2: カウント数の一致検証

**要件**: サイドバーのカウント数と一覧の表示件数が常に一致することを保証する

**詳細**:
- サイドバーに「2件」と表示されている場合、一覧にも2件表示される
- カウント数が0件の場合、サイドバーのカテゴリは非表示になる

### 非機能要件

#### NFR-1: パフォーマンス

- カウント計算とフィルタリングの処理時間は100ms以内
- APIレスポンスタイムは500ms以内

#### NFR-2: 保守性

- 判定ロジックは1箇所に集約し、重複を排除
- コードコメントで判定条件を明記

#### NFR-3: テスタビリティ

- 判定ロジックは単体テスト可能な形で実装
- エッジケース（木曜訪問、日曜訪問、祝日など）のテストケースを作成

## 成功基準

### AC-1: カウント数の一致

**Given**: 売主リストページを開く  
**When**: サイドバーの「①訪問日前日」カテゴリを確認  
**Then**: サイドバーのカウント数と一覧の表示件数が一致する

**例**:
- サイドバー: 「①訪問日前日: 2」
- 一覧: 2件の売主が表示される（AA10538、もう1件）

### AC-2: 判定条件の正確性

**Given**: 以下の売主データが存在する
- 売主A: 営担=Y、訪問日=明日（木曜）
- 売主B: 営担=I、訪問日=明後日（金曜）
- 売主C: 営担=空欄、訪問日=明日

**When**: 今日が水曜日の場合  
**Then**: 
- 売主Aは「訪問日前日」に含まれる（木曜訪問の2日前）
- 売主Bは「訪問日前日」に含まれない（金曜訪問の2日前ではない）
- 売主Cは「訪問日前日」に含まれない（営担が空欄）

### AC-3: リアルタイム更新

**Given**: サイドバーのカウントが「2件」と表示されている  
**When**: 売主の訪問日を変更して保存  
**Then**: サイドバーのカウントと一覧の表示件数が即座に更新される

## 制約条件

### 技術的制約

- フロントエンド: React + TypeScript
- バックエンド: Node.js + TypeScript + Supabase
- 既存の`isVisitDayBefore()`関数を修正する（新規作成ではない）

### ビジネス制約

- 水曜日は定休日（木曜訪問の場合のみ2日前に通知）
- 日曜訪問は土曜に通知（1日前）
- 営担が「外す」の場合は有効な営業担当として扱う
