# Bugfix Requirements Document

## Introduction

買主リストページにおいて、サイドバーのカテゴリ件数表示は正しいが、カテゴリをクリックすると全件が表示されてしまう問題を修正します。この問題は、サイドバーカウントの高速取得（300ms）と全件データのバックグラウンド取得（23秒）の間にタイミングギャップが存在し、ユーザーが早期にカテゴリをクリックした場合にフィルタリングロジックが実行されないことが原因です。

過去にコミット9781d6cf、c43e05e8、3dc661bfで同様の問題を修正した履歴があり、今回も同じパターンの問題が再発しています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN ユーザーがサイドバーカウント取得後（300ms）、全件データ取得前（23秒）にカテゴリをクリックした場合 THEN システムは`allBuyersWithStatusRef.current.length > 0`の条件を満たさず、フィルタリングロジックを実行せずにAPIから全件データを取得してしまう

1.2 WHEN サイドバーのカテゴリ（例: 「担当(Y)」「当日TEL(Y)」など）をクリックした場合 THEN システムは該当カテゴリのみを表示すべきだが、全件が表示される

1.3 WHEN `BuyersPage.tsx`の148-150行目の条件`if (sidebarLoadedRef.current && allBuyersWithStatusRef.current.length > 0)`が評価される場合 THEN `allBuyersWithStatusRef.current`が空の場合、フロント側フィルタリングがスキップされる

### Expected Behavior (Correct)

2.1 WHEN ユーザーがサイドバーのカテゴリをクリックした場合 THEN システムは該当カテゴリに属する買主のみを表示する

2.2 WHEN 全件データがまだ取得できていない状態でカテゴリをクリックした場合 THEN システムはAPIにフィルタパラメータを渡して該当データのみを取得する

2.3 WHEN 全件データが既に取得済みの状態でカテゴリをクリックした場合 THEN システムはフロント側でフィルタリングを実行してAPIコールを回避する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN サイドバーの「All」カテゴリをクリックした場合 THEN システムは全件を表示し続ける

3.2 WHEN サイドバーカウントの表示が正しく動作している場合 THEN システムはカウント表示を変更せずに維持する

3.3 WHEN 全件データが取得済みの状態でフィルタリングが実行される場合 THEN システムは既存のフロント側フィルタリングロジックを使用し続ける

3.4 WHEN 検索クエリが入力されている状態でカテゴリフィルタが適用される場合 THEN システムは検索とカテゴリフィルタの両方を適用し続ける
