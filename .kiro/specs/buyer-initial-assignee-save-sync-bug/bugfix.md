# バグ修正要件ドキュメント

## Introduction

買主詳細画面で初動担当フィールドに「久」を選択して保存ボタンを押すと、DBへの保存は成功するがスプレッドシートへの同期に失敗する問題を修正します。この問題により、DBとスプレッドシートのデータ不整合が発生し、ユーザーがスプレッドシートで最新の初動担当情報を確認できない状態になっています。

エラーログには404エラー（`/api/buyers/7260/related`）、409エラー（`/api/buyers/7260?sync=true`）、および非同期メッセージチャネルエラーが記録されています。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 買主詳細画面で初動担当フィールドに「久」を選択して保存ボタンを押す THEN システムはDBへの保存を成功させるが、スプレッドシートへの同期に失敗し、エラーメッセージ「DBへの保存は完了しましたが、スプレッドシートへの同期に失敗しました」を表示する

1.2 WHEN 初動担当フィールドの保存処理が実行される THEN システムは404エラー（`/api/buyers/7260/related`）を発生させる

1.3 WHEN 初動担当フィールドの保存処理が実行される THEN システムは409エラー（`/api/buyers/7260?sync=true`）を発生させる

1.4 WHEN 初動担当フィールドの保存処理が実行される THEN システムは非同期メッセージチャネルエラー「A listener indicated an asynchronous response by returning true, but the message channel closed before a response was received」を発生させる

### Expected Behavior (Correct)

2.1 WHEN 買主詳細画面で初動担当フィールドに「久」を選択して保存ボタンを押す THEN システムはDBへの保存とスプレッドシートへの同期の両方を成功させ、エラーメッセージを表示しない

2.2 WHEN 初動担当フィールドの保存処理が実行される THEN システムは404エラーを発生させず、関連買主情報を正常に取得する

2.3 WHEN 初動担当フィールドの保存処理が実行される THEN システムは409エラーを発生させず、買主データの更新とスプレッドシート同期を正常に完了する

2.4 WHEN 初動担当フィールドの保存処理が実行される THEN システムは非同期メッセージチャネルエラーを発生させず、非同期処理を正常に完了する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 買主詳細画面で初動担当フィールドに「久」以外の値（例：「Y」「I」など）を選択して保存する THEN システムはDBへの保存とスプレッドシートへの同期の両方を引き続き正常に実行する

3.2 WHEN 買主詳細画面で初動担当以外のフィールド（例：問合せ元、最新状況など）を編集して保存する THEN システムはDBへの保存とスプレッドシートへの同期の両方を引き続き正常に実行する

3.3 WHEN 買主番号7260以外の買主で初動担当フィールドを編集して保存する THEN システムはDBへの保存とスプレッドシートへの同期の両方を引き続き正常に実行する

3.4 WHEN 保存ボタンを押さずに初動担当フィールドを変更する THEN システムは引き続きフィールドの変更を検知し、保存ボタンを有効化する
