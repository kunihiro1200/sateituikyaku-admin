# バグ修正要件ドキュメント

## はじめに

売主AA13831がスプレッドシートの売主リストから削除されて1時間以上経過しているにもかかわらず、データベース（DB）にはまだ表示されている（`deleted_at` が設定されていない）。

GASの `syncSellerList` 関数が10分ごとに実行されており、削除同期（Phase 3）でDBにあってスプレッドシートにない売主をソフトデリートする仕組みになっている。正常に動作していれば最大10分以内に削除されるはずだが、1時間以上経過しても削除されていない。

このバグは、削除同期が何らかの理由でAA13831に対して正しく機能していないことを示している。

---

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 売主がスプレッドシートの売主リストから削除されてから10分以上経過した場合 THEN システムはDBの該当売主レコードに `deleted_at` を設定せず、売主がDBに残り続ける

1.2 WHEN GASの `syncSellerList` が実行される場合 THEN システムはDBにあってスプレッドシートにない売主（AA13831）を削除対象として検出しない、またはソフトデリートを実行しない

1.3 WHEN 削除同期（Phase 3）が実行される場合 THEN システムはAA13831に対してバリデーションエラーまたは処理スキップが発生し、`deleted_at` が設定されない

### 期待される動作（正常）

2.1 WHEN 売主がスプレッドシートの売主リストから削除されてから最大10分が経過した場合 THEN システムはDBの該当売主レコードの `deleted_at` に削除日時を設定する（ソフトデリート）

2.2 WHEN GASの `syncSellerList` が実行される場合 THEN システムはDBにあってスプレッドシートにない全ての売主を削除対象として正しく検出する

2.3 WHEN 削除同期（Phase 3）がAA13831に対して実行される場合 THEN システムはバリデーションを通過し、`deleted_at` を設定してソフトデリートを完了する

### 変更してはいけない動作（リグレッション防止）

3.1 WHEN スプレッドシートに存在する売主に対して同期が実行される場合 THEN システムはその売主の `deleted_at` を変更せず、アクティブな状態を維持し続ける

3.2 WHEN アクティブな契約ステータス（「専任契約中」「一般契約中」）を持つ売主がスプレッドシートから削除された場合 THEN システムはその売主の削除をブロックし、手動レビュー対象としてマークする

3.3 WHEN 削除同期が正常に完了した場合 THEN システムは新規追加同期（Phase 1）および更新同期（Phase 2）の動作に影響を与えない

3.4 WHEN ソフトデリートされた売主が存在する場合 THEN システムはその売主を `getAllActiveDbSellerNumbers()` の結果から除外し続ける
