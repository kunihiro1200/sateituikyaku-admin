# 実装計画

- [ ] 1. バグ条件探索テストを作成
  - **Property 1: Bug Condition** - フィルタが正しく動作しない
  - **重要**: このテストは修正前のコードで実行し、**失敗することを確認**する（失敗がバグの存在を証明）
  - **修正しようとしないこと**: テストが失敗してもコードを修正しない
  - **注意**: このテストは期待される動作をエンコードしており、実装後にパスすることで修正を検証する
  - **目標**: バグを実証する反例を見つける
  - **スコープ付きPBTアプローチ**: 決定論的バグのため、具体的な失敗ケースにプロパティをスコープして再現性を確保
  - サイトフィルタ「す」を指定した場合、`inquiry_site`カラム（存在しない）で検索されるため全データが返される
  - 種別フィルタ「土地」を指定した場合、`property_type`カラム（存在しない）で検索されるため全データが返される
  - 状況（当社）フィルタ「追客中」を指定した場合、部分一致検索（`ilike`）のため意図しないデータも含まれる
  - テストアサーションはdesignのExpected Behavior Propertiesと一致させる
  - 修正前のコードでテストを実行
  - **期待される結果**: テスト失敗（これが正しい - バグの存在を証明）
  - 反例を文書化して根本原因を理解する
  - テストが作成され、実行され、失敗が文書化されたらタスク完了とする
  - _Requirements: 1.1, 1.2, 1.3_

- [ ] 2. 保存プロパティテストを作成（修正実装前）
  - **Property 2: Preservation** - 他のフィルタ機能の動作
  - **重要**: 観察優先の方法論に従う
  - 修正前のコードで非バグ入力（フィルタ未指定、または他のフィルタ）の動作を観察
  - Preservation Requirementsから観察された動作パターンをキャプチャするプロパティベーステストを作成
  - プロパティベーステストは多くのテストケースを自動生成し、より強力な保証を提供
  - 修正前のコードでテストを実行
  - **期待される結果**: テストパス（ベースライン動作を確認）
  - テストが作成され、実行され、修正前のコードでパスしたらタスク完了とする
