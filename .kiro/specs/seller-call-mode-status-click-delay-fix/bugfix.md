# バグ修正要件ドキュメント

## はじめに

売主リストの通話モードページ（`/sellers/:id/call`）において、ステータスセクションの「次電日」フィールドを含む各フィールドを変更して「ステータスを更新」ボタンを押しても、複数回クリックしないと反映されないバグが継続して発生している。

コミット `9e62391` で「ステータス2回保存バグ」の修正が行われたが、依然として何度かクリックしないと反応しない状態が続いている。

コードを調査した結果、`statusChanged` フラグは `onChange` イベントで手動に `true` に設定される設計になっている。次電日フィールド（`type="date"`）の場合、`onChange` が発火するのはカレンダーUIで日付を確定したタイミングであり、フィールドをクリックしてカレンダーを開いただけでは `statusChanged` が `true` にならない。また、`onClick` ハンドラは `showPicker()` を呼ぶだけで `setStatusChanged(true)` を呼ばない。

このため、ユーザーが次電日フィールドをクリックしてカレンダーから日付を選択しても、`onChange` が正しく発火しないケース（ブラウザ差異・MUI TextField の挙動）や、日付を選択後に保存ボタンが有効化されない状態が発生し、複数回の操作が必要になっている。

対象ファイル：
- フロントエンド: `frontend/frontend/src/pages/CallModePage.tsx`

---

## バグ分析

### 現在の動作（不具合）

1.1 WHEN ユーザーが次電日フィールドをクリックしてカレンダーから日付を選択する THEN システムは `onChange` が発火しない場合があり、`statusChanged` フラグが `true` にならず「ステータスを更新」ボタンが有効化されない

1.2 WHEN ユーザーが次電日フィールドで日付を選択して「ステータスを更新」ボタンを1回押す THEN システムはボタンが無効状態（`statusChanged=false`）のため押下を受け付けず、再度フィールドを操作しないと保存できない

1.3 WHEN ユーザーが次電日フィールドを複数回クリックまたは操作する THEN システムはある操作のタイミングで `onChange` が発火し、ようやく `statusChanged` が `true` になって「ステータスを更新」ボタンが有効化される

### 期待される動作（正常）

2.1 WHEN ユーザーが次電日フィールドをクリックしてカレンダーから日付を選択する THEN システムは1回の操作で `statusChanged` フラグを `true` に設定し、「ステータスを更新」ボタンを即座に有効化する

2.2 WHEN ユーザーが「ステータスを更新」ボタンを1回押す THEN システムはAPIを呼び出して保存処理を実行し、成功後に「ステータスを更新しました」の成功メッセージを表示して保存ボタンを無効化する

2.3 WHEN 次電日フィールドで日付が選択される THEN システムは選択された日付を `editedNextCallDate` に反映し、`statusChanged` を `true` に設定する

### 変更してはいけない動作（リグレッション防止）

3.1 WHEN ユーザーが次電日フィールドで日付を変更して保存が成功する THEN システムは引き続き `savedNextCallDate` を更新し、`statusChanged` フラグを `false` にリセットする

3.2 WHEN ユーザーが状況（当社）・確度・Pinrichステータスなど他のフィールドを変更する THEN システムは引き続き `statusChanged` フラグを `true` に設定し、「ステータスを更新」ボタンを有効化する

3.3 WHEN ページを初回ロードする THEN システムは引き続き売主データを取得してステータスフィールドを初期化し、`statusChanged` フラグを `false` に設定する

3.4 WHEN 保存処理中（`savingStatus=true`）である THEN システムは引き続き「ステータスを更新」ボタンを無効化してローディングインジケーターを表示する

3.5 WHEN バリデーションエラーが発生する（確度未入力、専任他決の必須フィールド未入力等） THEN システムは引き続きエラーメッセージを表示してAPIを呼び出さない

3.6 WHEN 追客中かつ次電日が未入力の状態でページを離れようとする THEN システムは引き続き遷移をブロックして次電日入力を促すダイアログを表示する
