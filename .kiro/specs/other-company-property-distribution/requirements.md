# 要件定義書：他社物件新着配信

## はじめに

買主リスト一覧ページのヘッダーに「他社物件新着配信」ボタンを追加し、買主の希望条件に基づいて他社物件の新着情報を配信するための機能を実装します。

## 用語集

- **System**: 買主管理システム（社内管理システム）
- **User**: 不動産会社の従業員
- **Buyer**: 買主
- **Property**: 物件
- **Area**: エリア（①～㊸の38エリア）
- **Property_Type**: 物件種別（戸建、マンション、土地）
- **Price_Range**: 価格帯（~1900万円、1000万円~2999万円、2000万円以上）
- **Buyer_Detail_Page**: 買主詳細画面
- **Nearby_Properties**: 近隣物件（買主詳細画面に表示される物件リスト）
- **Distribution_Page**: 他社物件新着配信ページ（新規作成するページ）

## 要件

### 要件1: ヘッダーボタンの追加

**ユーザーストーリー**: 従業員として、買主リスト一覧ページから他社物件新着配信ページに簡単にアクセスしたい。

#### 受入基準

1. THE System SHALL 買主リスト一覧ページのヘッダーに「他社物件新着配信」ボタンを表示する
2. THE System SHALL 「他社物件新着配信」ボタンを「公開物件サイト」ボタンの右隣に配置する
3. WHEN User が「他社物件新着配信」ボタンをクリックする、THE System SHALL 他社物件新着配信ページ（`/buyers/other-company-distribution`）に遷移する

### 要件2: 配信ページのヘッダー部分

**ユーザーストーリー**: 従業員として、希望エリア、価格帯、物件種別を選択して、条件に合う買主を絞り込みたい。

#### 受入基準

1. THE System SHALL 配信ページのヘッダーに「希望エリア」ドロップダウンを表示する
2. THE System SHALL 「希望エリア」ドロップダウンに①～㊸の38エリアを選択肢として表示する
3. THE System SHALL 配信ページのヘッダーに「価格種別」ドロップダウンを表示する
4. THE System SHALL 「価格種別」ドロップダウンに以下の選択肢を表示する：「指定なし」「~1900万円」「1000万円~2999万円」「2000万円以上」
5. THE System SHALL 配信ページのヘッダーに「物件種別」ボタン（戸建、マンション、土地）を表示する
6. THE System SHALL 「物件種別」ボタンを複数選択可能にする

### 要件3: エリアグループルール

**ユーザーストーリー**: 従業員として、特定のエリアを選択したときに関連エリアも自動的に含めたい。

#### 受入基準

1. WHEN User が①～⑧（大分）のいずれかを選択する、THE System SHALL ㊵も自動的に選択に含める
2. WHEN User が⑨～⑮（別府市）のいずれかを選択する、THE System SHALL ㊶も自動的に選択に含める

### 要件4: 買主リストの表示ロジック

**ユーザーストーリー**: 従業員として、選択した条件に基づいて、条件に合う買主のリストを表示したい。

#### 受入基準

1. WHEN User が希望エリア、価格帯、物件種別を選択する、THE System SHALL 以下の条件を満たす買主を表示する
   - 買主の希望エリアが選択されたエリアまたは関連エリアに含まれる
   - 買主の希望種別が選択された物件種別に含まれる
   - 買主の希望価格が選択された価格帯に含まれる
2. THE System SHALL 買主詳細画面の「近隣物件」と同じ画面レイアウトで買主リストを表示する
3. THE System SHALL 買主リストに以下の情報を表示する：買主番号、氏名、希望エリア、希望種別、希望価格

### 要件5: 価格帯フィルタリングロジック

**ユーザーストーリー**: 従業員として、価格帯を選択したときに、買主の希望価格が選択された価格帯に含まれるかを正確に判定したい。

#### 受入基準

1. WHEN User が「~1900万円」を選択する、THE System SHALL 買主の希望価格が1900万円以下の買主を表示する
2. WHEN User が「1000万円~2999万円」を選択する、THE System SHALL 買主の希望価格が1000万円以上2999万円以下の買主を表示する
3. WHEN User が「2000万円以上」を選択する、THE System SHALL 買主の希望価格が2000万円以上の買主を表示する
4. WHEN User が「指定なし」を選択する、THE System SHALL 全ての買主を表示する（価格帯フィルタを適用しない）

### 要件6: 希望エリアフィルタリングロジック

**ユーザーストーリー**: 従業員として、希望エリアを選択したときに、買主の希望エリアが選択されたエリアに含まれるかを正確に判定したい。

#### 受入基準

1. WHEN User が希望エリアを選択する、THE System SHALL 買主の`desired_area`フィールドに選択されたエリアが含まれる買主を表示する
2. THE System SHALL 買主の`desired_area`フィールドが複数のエリアを含む場合（例：「①、②、③」）、いずれか1つでも選択されたエリアに一致すれば表示する
3. WHEN User が①～⑧のいずれかを選択する、THE System SHALL 買主の`desired_area`フィールドに①～⑧または㊵が含まれる買主を表示する
4. WHEN User が⑨～⑮のいずれかを選択する、THE System SHALL 買主の`desired_area`フィールドに⑨～⑮または㊶が含まれる買主を表示する

### 要件7: 希望種別フィルタリングロジック

**ユーザーストーリー**: 従業員として、物件種別を選択したときに、買主の希望種別が選択された物件種別に含まれるかを正確に判定したい。

#### 受入基準

1. WHEN User が「戸建」を選択する、THE System SHALL 買主の`desired_property_type`フィールドに「戸建て」が含まれる買主を表示する
2. WHEN User が「マンション」を選択する、THE System SHALL 買主の`desired_property_type`フィールドに「マンション」が含まれる買主を表示する
3. WHEN User が「土地」を選択する、THE System SHALL 買主の`desired_property_type`フィールドに「土地」が含まれる買主を表示する
4. WHEN User が複数の物件種別を選択する、THE System SHALL いずれか1つでも買主の`desired_property_type`フィールドに含まれる買主を表示する

### 要件8: 初期表示状態

**ユーザーストーリー**: 従業員として、配信ページを開いたときに、デフォルトの状態で買主リストを表示したい。

#### 受入基準

1. WHEN User が配信ページを開く、THE System SHALL 希望エリアを「未選択」状態で表示する
2. WHEN User が配信ページを開く、THE System SHALL 価格種別を「指定なし」状態で表示する
3. WHEN User が配信ページを開く、THE System SHALL 物件種別を「未選択」状態で表示する
4. WHEN User が配信ページを開く、THE System SHALL 買主リストを空の状態で表示する（条件未選択のため）

### 要件9: レスポンシブデザイン

**ユーザーストーリー**: 従業員として、モバイルデバイスでも配信ページを快適に使用したい。

#### 受入基準

1. THE System SHALL 配信ページをモバイルデバイスで表示可能にする
2. WHEN User がモバイルデバイスで配信ページを開く、THE System SHALL ヘッダー部分を縦に並べて表示する
3. WHEN User がモバイルデバイスで配信ページを開く、THE System SHALL 買主リストをカード形式で表示する

### 要件10: パフォーマンス

**ユーザーストーリー**: 従業員として、条件を選択したときに、買主リストが素早く表示されることを期待する。

#### 受入基準

1. WHEN User が条件を選択する、THE System SHALL 2秒以内に買主リストを表示する
2. THE System SHALL 買主リストの取得にデータベースインデックスを使用する
3. THE System SHALL 買主リストの取得結果を10分間キャッシュする
