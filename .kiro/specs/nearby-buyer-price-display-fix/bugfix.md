# バグ修正要件ドキュメント

## はじめに

売主リストの近隣買主候補画面において、買主の「価格」列に誤った値が表示されるバグを修正する。
具体的には、買主番号7363（西田卓司）の価格が「0.238万円」と表示されているが、正しくは「2380万円」であるべき。
推測される原因は、すでに万円単位の値（2380）をさらに10000で割るなど、単位変換処理の二重適用または誤った変換係数の使用。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 近隣買主候補リストを表示する THEN システムは価格を誤った単位変換で計算し、正しい価格（2380万円）が「0.238万円」として表示される

1.2 WHEN 万円単位の価格データ（例: 2380）が価格列に渡される THEN システムはその値をさらに10000（または別の係数）で割り、極端に小さい値（0.238）を万円表示する

### 期待される動作（正しい動作）

2.1 WHEN 近隣買主候補リストを表示する THEN システムは価格を正しく表示し、2380万円のデータは「2380万円」として表示されるものとする

2.2 WHEN 万円単位の価格データ（例: 2380）が価格列に渡される THEN システムはその値をそのまま万円として表示し、追加の単位変換を行わないものとする

### 変化しない動作（リグレッション防止）

3.1 WHEN 価格が正常に表示されている他の買主候補の行を表示する THEN システムは引き続き各買主の価格を正しく表示し続けるものとする

3.2 WHEN 近隣買主候補リスト全体（197件）を表示する THEN システムは引き続き件数・氏名・配布エリア・問合せ物件情報などの他の列を正しく表示し続けるものとする

3.3 WHEN 価格が未設定（null または 0）の買主候補を表示する THEN システムは引き続き価格なしの状態を正しく表示し続けるものとする

## バグ条件の定式化

### バグ条件関数

```pascal
FUNCTION isBugCondition(X)
  INPUT: X of type BuyerCandidatePrice
  OUTPUT: boolean

  // 価格データが万円単位で格納されているにもかかわらず、
  // 表示時にさらに単位変換（÷10000 等）が適用される場合にtrueを返す
  RETURN X.price_unit = "万円" AND display_applies_extra_conversion(X)
END FUNCTION
```

### 修正確認プロパティ

```pascal
// プロパティ: 修正確認 - 価格の正しい万円表示
FOR ALL X WHERE isBugCondition(X) DO
  result ← displayPrice'(X)
  ASSERT result = X.price_in_man_yen AND no_extra_conversion(result)
END FOR
```

### 保全確認プロパティ

```pascal
// プロパティ: 保全確認 - 正常な価格データへの影響なし
FOR ALL X WHERE NOT isBugCondition(X) DO
  ASSERT displayPrice(X) = displayPrice'(X)
END FOR
```
