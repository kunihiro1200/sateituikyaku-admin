# バグフィックス要件ドキュメント

## はじめに

物件詳細画面（`PropertyListingDetailPage`）の「特記・備忘録」セクションに保存ボタンが存在しないため、ユーザーが「特記」（`special_notes`）および「備忘録」（`memo`）テキストエリアに入力した内容がデータベースに保存されない。また、保存後にスプレッドシートへの自動同期も行われない。

バックエンドの更新API（`PUT /api/property-listings/:propertyNumber`）およびスプレッドシート同期処理（`syncToSpreadsheet`）は既に実装済みであり、カラムマッピング（`●特記` → `special_notes`、`備忘録` → `memo`）も設定済みである。フロントエンドの保存ボタンが欠落していることが根本原因。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN ユーザーが「特記」テキストエリアに文字を入力する THEN システムは `editedData.special_notes` に値を保持するが、保存ボタンが存在しないためデータベースに保存されない

1.2 WHEN ユーザーが「備忘録」テキストエリアに文字を入力する THEN システムは `editedData.memo` に値を保持するが、保存ボタンが存在しないためデータベースに保存されない

1.3 WHEN ユーザーが「特記・備忘録」セクションを編集して画面を離れる THEN システムは入力内容を破棄し、スプレッドシートへの同期も実行されない

### 期待される動作（正しい動作）

2.1 WHEN ユーザーが「特記」または「備忘録」テキストエリアを編集した後に保存ボタンを押す THEN システムは `PUT /api/property-listings/:propertyNumber` を呼び出して `special_notes` および `memo` をデータベースに保存しなければならない

2.2 WHEN データベースへの保存が成功する THEN システムは `syncToSpreadsheet` を通じてスプレッドシートのAT列（特記）およびDG列（備忘録）に自動同期しなければならない

2.3 WHEN 保存が成功する THEN システムは「特記・備忘録を保存しました」というスナックバー通知を表示しなければならない

2.4 WHEN 保存が失敗する THEN システムはエラーメッセージをスナックバーで表示しなければならない

### 変更されない動作（リグレッション防止）

3.1 WHEN ユーザーが他のセクション（価格情報、基本情報、物件詳細など）を編集して保存する THEN システムは引き続き各セクションの保存処理を正常に実行しなければならない

3.2 WHEN 「特記・備忘録」セクション以外のフィールドが更新される THEN システムはスプレッドシートへの既存の同期処理を変更せずに継続して実行しなければならない

3.3 WHEN ユーザーが「特記・備忘録」セクションを編集せずに他のセクションを保存する THEN システムは「特記・備忘録」の既存データを変更してはならない
