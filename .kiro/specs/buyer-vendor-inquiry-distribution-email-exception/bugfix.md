# Bugfix Requirements Document

## Introduction

買主リストの「配信メール」必須条件において、「業者問合せ」が「業者問合せ」の場合に希望条件が不要であるにもかかわらず、必須扱いになってしまうバグを修正します。業者問合せの場合は配信メールを送らないため、希望条件の入力は不要です。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 「業者問合せ」が「業者問合せ」であり、かつ「配信メール」が「要」の場合 THEN システムは希望条件（エリア、希望種別、価格帯）を必須項目として扱う

1.2 WHEN 「業者問合せ」が「業者問合せ」の買主が希望条件を未入力のまま保存しようとする場合 THEN システムはバリデーションエラーを表示して保存を阻止する

### Expected Behavior (Correct)

2.1 WHEN 「業者問合せ」が「業者問合せ」の場合 THEN システムは「配信メール」の値に関係なく希望条件を必須項目として扱わない

2.2 WHEN 「業者問合せ」が「業者問合せ」の買主が希望条件を未入力のまま保存しようとする場合 THEN システムはバリデーションエラーを表示せず正常に保存する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 「業者問合せ」が空欄または「業者問合せ」以外の値であり、かつ「配信メール」が「要」の場合 THEN システムは引き続き希望条件（エリア、希望種別、価格帯）を必須項目として扱う

3.2 WHEN 「業者問合せ」が空欄または「業者問合せ」以外の値であり、かつ「配信メール」が「不要」の場合 THEN システムは引き続き希望条件を必須項目として扱わない

3.3 WHEN 「業者問合せ」が「業者問合せ」の場合 THEN システムは引き続き「問合せ元」を必須項目として扱わない（既存の例外処理を維持）
