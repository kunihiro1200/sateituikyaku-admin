# バグ修正要件ドキュメント

## はじめに

物件リストのサイドバーカテゴリーに「SUUMO URL 要登録」（および「レインズ登録＋SUUMO URL 要登録」）が表示されないバグ。
対象物件番号 AA3959 で確認されており、以下の4条件をすべて満たす物件がサイドバーに表示されるべきにもかかわらず、表示されていない。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN atbb成約済み/非公開 が「一般・公開中」または「専任・公開中」であり、かつ業務依頼テーブルに公開予定日が TODAY()-1 以前のレコードが存在し、かつ Suumo URL が空白であり、かつ Suumo登録 が「S不要」ではない場合 THEN システムはサイドバーカテゴリーに「SUUMO URL 要登録」または「レインズ登録＋SUUMO URL 要登録」を表示しない

1.2 WHEN 物件番号 AA3959 がすべての表示条件を満たしている場合 THEN システムはその物件をサイドバーの該当カテゴリーに含めない

### 期待される動作（正しい動作）

2.1 WHEN atbb成約済み/非公開 が「一般・公開中」であり、かつ業務依頼テーブルに公開予定日が TODAY()-1 以前のレコードが存在し、かつ Suumo URL が空白であり、かつ Suumo登録 が「S不要」ではない場合 THEN システムはサイドバーカテゴリーに「SUUMO URL 要登録」を表示しなければならない

2.2 WHEN atbb成約済み/非公開 が「専任・公開中」であり、かつ業務依頼テーブルに公開予定日が TODAY()-1 以前のレコードが存在し、かつ Suumo URL が空白であり、かつ Suumo登録 が「S不要」ではない場合 THEN システムはサイドバーカテゴリーに「レインズ登録＋SUUMO URL 要登録」を表示しなければならない

2.3 WHEN 物件番号 AA3959 がすべての表示条件を満たしている場合 THEN システムはその物件をサイドバーの該当カテゴリーに含めなければならない

### 変わらない動作（リグレッション防止）

3.1 WHEN Suumo URL が入力済みの場合 THEN システムは引き続きその物件を「SUUMO URL 要登録」カテゴリーに含めないようにしなければならない

3.2 WHEN Suumo登録 が「S不要」の場合 THEN システムは引き続きその物件を「SUUMO URL 要登録」カテゴリーに含めないようにしなければならない

3.3 WHEN atbb成約済み/非公開 が「一般・公開中」でも「専任・公開中」でもない場合 THEN システムは引き続きその物件を「SUUMO URL 要登録」カテゴリーに含めないようにしなければならない

3.4 WHEN 業務依頼テーブルに公開予定日が TODAY()-1 以前のレコードが存在しない場合 THEN システムは引き続きその物件を「SUUMO URL 要登録」カテゴリーに含めないようにしなければならない

3.5 WHEN 他のサイドバーカテゴリー（例：「レインズ登録要」「本日電話」など）の表示条件を満たす物件がある場合 THEN システムは引き続きそれらのカテゴリーを正しく表示しなければならない
