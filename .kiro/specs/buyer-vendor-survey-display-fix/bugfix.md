# バグ修正要件ドキュメント

## Introduction

買主リストにおける2つの表示・バリデーション問題を修正する：

1. **買主リスト7260の「業者向けアンケート」表示問題**: スプレッドシートからデータベースに正しく同期されているが、ブラウザで表示されない
   - 根本原因: フロントエンドとバックエンドでフィールド名が不一致
   - データベース・GAS: `vendor_survey` ✅
   - フロントエンド: `broker_survey` ❌

2. **買主リスト7267の「持家ヒアリング結果」誤必須問題**: 「問合時持家ヒアリング」が完全に空欄なのに「持家ヒアリング結果」が必須扱いになる
   - 根本原因: データベースに見えない空白文字・改行・特殊文字が保存されている可能性
   - 現在のロジック: `owned_home_hearing_inquiry` に値があれば必須
   - 問題: 空白文字のみの値を「値あり」と判定している

## Bug Analysis

### Current Behavior (Defect)

**問題1: 業者向けアンケート表示問題（買主7260）**

1.1 WHEN 買主詳細ページ（BuyerDetailPage.tsx）を表示する THEN システムは `buyer?.broker_survey` を参照するが、APIレスポンスには `vendor_survey` しか含まれないため、値が表示されない

1.2 WHEN 新規買主ページ（NewBuyerPage.tsx）で「業者向けアンケート」を保存する THEN システムは `broker_survey` フィールド名でデータを送信するが、バックエンドは `vendor_survey` を期待しているため、データが正しく保存されない可能性がある

1.3 WHEN 買主詳細ページで「業者向けアンケート」フィールドを条件付き表示する THEN システムは `buyer?.broker_survey` の値をチェックするが、実際のデータは `vendor_survey` に保存されているため、値があっても非表示になる

**問題2: 持家ヒアリング結果の誤必須問題（買主7267）**

1.4 WHEN 買主番号7267の詳細ページを表示する THEN 「問合時持家ヒアリング」が完全に空欄（イニシャルボタンが全て未選択）なのに、「持家ヒアリング結果」フィールドが赤枠で必須扱いになる

1.5 WHEN データベースの `owned_home_hearing_inquiry` カラムを確認する THEN 見えない空白文字・改行・特殊文字（スペース、タブ、改行など）が保存されている可能性がある

1.6 WHEN `isHomeHearingResultRequired()` 関数が実行される THEN `String(data.owned_home_hearing_inquiry).trim()` が空文字列を返すべきだが、trim前の値が truthy と判定されている

### Expected Behavior (Correct)

**問題1: 業者向けアンケート表示問題（買主7260）**

2.1 WHEN 買主詳細ページ（BuyerDetailPage.tsx）を表示する THEN システムは `buyer?.vendor_survey` を参照し、データベースから取得した値（「確認済み」など）を正しく表示する

2.2 WHEN 新規買主ページ（NewBuyerPage.tsx）で「業者向けアンケート」を保存する THEN システムは `vendor_survey` フィールド名でデータを送信し、バックエンドが正しく保存する

2.3 WHEN 買主詳細ページで「業者向けアンケート」フィールドを条件付き表示する THEN システムは `buyer?.vendor_survey` の値をチェックし、値がある場合のみ表示する

**問題2: 持家ヒアリング結果の誤必須問題（買主7267）**

2.4 WHEN 買主番号7267の詳細ページを表示する THEN 「問合時持家ヒアリング」が完全に空欄の場合、「持家ヒアリング結果」フィールドは必須扱いにならない（赤枠が表示されない）

2.5 WHEN `isHomeHearingResultRequired()` 関数が実行される THEN 空白文字のみの値を「空欄」として正しく判定し、`false` を返す

2.6 WHEN データベースの `owned_home_hearing_inquiry` カラムに空白文字のみが保存されている THEN システムは自動的にクリーンアップして `null` または空文字列に正規化する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 他のフィールド（問合時ヒアリング、初動担当など）を表示・編集する THEN システムは引き続き正しく動作する

3.2 WHEN 「業者向けアンケート」以外のボタン選択フィールド（Pinrich、3コール確認など）を操作する THEN システムは引き続き正しく動作する

3.3 WHEN データベースの `vendor_survey` カラムに保存されているデータを読み取る THEN システムは引き続き正しくデータを取得する

3.4 WHEN 「問合時持家ヒアリング」に実際に値が入力されている買主の詳細ページを表示する THEN 「持家ヒアリング結果」は引き続き必須扱いになる（正常な条件付き必須動作）

3.5 WHEN 他の条件付き必須フィールド（初動担当、★最新状況など）を表示・編集する THEN システムは引き続き正しく動作する
