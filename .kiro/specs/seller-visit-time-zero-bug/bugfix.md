# バグ修正要件ドキュメント

## はじめに

売主リストの通話モードページ（`/sellers/:id/call`）において、訪問日の日時入力フィールドに時刻を入力して保存すると、保存後に時刻が `0:00`（深夜0時）にリセットされてしまうバグ。

バックエンドから返される `visit_date` は UTC の ISO 8601 形式（例: `2026-05-10T15:00:00.000Z`）であり、フロントエンドで `new Date(sellerData.visitDate)` を使って変換する際に、ブラウザのローカルタイムゾーン（JST = UTC+9）が適用される。しかし保存時は `datetime-local` の値をそのままローカル時刻として扱い、タイムゾーン変換なしで送信している。この非対称な変換により、表示と保存の間でタイムゾーンのズレが生じ、時刻が `0:00` になる現象が発生している。

---

## バグ分析

### 現在の動作（不具合）

1.1 WHEN 通話モードページで訪問日に日時（例: 2026-05-10 10:00）を入力して保存する THEN システムは保存後に時刻を `0:00` にリセットして表示する

1.2 WHEN バックエンドが `visit_date` を UTC の ISO 8601 形式（例: `2026-05-10T01:00:00.000Z`）で返す THEN システムは `new Date()` でブラウザのローカルタイムゾーン（JST）に変換し、`datetime-local` フィールドに JST 時刻（例: `2026-05-10T10:00`）を表示する

1.3 WHEN ユーザーが `datetime-local` フィールドの値（例: `2026-05-10T10:00`）を保存する THEN システムはその値をタイムゾーン変換なしで `YYYY-MM-DD HH:mm:ss` 形式（例: `2026-05-10 10:00:00`）に変換してバックエンドに送信する

1.4 WHEN バックエンドが `2026-05-10 10:00:00`（JST として解釈）を受け取り Supabase に保存する THEN Supabase は UTC に変換して `2026-05-10T01:00:00.000Z` として保存する

1.5 WHEN 再度ページを読み込む THEN システムは `2026-05-10T01:00:00.000Z` を `new Date()` で JST に変換し `2026-05-10T10:00` を表示するため、一見正常に見える

1.6 WHEN 訪問日に日付のみ（時刻なし、例: `2026-05-10T00:00`）を入力して保存する THEN システムは `2026-05-10 00:00:00` をバックエンドに送信し、Supabase は `2026-05-09T15:00:00.000Z`（UTC）として保存する

1.7 WHEN 上記の `2026-05-09T15:00:00.000Z` を再読み込みする THEN システムは `new Date()` で JST に変換し `2026-05-10T00:00` を表示するため正常に見えるが、日付のみ入力の場合でも同様の問題が潜在する

1.8 WHEN `visit_date` が `YYYY-MM-DD` 形式（時刻なし）でバックエンドから返される THEN システムは `new Date("2026-05-10")` を UTC 深夜0時として解釈し、JST では `2026-05-10T09:00` と表示されるため、日付が正しく見えても時刻が `09:00` にずれる

### 期待される動作（正しい動作）

2.1 WHEN 通話モードページで訪問日に日時（例: 2026-05-10 10:00）を入力して保存する THEN システムは保存後も入力した時刻（10:00）を正しく表示する

2.2 WHEN バックエンドが `visit_date` を UTC の ISO 8601 形式で返す THEN システムは UTC 時刻をタイムゾーン変換なしで `YYYY-MM-DDTHH:mm` 形式に変換し、`datetime-local` フィールドに表示する（または JST 変換を一貫して適用する）

2.3 WHEN ユーザーが `datetime-local` フィールドの値を保存する THEN システムは表示時と同じタイムゾーン解釈で `YYYY-MM-DD HH:mm:ss` 形式に変換してバックエンドに送信する

2.4 WHEN バックエンドが日時文字列を受け取る THEN システムは受け取った値をタイムゾーン変換なしでそのまま Supabase に保存する（ローカル時刻として扱う）

2.5 WHEN 訪問日を保存して再読み込みする THEN システムは保存前と同じ日時を表示する（ラウンドトリップ一貫性）

### 変更しない動作（リグレッション防止）

3.1 WHEN 訪問日が設定されていない売主の通話モードページを開く THEN システムは訪問日フィールドを空欄のまま表示し続ける

3.2 WHEN 訪問日を保存する THEN システムは訪問取得日（visitAcquisitionDate）の自動設定ロジックを引き続き正しく動作させる

3.3 WHEN 訪問日を削除（空欄に）して保存する THEN システムは訪問取得日もクリアする既存の動作を維持する

3.4 WHEN 訪問日と営担（visitAssignee）を保存する THEN システムはカレンダーイベントの作成・更新を引き続き正しく行う

3.5 WHEN スプレッドシートと同期する THEN システムは `EnhancedAutoSyncService` の `combineVisitDateAndTime` による訪問日時の結合ロジックに影響を与えない

3.6 WHEN 訪問日前日のサイドバーステータス判定を行う THEN システムは `visit_date` の日付部分を正しく参照し続ける
