# バグ修正要件ドキュメント

## はじめに

売主管理システムの通話モードページ（CallModePage）において、「状況（当社）」フィールドを「一般媒介」に変更したとき、「専任（他決）決定日」フィールドの表示が即座に反映されないバグが存在します。保存操作後に初めて表示されることがあり、ユーザーが入力すべきフィールドに気づかないまま操作を進めてしまう可能性があります。

## バグ分析

### 現在の動作（不具合）

1.1 WHEN ユーザーが「状況（当社）」フィールドを「一般媒介」に変更したとき THEN システムは「専任（他決）決定日」フィールドを即座に表示しない

1.2 WHEN ユーザーが「状況（当社）」を「一般媒介」に変更して保存操作を行ったとき THEN システムは保存後に「専任（他決）決定日」フィールドを表示する（遅延表示）

### 期待される動作（正しい動作）

2.1 WHEN ユーザーが「状況（当社）」フィールドを「一般媒介」に変更したとき THEN システムは「専任（他決）決定日」フィールドを保存操作なしに即座に表示しなければならない（SHALL）

2.2 WHEN ユーザーが「状況（当社）」を「一般媒介」に変更したとき THEN システムは「一般媒介通知を送る」ボタンも即座に表示しなければならない（SHALL）

### 変更されない動作（リグレッション防止）

3.1 WHEN ユーザーが「状況（当社）」を「専任媒介」または「他決→追客」などの専任・他決系ステータスに変更したとき THEN システムは引き続き「専任（他決）決定日」フィールドを即座に表示し続けなければならない（SHALL CONTINUE TO）

3.2 WHEN ユーザーが「状況（当社）」を「追客中」などの一般媒介・専任・他決以外のステータスに変更したとき THEN システムは引き続き「専任（他決）決定日」フィールドを非表示にし続けなければならない（SHALL CONTINUE TO）

3.3 WHEN ユーザーが「専任（他決）決定日」フィールドに日付を入力したとき THEN システムは引き続きその値を保存操作時に正しく送信し続けなければならない（SHALL CONTINUE TO）

3.4 WHEN ページが初期表示されたとき THEN システムは引き続き保存済みの「状況（当社）」に応じて「専任（他決）決定日」フィールドの表示・非表示を正しく制御し続けなければならない（SHALL CONTINUE TO）
