# Bugfix Requirements Document

## Introduction

買主リストで新規買主を登録する際、買主番号が正しく採番されないバグ。
採番スプレッドシートのセルから現在の最大番号（例: 7358）を取得し、+1した値（7359）を新規番号として登録すべきところ、文字列連結により桁が増えた番号（73581）が登録されてしまう。

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN 採番スプレッドシートのセルに現在の最大買主番号（例: 7358）が格納されている状態で新規買主を登録する THEN システムは文字列連結により誤った番号（例: 73581）を生成し登録する

1.2 WHEN 採番ロジックが数値加算ではなく文字列連結で次の番号を計算する THEN システムは桁数が増えた不正な買主番号（例: "7358" + "1" = "73581"）をデータベースに保存する

### Expected Behavior (Correct)

2.1 WHEN 採番スプレッドシートのセルに現在の最大買主番号（例: 7358）が格納されている状態で新規買主を登録する THEN システムは数値として加算し正しい次の番号（例: 7359）を生成して登録する

2.2 WHEN 採番ロジックが次の番号を計算する THEN システムは現在の番号を数値型として扱い、整数加算（+1）により正しい連番を生成する

### Unchanged Behavior (Regression Prevention)

3.1 WHEN 採番スプレッドシートのセルに正常な数値型の買主番号が格納されている THEN システムは引き続き正しく+1した番号を採番する

3.2 WHEN 新規買主登録が成功する THEN システムは引き続き採番スプレッドシートのセルを新しい買主番号で更新する

3.3 WHEN 採番スプレッドシートへのアクセスが失敗する THEN システムは引き続きエラーをスローし登録を中断する

3.4 WHEN 採番スプレッドシートのセルが空欄または数値でない値を含む THEN システムは引き続き適切なエラーメッセージをスローする
