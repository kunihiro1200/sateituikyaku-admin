# 要件定義書

## はじめに

売主リストの通話モードページのステータスセクションには、4つのフィールド（専任（他決）決定日、競合（複数選択可）、専任・他決要因（複数選択可）、競合名、理由（他決、専任））が存在します。現在、これらのフィールドには保存機能がなく、DB→スプレッドシートへの即時同期ができない状態です。本要件では、これらのフィールドの保存機能とDB→スプレッドシート即時同期機能を追加します。

## 用語集

- **System**: 売主管理システム（社内管理システム）
- **CallModePage**: 通話モードページ（`/sellers/:id/call`）
- **StatusSection**: ステータスセクション（通話モードページ内の売主ステータス編集セクション）
- **SpreadsheetSync**: スプレッドシート同期サービス（DB→スプレッドシートへのデータ同期）
- **ExclusiveDecisionDate**: 専任（他決）決定日（AM列「契約年月 他決は分かった時点」）
- **Competitors**: 競合（複数選択可）（AN列「競合名」）
- **ExclusiveOtherDecisionFactors**: 専任・他決要因（複数選択可）（AQ列「専任・他決要因」）
- **CompetitorNameAndReason**: 競合名、理由（他決、専任）（AI列「競合名、理由（他決、専任）」、カラム名に改行あり）

## 要件

### 要件1: 専任（他決）決定日の保存とDB→スプレッドシート即時同期

**ユーザーストーリー**: 営業担当者として、通話モードページで専任（他決）決定日を入力・保存し、スプレッドシートに即座に反映させたい。

#### 受入基準

1. WHEN ユーザーが通話モードページのステータスセクションで専任（他決）決定日を入力し保存ボタンをクリックする、THEN THE System SHALL データベースの`sellers.contract_year_month`カラムに日付を保存する
2. WHEN データベースへの保存が成功する、THEN THE SpreadsheetSync SHALL スプレッドシートのAM列「契約年月 他決は分かった時点」に日付を即座に同期する
3. WHEN ユーザーが専任（他決）決定日を空欄にして保存する、THEN THE System SHALL データベースとスプレッドシートの両方から日付を削除する
4. WHEN スプレッドシート同期が失敗する、THEN THE System SHALL エラーメッセージを表示し、ユーザーに再試行を促す

### 要件2: 競合（複数選択可）の保存とDB→スプレッドシート即時同期

**ユーザーストーリー**: 営業担当者として、通話モードページで競合他社を複数選択し保存し、スプレッドシートに即座に反映させたい。

#### 受入基準

1. WHEN ユーザーが通話モードページのステータスセクションで競合を複数選択し保存ボタンをクリックする、THEN THE System SHALL データベースの`sellers.competitor_name`カラムにカンマ区切り文字列として保存する
2. WHEN データベースへの保存が成功する、THEN THE SpreadsheetSync SHALL スプレッドシートのAN列「競合名」にカンマ区切り文字列として即座に同期する
3. WHEN ユーザーが競合を全て解除して保存する、THEN THE System SHALL データベースとスプレッドシートの両方から競合名を削除する
4. WHEN スプレッドシート同期が失敗する、THEN THE System SHALL エラーメッセージを表示し、ユーザーに再試行を促す

### 要件3: 専任・他決要因（複数選択可）の保存とDB→スプレッドシート即時同期

**ユーザーストーリー**: 営業担当者として、通話モードページで専任・他決要因を複数選択し保存し、スプレッドシートに即座に反映させたい。

#### 受入基準

1. WHEN ユーザーが通話モードページのステータスセクションで専任・他決要因を複数選択し保存ボタンをクリックする、THEN THE System SHALL データベースの`sellers.exclusive_other_decision_factor`カラムにカンマ区切り文字列として保存する
2. WHEN データベースへの保存が成功する、THEN THE SpreadsheetSync SHALL スプレッドシートのAQ列「専任・他決要因」にカンマ区切り文字列として即座に同期する
3. WHEN ユーザーが専任・他決要因を全て解除して保存する、THEN THE System SHALL データベースとスプレッドシートの両方から専任・他決要因を削除する
4. WHEN スプレッドシート同期が失敗する、THEN THE System SHALL エラーメッセージを表示し、ユーザーに再試行を促す

### 要件4: 競合名、理由（他決、専任）の保存とDB→スプレッドシート即時同期

**ユーザーストーリー**: 営業担当者として、通話モードページで競合名や専任・他決になった理由の詳細を入力・保存し、スプレッドシートに即座に反映させたい。

#### 受入基準

1. WHEN ユーザーが通話モードページのステータスセクションで競合名、理由（他決、専任）を入力し保存ボタンをクリックする、THEN THE System SHALL データベースの`sellers.competitor_name_and_reason`カラムにテキストを保存する
2. WHEN データベースへの保存が成功する、THEN THE SpreadsheetSync SHALL スプレッドシートのAI列「競合名、理由（他決、専任）」にテキストを即座に同期する
3. WHEN ユーザーが競合名、理由（他決、専任）を空欄にして保存する、THEN THE System SHALL データベースとスプレッドシートの両方からテキストを削除する
4. WHEN スプレッドシート同期が失敗する、THEN THE System SHALL エラーメッセージを表示し、ユーザーに再試行を促す

### 要件5: スプレッドシート同期の即時性

**ユーザーストーリー**: 営業担当者として、保存ボタンをクリックした直後にスプレッドシートに反映されることを確認したい。

#### 受入基準

1. WHEN ユーザーが保存ボタンをクリックする、THEN THE System SHALL データベース保存とスプレッドシート同期を同期的に実行する
2. WHEN スプレッドシート同期が完了する、THEN THE System SHALL 成功メッセージを表示する
3. WHEN スプレッドシート同期が5秒以内に完了しない、THEN THE System SHALL タイムアウトエラーを表示する
4. WHEN ユーザーがスプレッドシートを開く、THEN THE SpreadsheetSync SHALL 保存した値が即座に反映されていることを確認できる

### 要件6: エラーハンドリング

**ユーザーストーリー**: 営業担当者として、保存やスプレッドシート同期が失敗した場合に、明確なエラーメッセージを受け取りたい。

#### 受入基準

1. WHEN データベース保存が失敗する、THEN THE System SHALL 「データベースへの保存に失敗しました」というエラーメッセージを表示する
2. WHEN スプレッドシート同期が失敗する、THEN THE System SHALL 「スプレッドシートへの同期に失敗しました」というエラーメッセージを表示する
3. WHEN ネットワークエラーが発生する、THEN THE System SHALL 「ネットワークエラーが発生しました。再試行してください」というエラーメッセージを表示する
4. IF エラーが発生する、THEN THE System SHALL ユーザーに再試行ボタンを表示する

### 要件7: バリデーション

**ユーザーストーリー**: 営業担当者として、必須項目が未入力の場合に保存できないようにしたい。

#### 受入基準

1. WHEN ユーザーが専任または他決を含むステータスを選択する、THEN THE System SHALL 専任（他決）決定日、競合（複数選択可）、専任・他決要因（複数選択可）を必須項目として扱う
2. WHEN ユーザーが必須項目を未入力のまま保存ボタンをクリックする、THEN THE System SHALL 「必須項目を入力してください」というエラーメッセージを表示する
3. WHEN ユーザーが一般媒介ステータスを選択する、THEN THE System SHALL 競合（複数選択可）と専任・他決要因（複数選択可）を任意項目として扱う
4. WHEN ユーザーが全ての必須項目を入力する、THEN THE System SHALL 保存ボタンを有効化する

### 要件8: スプレッドシートカラムマッピング

**ユーザーストーリー**: システム管理者として、データベースカラムとスプレッドシートカラムの正しいマッピングを確認したい。

#### 受入基準

1. THE System SHALL データベースの`sellers.contract_year_month`カラムをスプレッドシートのAM列「契約年月 他決は分かった時点」にマッピングする
2. THE System SHALL データベースの`sellers.competitor_name`カラムをスプレッドシートのAN列「競合名」にマッピングする
3. THE System SHALL データベースの`sellers.exclusive_other_decision_factor`カラムをスプレッドシートのAQ列「専任・他決要因」にマッピングする
4. THE System SHALL データベースの`sellers.competitor_name_and_reason`カラムをスプレッドシートのAI列「競合名、理由（他決、専任）」にマッピングする
