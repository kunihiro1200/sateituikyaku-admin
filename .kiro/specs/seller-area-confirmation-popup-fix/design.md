# 土地面積確認ポップアップ繰り返し表示問題 Bugfix Design

## Overview

売主通話モードページの査定計算セクションにおいて、固定資産税路線価フィールドに入力する際に表示される土地面積警告ポップアップが、ユーザーが「確認しました」ボタンを押した後も繰り返し表示される問題を修正する。

現在の実装では、警告確認状態がReactのステート（`landAreaWarning`）のみで管理されており、フィールドに再度フォーカスするたびに警告条件が再評価されて同じポップアップが表示される。この問題を解決するため、セッションストレージを使用して確認状態を保存し、同一セッション内では一度確認した警告を再表示しないようにする。

## Glossary

- **Bug_Condition (C)**: 土地面積が99㎡以下または建物面積より小さい場合に、固定資産税路線価フィールドへの入力時に警告ポップアップが繰り返し表示される条件
- **Property (P)**: ユーザーが「確認しました」ボタンを押した後、同じ警告ポップアップが再表示されないという期待される動作
- **Preservation**: 土地面積が100㎡以上かつ建物面積以上の場合は警告を表示しない、自動計算機能は継続する、ページリロード時は確認状態をリセットするという既存の動作
- **landAreaWarning**: `CallModePage.tsx`のReactステート変数で、警告メッセージを保持する（現在の実装）
- **sessionStorage**: ブラウザのセッションストレージAPI。ページリロードで自動的にクリアされ、タブごとに独立したストレージを提供する

## Bug Details

### Bug Condition

バグは、ユーザーが固定資産税路線価フィールドに入力しようとする際に発生する。`CallModePage.tsx`の`onChange`ハンドラーが、土地面積と建物面積の条件を毎回評価し、条件に該当する場合は`setLandAreaWarning()`を呼び出して警告ポップアップを表示する。ユーザーが「確認しました」ボタンを押すと`setLandAreaWarning(null)`が実行されてポップアップは閉じるが、確認状態は保存されない。そのため、フィールドに再度フォーカスして入力しようとすると、同じ条件評価が再度実行され、同じ警告が表示される。

**Formal Specification:**
```
FUNCTION isBugCondition(input)
  INPUT: input of type { landArea: number, buildingArea: number, hasConfirmed: boolean }
  OUTPUT: boolean
  
  RETURN input.landArea > 0 
         AND (input.landArea <= 99 OR (input.buildingArea > 0 AND input.landArea < input.buildingArea))
         AND NOT input.hasConfirmed
END FUNCTION
```

### Examples

- **AA13888の事例**: 土地面積30㎡、建物面積不明。固定資産税路線価フィールドをクリックするたびに「土地面積が30㎡（約9坪）ですが確認大丈夫ですか？」が表示される。「確認しました」を押しても、再度フィールドをクリックすると同じポップアップが表示される。
- **土地面積50㎡、建物面積80㎡の場合**: 土地面積が建物面積より小さいため、警告が表示される。確認後も繰り返し表示される。
- **土地面積99㎡の場合**: 99㎡以下のため警告が表示される。確認後も繰り返し表示される。
- **土地面積100㎡の場合**: 100㎡以上のため警告は表示されない（正常動作）。

## Expected Behavior

### Preservation Requirements

**Unchanged Behaviors:**
- 土地面積が100㎡以上かつ建物面積以上の場合は、警告ポップアップを表示しない
- 固定資産税路線価フィールドへの入力時、自動計算機能は引き続き実行される
- ページをリロードした場合、警告確認状態はリセットされ、次回の入力時に再度警告を表示する

**Scope:**
土地面積が警告条件に該当しない入力（100㎡以上かつ建物面積以上）は、この修正の影響を受けない。また、ページリロードや別の売主ページへの遷移時は、確認状態がリセットされるため、新しいセッションとして扱われる。

## Hypothesized Root Cause

バグの根本原因は以下の通り：

1. **確認状態の未保存**: `landAreaWarning`ステートは警告メッセージの表示/非表示のみを管理しており、「ユーザーが確認済みかどうか」という状態を保持していない。

2. **条件評価の繰り返し**: 固定資産税路線価フィールドの`onChange`ハンドラーが、入力のたびに土地面積の条件を評価し、条件に該当すれば無条件に`setLandAreaWarning()`を呼び出す。

3. **確認フラグの欠如**: 「確認しました」ボタンをクリックしても、`setLandAreaWarning(null)`でポップアップを閉じるだけで、「この警告は確認済み」というフラグが設定されない。

4. **セッション管理の欠如**: 確認状態をセッション単位で管理する仕組みがないため、同じセッション内で同じ警告が繰り返し表示される。

## Correctness Properties

Property 1: Bug Condition - 警告確認後の再表示防止

_For any_ 入力において、土地面積が99㎡以下または建物面積より小さい場合に警告ポップアップが表示され、ユーザーが「確認しました」ボタンを押した後は、同じセッション内で固定資産税路線価フィールドに再度入力しても、同じ警告ポップアップは表示されない。

**Validates: Requirements 2.1, 2.2, 2.3, 2.4**

Property 2: Preservation - 既存動作の保持

_For any_ 入力において、土地面積が100㎡以上かつ建物面積以上の場合は警告ポップアップを表示せず、自動計算機能は引き続き実行され、ページリロード時は確認状態がリセットされて次回の入力時に再度警告を表示する。

**Validates: Requirements 3.1, 3.2, 3.3, 3.4**

## Fix Implementation

### Changes Required

修正は`frontend/frontend/src/pages/CallModePage.tsx`のみで完結する。

**File**: `frontend/frontend/src/pages/CallModePage.tsx`

**Specific Changes**:

1. **確認状態管理用のステート追加**:
   - `landAreaWarningConfirmed`という新しいステート変数を追加
   - 初期値は`sessionStorage`から取得（キー: `landAreaWarningConfirmed_${sellerId}`）
   - ページリロード時は自動的にクリアされる（sessionStorageの仕様）

2. **警告表示条件の修正**:
   - 固定資産税路線価フィールドの`onChange`ハンドラー内で、警告条件に該当する場合でも、`landAreaWarningConfirmed`が`true`の場合は警告を表示しない
   - 条件: `if (landNum > 0 && (landNum <= 99 || (buildingNum > 0 && landNum < buildingNum)) && !landAreaWarningConfirmed)`

3. **確認ボタンのハンドラー修正**:
   - 「確認しました」ボタンをクリックした際、`setLandAreaWarning(null)`に加えて、`setLandAreaWarningConfirmed(true)`を実行
   - `sessionStorage.setItem('landAreaWarningConfirmed_${sellerId}', 'true')`で確認状態を保存

4. **売主変更時の確認状態リセット**:
   - `useEffect`で`sellerId`の変化を監視
   - 売主が変更された場合、`setLandAreaWarningConfirmed(false)`を実行
   - 新しい売主の確認状態を`sessionStorage`から読み込む

5. **ページリロード時の自動リセット**:
   - `sessionStorage`はページリロード時に自動的にクリアされるため、追加の実装は不要
   - ただし、明示的にクリアする場合は`sessionStorage.removeItem('landAreaWarningConfirmed_${sellerId}')`を使用

## Testing Strategy

### Validation Approach

テスト戦略は2段階のアプローチを採用する。まず、修正前のコードで警告が繰り返し表示されることを確認し（探索的バグ条件チェック）、次に修正後のコードで警告が一度だけ表示され、確認後は再表示されないことを検証する（修正チェック）。さらに、既存の動作（土地面積が100㎡以上の場合は警告を表示しない、ページリロード時は確認状態をリセット）が保持されることを確認する（保全チェック）。

### Exploratory Bug Condition Checking

**Goal**: 修正前のコードで、警告ポップアップが繰り返し表示されることを確認する。

**Test Plan**: 売主AA13888の通話モードページを開き、固定資産税路線価フィールドに値を入力する。警告ポップアップが表示されたら「確認しました」ボタンを押し、再度フィールドに入力する。警告が再度表示されることを確認する。

**Test Cases**:
1. **AA13888の警告繰り返しテスト**: 土地面積30㎡の売主で、固定資産税路線価フィールドに「100000」を入力 → 警告表示 → 「確認しました」 → 再度「200000」を入力 → 警告が再度表示される（修正前は失敗）
2. **土地面積50㎡、建物面積80㎡のテスト**: 土地面積が建物面積より小さい売主で、同様の操作を実行 → 警告が繰り返し表示される（修正前は失敗）
3. **土地面積99㎡のテスト**: 99㎡以下の売主で、同様の操作を実行 → 警告が繰り返し表示される（修正前は失敗）
4. **土地面積100㎡のテスト**: 100㎡以上の売主で、固定資産税路線価フィールドに入力 → 警告は表示されない（修正前でも成功）

**Expected Counterexamples**:
- 「確認しました」ボタンを押した後も、固定資産税路線価フィールドに再度入力すると同じ警告が表示される
- 確認状態が保存されていないため、何度でも同じ警告が表示される

### Fix Checking

**Goal**: 修正後のコードで、警告ポップアップが一度だけ表示され、確認後は再表示されないことを検証する。

**Pseudocode:**
```
FOR ALL input WHERE isBugCondition(input) DO
  result := handleFixedAssetTaxInput_fixed(input)
  ASSERT expectedBehavior(result)
END FOR
```

**Test Plan**: 修正後のコードで、売主AA13888の通話モードページを開き、固定資産税路線価フィールドに値を入力する。警告ポップアップが表示されたら「確認しました」ボタンを押し、再度フィールドに入力する。警告が再表示されないことを確認する。

**Test Cases**:
1. **AA13888の警告一度のみテスト**: 土地面積30㎡の売主で、固定資産税路線価フィールドに「100000」を入力 → 警告表示 → 「確認しました」 → 再度「200000」を入力 → 警告は表示されない（修正後は成功）
2. **土地面積50㎡、建物面積80㎡のテスト**: 土地面積が建物面積より小さい売主で、同様の操作を実行 → 警告は一度だけ表示される（修正後は成功）
3. **土地面積99㎡のテスト**: 99㎡以下の売主で、同様の操作を実行 → 警告は一度だけ表示される（修正後は成功）
4. **複数回入力テスト**: 確認後、固定資産税路線価フィールドに複数回入力しても、警告は表示されない（修正後は成功）

### Preservation Checking

**Goal**: 修正後のコードで、既存の動作（土地面積が100㎡以上の場合は警告を表示しない、ページリロード時は確認状態をリセット）が保持されることを確認する。

**Pseudocode:**
```
FOR ALL input WHERE NOT isBugCondition(input) DO
  ASSERT handleFixedAssetTaxInput_original(input) = handleFixedAssetTaxInput_fixed(input)
END FOR
```

**Test Plan**: 修正後のコードで、土地面積が100㎡以上の売主の通話モードページを開き、固定資産税路線価フィールドに入力する。警告が表示されないことを確認する。また、警告を確認した後にページをリロードし、再度フィールドに入力すると警告が再度表示されることを確認する。

**Test Cases**:
1. **土地面積100㎡以上のテスト**: 土地面積が100㎡以上かつ建物面積以上の売主で、固定資産税路線価フィールドに入力 → 警告は表示されない（修正後も成功）
2. **自動計算機能のテスト**: 固定資産税路線価フィールドに値を入力した後、自動計算が実行されることを確認（修正後も成功）
3. **ページリロード後のテスト**: 警告を確認した後、ページをリロード → 固定資産税路線価フィールドに入力 → 警告が再度表示される（修正後も成功）
4. **別の売主への遷移テスト**: 警告を確認した後、別の売主の通話モードページに遷移 → 固定資産税路線価フィールドに入力 → 新しい売主で警告条件に該当する場合は警告が表示される（修正後も成功）
