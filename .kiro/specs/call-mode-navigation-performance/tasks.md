- [x] 3. 通話モードナビゲーションパフォーマンスの修正

  - [x] 3.1 CallModePage.tsx のサイドバーデータ取得を非同期化する
    - `frontend/frontend/src/pages/CallModePage.tsx` を修正
    - `loadAllData` で売主詳細データ（`/api/sellers/:id`）を優先的に表示する
    - `fetchSidebarSellers`（pageSize=500）と `fetchSidebarCounts` をバックグラウンドで非同期取得する
    - サイドバーデータ取得中はローディングインジケーターを表示し、メインコンテンツはすでに表示済みにする
    - _Bug_Condition: `loadAllData` 完了後に `fetchSidebarSellers` と `fetchSidebarCounts` を逐次実行（合計5本のAPIコール）_
    - _Expected_Behavior: 売主詳細データを優先表示し、サイドバーデータはバックグラウンドで取得_
    - _Preservation: 売主情報の正確な表示・サイドバーフィルタリングの正確性を維持_

  - [x] 3.2 SellerService.supabase.ts に listSellers キャッシュを追加する
    - `backend/src/services/SellerService.supabase.ts` を修正（売主管理システム用、ポート3000）
    - `listSellers` メソッドにインメモリキャッシュを追加する
    - キャッシュが有効な場合はDBクエリと `decryptSeller`（暗号化復号 + 従業員名変換）をスキップする
    - キャッシュTTLを適切な値（例: 60秒）に設定する
    - _Bug_Condition: `listSellers` 呼び出し時に毎回全件 `decryptSeller` を `Promise.all` で並列実行_
    - _Expected_Behavior: キャッシュが有効な場合はDBクエリと復号処理をスキップして即座にレスポンスを返す_
    - _Preservation: キャッシュ切れ時は引き続き正確なデータを返す_
    - _Requirements: 2.2, 2.3, 3.1, 3.3_

  - [x] 3.3 SellerService.supabase.ts に getSidebarCounts キャッシュを追加する
    - `backend/src/services/SellerService.supabase.ts` を修正（売主管理システム用、ポート3000）
    - `getSidebarCounts` メソッドにインメモリキャッシュを追加する
    - キャッシュが有効な場合は複数の独立したDBクエリをスキップする
    - キャッシュTTLを適切な値（例: 60秒）に設定する
    - _Bug_Condition: `getSidebarCounts` 呼び出し時に毎回複数の独立したDBクエリを逐次実行_
    - _Expected_Behavior: キャッシュが有効な場合はキャッシュから即座にカウントを返す_
    - _Preservation: キャッシュ切れ時は引き続き正確なカウントを返す_
