# Google Maps URL 座標抽出ルール

## ⚠️ 重要：Google Maps URLには座標が2種類含まれる

Google Maps のURLには**2種類の座標**が含まれており、必ず正しい方を使うこと。

---

## 📍 座標の種類

### ❌ 間違い：表示中心座標（`/@lat,lng,zoom`）

```
https://www.google.com/maps/place/.../@33.551585,130.3401374,17z/...
```

- `/@` の直後に続く座標
- **地図の表示中心位置**であり、ピンの場所ではない
- ズームレベル（例: `17z`）が後ろに続く
- **この座標を使うと場所がズレる**

### ✅ 正解：ピン座標（`!3d lat !4d lng`）

```
https://www.google.com/maps/place/...!3d33.5507075!4d130.3450583...
```

- `!3d` の後が緯度、`!4d` の後が経度
- **実際にピンが立っている正確な場所**
- **必ずこちらを優先して使うこと**

---

## 📋 実例（FI101の地図URL）

展開後URL:
```
https://www.google.com/maps/place/〒814-0164+福岡県福岡市早良区賀茂２丁目１０−１１
/@33.551585,130.3401374,17z
/data=!4m6!3m5!1s0x3541948929b4f285:0x4073776df5fe9280
!8m2!3d33.5507075!4d130.3450583
!16s/g/11vkfhf3f8
```

| 種類 | 緯度 | 経度 | 正確か |
|------|------|------|--------|
| `/@`（表示中心） | 33.551585 | 130.3401374 | ❌ ズレる |
| `!3d!4d`（ピン） | 33.5507075 | 130.3450583 | ✅ 正確 |

---

## 🔧 `COORD_PATTERNS` の正しい優先順位

`NearbyMapModal.tsx` の `COORD_PATTERNS` は以下の順序にすること：

```typescript
const COORD_PATTERNS = [
  /!3d(-?\d+\.?\d*)!4d(-?\d+\.?\d*)/,          // 最優先：ピン座標
  /[?&]q=(-?\d+\.?\d*),(-?\d+\.?\d*)/,          // 検索クエリ座標
  /\/search\/(-?\d+\.?\d*),\+?(-?\d+\.?\d*)/,
  /\/place\/[^/]*\/@(-?\d+\.?\d*),(-?\d+\.?\d*)/,
  /\/place\/(-?\d+\.?\d*),(-?\d+\.?\d*)/,
  /\/@(-?\d+\.?\d*),(-?\d+\.?\d*),/,            // 表示中心座標（最後の手段）
  /ll=(-?\d+\.?\d*),(-?\d+\.?\d*)/,
];
```

**`!3d!4d` を先頭に、`/@` を後ろに置くこと。絶対に逆にしない。**

## 🚨 過去の障害

**発生日**: 2026年7月  
**症状**: きんりんMAP（近隣MAP）で物件の場所が大きくズレて表示された  
**原因**: `COORD_PATTERNS` で `/@`（表示中心座標）が `!3d!4d`（ピン座標）より前にあったため、間違った座標を使用  
**修正**: `!3d!4d` パターンを先頭に移動

---

## 📝 短縮URL（maps.app.goo.gl）の展開について

`maps.app.goo.gl` の短縮URLはサーバーサイドから展開すると以下の問題がある：

1. **consent.google.com にリダイレクトされる**ことがある
   - `continue` パラメータのURLをさらに追跡すること
   - `continue` パラメータを直接返して終了しない

2. **展開後URLに `!3d!4d` と `/@` 両方が含まれる**
   - 必ず `!3d!4d`（ピン座標）を優先すること
