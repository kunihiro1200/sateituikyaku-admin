# GASデバッグチェックリスト（絶対に守るべきルール）

## ⚠️ 重要：バグ修正は全体分析から始める

**このチェックリストを守ることで、10時間かかる修正を1時間で終わらせることができます。**

---

## 📋 バグ修正の手順（必ず守る）

### ステップ1: 問題のフィールドを特定

**質問**:
- どのフィールドが正しく同期されていないか？
- スプレッドシートの列名は？
- データベースのカラム名は？

**例**:
- フィールド: `viewing_date`（内覧日）
- スプレッドシート: 列I「●内覧日(最新）」
- データベース: `viewing_date`カラム

---

### ステップ2: コード全体を検索（最重要）

**❌ 間違ったアプローチ**:
```
部分的に修正 → テスト → 別の問題発見 → 修正 → テスト...
```

**✅ 正しいアプローチ**:
```bash
# 問題のフィールドが何回設定されているか確認
grep -n "viewing_date" gas_buyer_complete_code.js

# 問題のフィールドが何回参照されているか確認
grep -n "row\['●内覧日" gas_buyer_complete_code.js
```

**確認ポイント**:
- [ ] フィールドが**複数回設定**されていないか？
- [ ] 2回目の設定で**上書き**されていないか？
- [ ] **空の列**から値を取得していないか？

---

### ステップ3: 全ての設定箇所をリストアップ

**例**（今回の問題）:
1. **1回目の設定**（552-557行目）:
   ```javascript
   var sheetViewingDate = formatDateToISO_(row['●内覧日(最新）']);
   if (sheetViewingDate) {
     updateData.viewing_date = sheetViewingDate;  // ✅ 正しい値
   }
   ```

2. **2回目の設定**（616-625行目）:
   ```javascript
   var sheetLatestViewingDate = formatDateToISO_(row['最新内覧日']);
   if (sheetLatestViewingDate) {
     updateData.viewing_date = sheetLatestViewingDate;  // ❌ 空の列から取得 → null
   }
   ```

**問題**: 2回目の設定で `null` に上書きされていた

---

### ステップ4: デバッグログを追加

**最初からやるべきこと**:
- 問題のフィールドに関する全ての処理にデバッグログを追加
- 「生データ」「変換後」「設定前」「設定後」を全て記録

**例**:
```javascript
// 🚨 デバッグ: 買主7282の内覧日を記録
if (buyerNumber === '7282') {
  Logger.log('  📅 [DEBUG] 内覧日（生データ）: ' + rawViewingDate);
  Logger.log('    型: ' + typeof rawViewingDate);
  Logger.log('    Date型か: ' + (rawViewingDate instanceof Date));
}

var sheetViewingDate = formatDateToISO_(rawViewingDate);

// 🚨 デバッグ: 買主7282の内覧日変換結果を記録
if (buyerNumber === '7282') {
  Logger.log('  📅 [DEBUG] 内覧日（変換後）: ' + sheetViewingDate);
}

if (sheetViewingDate) {
  updateData.viewing_date = sheetViewingDate;
  Logger.log('  📅 ' + buyerNumber + ': 内覧日を設定 (' + sheetViewingDate + ')');
} else if (buyerNumber === '7282') {
  Logger.log('  ⚠️ [DEBUG] 内覧日の変換に失敗しました');
}
```


**正解**: 最初にユーザーに詳細を確認する
