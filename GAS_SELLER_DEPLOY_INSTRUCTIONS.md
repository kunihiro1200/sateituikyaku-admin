# 売主リストGAS同期スクリプト デプロイ手順

## 📋 概要

ローカルの`gas_complete_code.js`の最新コードをGASプロジェクト（売主リストスプレッドシートに紐づいたGASエディタ）にデプロイします。

**重要**: サイドバーカウント自動更新機能が含まれています。

---

## 🚨 今回の修正内容

### 問題

AA13899が「当日TEL」カテゴリに含まれていなかった。

### 根本原因

1. GASの10分トリガーで`updateSidebarCounts_`関数が無効化されていた
2. そのため、売主データが更新されても、サイドバーカウントが自動更新されなかった

### 修正内容

GASの10分トリガーで、バックエンドのAPIエンドポイント`/api/sellers/sidebar-counts/update`を呼び出すように修正しました。

```javascript
// サイドバーカウント更新（バックエンドAPIを呼び出し）
try {
  Logger.log('📊 サイドバーカウント更新を開始...');
  var sidebarResponse = postToBackend('/api/sellers/sidebar-counts/update', {});
  var sidebarStatusCode = sidebarResponse.getResponseCode();
  if (sidebarStatusCode >= 200 && sidebarStatusCode < 300) {
    Logger.log('✅ サイドバーカウント更新成功');
  } else {
    Logger.log('❌ サイドバーカウント更新失敗: HTTP ' + sidebarStatusCode);
  }
} catch (e) { Logger.log('❌ サイドバーカウント更新エラー: ' + e.toString()); }
```

## 🚀 デプロイ手順

### ステップ1: ローカルのGASコードを確認

1. `gas_complete_code.js`を開く
2. 以下のコードが含まれていることを確認：
   ```javascript
   // サイドバーカウント更新（バックエンドAPIを呼び出し）
   try {
     Logger.log('📊 サイドバーカウント更新を開始...');
     var sidebarResponse = postToBackend('/api/sellers/sidebar-counts/update', {});
   ```

### ステップ2: GASエディタを開く

1. **売主リストスプレッドシート**を開く
2. メニューから「拡張機能」→「Apps Script」を選択
3. GASエディタが開く

### ステップ3: 最新コードをコピー

1. ローカルの`gas_complete_code.js`の内容を**全て**コピー
2. GASエディタの`コード.gs`（またはメインスクリプトファイル）に貼り付け
3. **保存**（Ctrl+S または ⌘+S）

### ステップ4: テスト実行

1. GASエディタで関数を選択：`syncSellersToSupabase_`
2. 「実行」ボタンをクリック
3. 初回実行時は権限の承認が必要
4. 実行ログを確認：
   ```
   === 同期開始 ===
   📊 サイドバーカウント更新を開始...
   ✅ サイドバーカウント更新成功
   === 同期完了 ===
   ```

### ステップ5: サイドバーカウント更新を確認

1. 実行ログに「✅ サイドバーカウント更新成功」が表示されることを確認
2. エラーがないことを確認

## 🔍 トラブルシューティング

### エラー: "❌ サイドバーカウント更新失敗: HTTP 500"

**原因**: バックエンドのAPIエンドポイントでエラーが発生

**解決策**: 
1. Vercelのログを確認
2. バックエンドのデプロイが完了しているか確認
3. `/api/sellers/sidebar-counts/update`エンドポイントが正常に動作しているか確認

### エラー: "❌ サイドバーカウント更新エラー: Exception: Request failed"

**原因**: バックエンドのURLが間違っている、またはネットワークエラー

**解決策**: 
1. `BACKEND_CONFIG.URL`が正しいか確認
2. バックエンドが起動しているか確認
