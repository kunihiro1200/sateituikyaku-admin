物件タイプフィルターボタン診断

1. ページアクセステスト

以下のURLにアクセスしてください:

http://localhost:5173/public/properties

2. ブラウザ開発者ツールでの確認手順

  1. F12キーを押して開発者ツールを開く
  2. Consoleタブを開く
  3. 以下のコマンドを実行: 
    document.querySelector('.space-y-3')
    結果がnull以外なら、コンポーネントは存在します
  4. 以下のコマンドで物件タイプボタンを確認: 
    document.querySelectorAll('button[aria-label*="で絞り込む"]')
    結果が4つのボタンなら正常です

3. Elementsタブでの確認

  1. Elementsタブを開く
  2. Ctrl+F で検索ボックスを開く
  3. 「物件タイプ」で検索
  4. 以下のような構造が見つかるか確認: 
    <div class="space-y-3">
  <div class="text-sm font-medium text-gray-700">
    物件タイプ
  </div>
  <div class="flex flex-wrap gap-2">
    <button ...>戸建て</button>
    <button ...>マンション</button>
    <button ...>土地</button>
    <button ...>収益物件</button>
  </div>
</div>

4. Networkタブでの確認

  1. Networkタブを開く
  2. ページをリロード (Ctrl+R)
  3. 「PropertyTypeFilterButtons」で検索
  4. 該当ファイルが読み込まれているか確認

5. コンソールでのデバッグコマンド

以下のコマンドをConsoleタブで実行して、詳細情報を取得:

// 物件タイプフィルターセクション全体を取得
const filterSection = document.querySelector('.space-y-3');
console.log('フィルターセクション:', filterSection);

// ボタンを全て取得
const buttons = document.querySelectorAll('button[aria-label*="で絞り込む"]');
console.log('ボタン数:', buttons.length);
buttons.forEach((btn, i) => {
    console.log(`ボタン${i+1}:`, btn.textContent, btn.className);
});

// 親要素を確認
if (filterSection) {
    console.log('親要素:', filterSection.parentElement);
    console.log('表示状態:', window.getComputedStyle(filterSection).display);
    console.log('可視性:', window.getComputedStyle(filterSection).visibility);
}

6. 問題が見つかった場合

  • コンポーネントが見つからない → Reactコンポーネントが正しくマウントされていない可能性
  • ボタンが0個 → レンダリングロジックに問題がある可能性
  • display: none → CSSで非表示になっている可能性
  • Networkエラー → ビルドやバンドルに問題がある可能性
