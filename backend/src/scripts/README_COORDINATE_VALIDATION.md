# 座標検証・自動修正スクリプト

## 概要

売主の物件住所と座標（緯度・経度）の整合性を自動検証し、問題がある場合は修正を提案するスクリプトです。

## 機能

### 1. 座標の妥当性検証

- **逆ジオコーディング**: 現在の座標から住所を取得
- **住所比較**: 元の住所と逆ジオコーディングで取得した住所を比較
- **距離計算**: 現在の座標と正しい座標の距離を計算
- **閾値チェック**: 1km以上ずれている場合は修正を推奨

### 2. 自動修正

- 問題のある座標を検出した場合、正しい座標を提案
- ユーザーの確認後、データベースを自動更新

## 使い方

### 全売主の座標を検証（最新100件）

```bash
cd backend
npm run validate-coordinates
```

### 特定の売主の座標を検証

```bash
cd backend
npm run validate-coordinates:single AA14088
```

## 検証の流れ

### ステップ1: 逆ジオコーディング

現在の座標（緯度・経度）から住所を取得します。

```
例: 座標 (33.2370297, 131.6016492)
→ 「日本、〒870-0028 大分県大分市新町３」
```

### ステップ2: 住所比較

元の住所と逆ジオコーディングで取得した住所を比較します。

```
元の住所: 「大分市大字光吉光吉新町3-5-10」
逆ジオコーディング: 「日本、〒870-0028 大分県大分市新町３」
→ 一致しない！
```

### ステップ3: 再ジオコーディング

元の住所を再度ジオコーディングして、正しい座標を取得します。

```
住所: 「大分市大字光吉光吉新町3-5-10」
→ 座標 (33.2370297, 131.6016492) ← 間違い（大分駅近く）

住所: 「大分市大字光吉3-5-10」（修正後）
→ 座標 (33.1975595, 131.598618) ← 正しい（光吉地区）
```

### ステップ4: 距離計算

現在の座標と正しい座標の距離を計算します。

```
距離: 4.40km
→ 1km以上ずれているため、修正を推奨
```

### ステップ5: 自動修正

ユーザーに確認を求め、承認された場合はデータベースを更新します。

```
=== 自動修正 ===
上記の座標を自動修正しますか？ (y/N)
y

修正中: AA14088
  ✅ 更新成功

✅ 全ての修正が完了しました
```

## よくある問題と原因

### 1. 住所に誤りがある

**例**: 「大分市大字光吉光吉新町3-5-10」

- 「光吉新町」という地名は存在しない
- Google Maps APIが「新町」として解釈してしまう
- 正しくは「大分市大字光吉3-5-10」

**対策**: 住所を修正してから再ジオコーディング

### 2. 座標が古い

- 住所は正しいが、座標が古いデータのまま
- 再ジオコーディングで最新の座標を取得

**対策**: 定期的に座標を検証・更新

### 3. Google Maps APIの誤認識

- 住所が曖昧で、Google Maps APIが間違った場所を返す
- 例: 「大字」「字」などの表記揺れ

**対策**: 住所を明確にする（「大分県」を追加するなど）

## 注意事項

### レート制限

Google Maps APIには1日あたりのリクエスト制限があります。

- **無料枠**: 1日あたり40,000リクエスト
- **このスクリプト**: 1売主あたり2〜3リクエスト（逆ジオコーディング + ジオコーディング）
- **推奨**: 一度に大量の売主を検証する場合は、複数回に分けて実行

### レート制限対策

スクリプトは自動的に200msの待機時間を設けています。

```typescript
// レート制限対策（200ms待機）
if (i > 0) {
  await new Promise((resolve) => setTimeout(resolve, 200));
}
```

### 環境変数

以下の環境変数が必要です：

- `GOOGLE_MAPS_API_KEY`: Google Maps Geocoding API キー
- `SUPABASE_URL`: Supabase URL
- `SUPABASE_SERVICE_ROLE_KEY`: Supabase Service Role Key

## トラブルシューティング

### エラー: GOOGLE_MAPS_API_KEY が設定されていません

**原因**: 環境変数が読み込まれていない

**対策**:
1. `backend/.env` に `GOOGLE_MAPS_API_KEY` が設定されているか確認
2. `.env.local` で上書きされていないか確認

### エラー: REQUEST_DENIED

**原因**: Google Maps APIキーが無効、または制限されている

**対策**:
1. Google Cloud Consoleで APIキーを確認
2. Geocoding APIが有効になっているか確認
3. APIキーの制限を確認（IPアドレス制限など）

### 座標が更新されない

**原因**: データベースの更新権限がない

**対策**:
1. `SUPABASE_SERVICE_ROLE_KEY` が正しく設定されているか確認
2. Supabaseのポリシーを確認

## 定期実行の推奨

### 月次実行

新しい売主が追加された際に、座標の妥当性を確認するため、月次で実行することを推奨します。

```bash
# 最新100件を検証
npm run validate-coordinates
```

### 新規売主追加時

新しい売主を追加した直後に、その売主の座標を検証することを推奨します。

```bash
# 特定の売主を検証
npm run validate-coordinates:single AA14088
```

## まとめ

このスクリプトを使用することで：

- ✅ 座標の誤りを自動検出
- ✅ 正しい座標を自動提案
- ✅ データベースを自動更新
- ✅ 手動修正の手間を削減

定期的に実行することで、地図表示の精度を維持できます。
